# カネとジカンとジョウネツと

> 人生における「遊ぶ時間」を、いかに充実させるか――。
> 1歳から100歳まで、99年間の選択を積み重ねながら「遊び尽くす」人生シミュレーションゲーム。

## 2. コンセプト

### 2.1 ひとことで言うと
**「人生99年間を、いかに全力で遊び尽くすか」** を問うシミュレーションゲーム。

### 2.2 ゲーム体験の核（コアループ）
1. **時間を割り振る**：1日24時間のうち、睡眠・食事・学業/仕事の必須時間を除いた「余剰時間」を、何の遊びに投資するかを決める。
2. **遊びを選ぶ**：ライフステージ・季節・所持金・体力・パラメーターなどの制約のなかから、遊びを選ぶ。
3. **原体験を得る**：遊びの結果、「原体験ポイント」が蓄積され、各種パラメーターが成長する。
4. **ジレンマに向き合う**：若いときは「時間はあるけどカネがない」、大人になると「カネはあるけど時間がない」という構造的ジレンマを味わう。
5. **人生の質を記録する**：最終的に「どれだけ自分の好きなことに没頭できたか」＝**ジョウネツ指数** がスコアとして残る。

### 2.3 コンセプトを支える3つの資源（=タイトルの由来）
- **カネ（お金）**：遊びの選択肢を広げる資源
- **ジカン（時間）**：そもそも遊びを実行するために必須の資源
- **ジョウネツ（情熱）**：好きなことに没頭し続ける意志・熱量。プレイヤーの「選択」によってのみ育つ
- この3つの資源を、ライフステージごとに **どう配分し、どう遊びに投資するか** が物語の主軸。

### 2.4 プレイヤーに残したい感情
- 「遊ぶって、こんなに大事だったんだ」
- 「自分の人生、今どのフェーズにいるんだろう」
- 「もう一度別のルートでやってみたい」
- プレイ後、**実生活の遊び・余暇の使い方を見直すきっかけ** になるゲーム体験を目指す。

## 3. ターゲット

### 3.1 メインターゲット
- **25〜45歳の社会人**（特に「最近遊んでないな…」と感じている層）
- スマホで短時間（1日10〜30分）のスキマ時間にプレイしたい層
- 人生シミュレーション系（『人生ゲーム』『BitLife』など）が好きな層

### 3.2 サブターゲット
- 中高生〜大学生：「大人になった自分」を擬似体験してみたい層
- 50代以降：自分の人生を振り返りつつ、別ルートをシミュレートしたい層

### 3.3 競合・参考タイトル
| タイトル | 参考にする点 | 差別化ポイント |
|---|---|---|
| BitLife | 選択式人生シミュレーション | **「遊び」に特化**している点 |
| 人生ゲーム | 日本人にとっての親しみやすさ | 1ターン=人生単位でなく **1日24時間単位** の解像度 |
| どうぶつの森 | 季節と遊びの関係性 | 季節による **遊びの強制変化** を人生設計に組み込む |
| Stardew Valley | 時間配分と余暇の設計 | **ジレンマ設計**（時間⇔カネ）を明示 |

## 4. ゲームシステム

### 4.1 基本構造
- 1プレイ = **1回の人生（99年間）**
- 1ターン = **1日（24時間）** or **1週間**（ライフステージで粒度が変化）
  - 未就学児〜小学生：1ターン = 1日（丁寧に遊びを体験させる）
  - 中高生〜大学生：1ターン = 1週間
  - 社会人以降：1ターン = 1ヶ月（ライフイベント+余暇の配分）
- ライフステージの進行は **年齢** によって自動的に切り替わる。

### 4.2 1日の時間配分（例：社会人）
```
[ 24時間 ]
 ├─ 睡眠（7h 固定、健康度で増減）
 ├─ 食事（2h 固定）
 ├─ 仕事/学業（8〜10h、職業で増減）
 └─ 余剰時間（4〜7h）← ★ここを「遊び」に投資する★
```

### 4.3 遊びの選択フロー
```
ライフステージ判定
  └─ 季節判定（春夏秋冬）
      └─ 遊び候補リスト表示
          └─ 制約チェック（年齢/身長/決定権/カネ/体力/パラメーター）
              └─ 遊びを選択
                  └─ 結果演出（原体験ポイント取得）
                      └─ パラメーター反映 & 資源消費
```

### 4.4 原体験ポイント
- 遊びに応じて **5種類の原体験カテゴリ** にポイントが蓄積される。
  - **身体系**：走る/登る/泳ぐ/雪遊びなど
  - **創作系**：粘土/工作/絵/音楽/プログラミングなど
  - **探求系**：本/ゲーム/科学/旅行など
  - **交流系**：友達遊び/恋愛/家族旅行/飲み会など
  - **競技系**：スポーツ/対戦ゲーム/大会出場など
- カテゴリごとにLv1〜Lv99まで成長、**Lvが上がると解放される遊び** がある（例：競技Lv50で「マラソン大会出場」解禁）。

### 4.5 ジョウネツ指数（スコア）
- 同じカテゴリに継続投資すると **ジョウネツ指数** が上昇（没頭ボーナス）。
- 逆に手広く浅くやるとジョウネツは伸びにくいが、原体験ポイントの総量は稼げる。
- **「広く浅く vs 狭く深く」** の戦略性をプレイヤーに委ねる。
- エンディングで「ジョウネツ指数」「原体験総量」「遊びのバリエーション」の3軸で評価。

### 4.6 遊びのジレンマ（本作の核）
| ライフステージ | 時間 | カネ | 代表的な制約 |
|---|---|---|---|
| 未就学児 | ◎ | × | 身長・決定権（親の許可） |
| 小学生 | ◎ | △ | 小遣い・行動範囲 |
| 中高生 | ○ | △ | 部活・勉学・門限 |
| 大学生 | ◎ | △ | 学費・バイト・単位 |
| 社会人（独身） | △ | ○ | 仕事時間・体力 |
| 社会人（既婚） | × | ○ | 家族時間・育児 |
| 育児期 | × | ○ | ほぼ全時間が育児に |
| 定年後 | ◎ | ○〜△ | 体力・健康 |
| 老後 | ◎ | △ | 体力・移動能力 |

この **「ある資源が潤沢なときは別の資源が足りない」** ジレンマを、全ライフステージで体験させる設計とする。

## 5. ライフステージ設計

日本基準で9段階＋エンディング。

### 5.1 未就学児（1〜6歳）
- **制約**：年齢、身長、決定権（親の許可制）、季節
- **リソース**：時間◎／カネ×／体力○
- **代表的な遊び**
  - 春：公園遊び、つくし取り、花見散歩
  - 夏：プール、水遊び、花火、虫取り、砂場
  - 秋：どんぐり拾い、落ち葉遊び、運動会
  - 冬：雪遊び、雪だるま、室内でのお絵かき
  - 通年：滑り台、粘土遊び、ブロック、絵本、トランポリン、お絵かき、ごっこ遊び
- **解禁条件の例**
  - トランポリン：身長80cm以上
  - プール：夏季のみ、親の同伴必須
  - 雪遊び：冬季かつ雪降る地域設定時のみ

### 5.2 小学生（7〜12歳）
- **制約**：小遣い、学校、親の決定権（徐々に緩和）、門限
- **リソース**：時間◎／カネ△（小遣い制）／体力◎
- **代表的な遊び**：自転車探検、公園でのスポーツ、ゲーム、読書、習い事、カードゲーム、川遊び、キャンプ（家族）、Nintendo系ゲーム、プログラミング教室など

### 5.3 中学生（13〜15歳）
- **制約**：部活、勉学、門限、交友関係
- **リソース**：時間○／カネ△／体力◎
- **代表的な遊び**：部活、スマホ、SNS、友達と買い物、ゲーム、文化祭、バンド

### 5.4 高校生（16〜18歳）
- **制約**：受験、バイト可能、行動範囲拡大
- **リソース**：時間△／カネ△（バイト解禁）／体力◎
- **代表的な遊び**：バイト+遊興、カラオケ、旅行（国内）、自転車旅、ライブ参戦

### 5.5 大学生（19〜22歳）
- **制約**：学費、授業、バイト、就活
- **リソース**：時間◎／カネ△／体力◎
- **代表的な遊び**：サークル、飲み会、恋愛、海外旅行、免許取得、車/バイク、一人暮らし、ボードゲーム会、徹夜ゲーム、夏フェス
- **本作の「時間は最強だがカネはない」ピーク期**

### 5.6 社会人（23〜30歳）
- **制約**：仕事時間、通勤、疲労
- **リソース**：時間△／カネ○／体力○
- **代表的な遊び**：海外旅行、高級ディナー、習い事、ジム、ゴルフ、趣味への本格投資

### 5.7 結婚・育児期（30〜45歳）
- **制約**：家族時間、育児、住宅ローン
- **リソース**：時間×／カネ○／体力△
- **代表的な遊び**：家族旅行、子どもと公園、キャンプ、子の習い事観覧（擬似体験）、夫婦デート
- ここで **「自分の遊び」が大きく制限される** 体験を意図的に入れる。

### 5.8 中年（46〜60歳）
- **制約**：体力低下、子育て後期、親の介護可能性
- **リソース**：時間△／カネ◎／体力△
- **代表的な遊び**：ゴルフ、釣り、再学習、登山、海外長期旅行、ワインなどの趣味

### 5.9 定年退職（61〜75歳）
- **制約**：健康、年金生活
- **リソース**：時間◎／カネ○／体力△
- **代表的な遊び**：家庭菜園、旅行、孫との遊び、若い頃の趣味の再燃、カメラ

### 5.10 老後（76〜100歳）
- **制約**：体力大幅低下、移動制限
- **リソース**：時間◎／カネ△／体力×
- **代表的な遊び**：散歩、読書、囲碁将棋、昔を語る、オンライン交流
- **エンディング**：100歳になった時点で、**人生アルバム**（原体験ポイントの履歴をビジュアル化）を提示。

## 7. パラメーター設計

### 7.1 基本パラメーター
| 名称 | 範囲 | 用途 |
|---|---|---|
| 年齢 | 1〜100 | 時間進行の主軸 |
| 身長 | 30〜180cm | 物理的遊びの解禁条件 |
| 体力 | 0〜100 | 遊びの実行可否、回復量 |
| 健康度 | 0〜100 | 寿命・病気イベント |
| 決定権 | 0〜100 | 親/会社の制約をどこまで突破できるか |
| カネ | 0〜∞ | 遊び・生活に必要 |
| 時間余剰 | 0〜24h | 1日ごとに計算 |
| ジョウネツ | 0〜∞ | スコアの主軸 |

### 7.2 原体験パラメーター（5カテゴリ）
- 身体系 / 創作系 / 探求系 / 交流系 / 競技系
- 各Lv1〜99

### 7.3 隠しパラメーター
- 運（ランダムイベントの発生率）
- 偶発出会い度（恋愛・親友・ライバルイベントに影響）

### 7.4 生活リズム（Bio Rhythm）
- **範囲**：0〜100
- **意味**：睡眠と起床の規則正しさを表すパラメーター
- **上昇要因**
  - 早寝（21:00以前）：**+10**
  - 3日以上連続で規則正しく寝る：ボーナス +5（達成ごと）
- **下降要因**
  - 夜更かし（25:00以降）：**-15**
  - 連続夜更かし：2日目以降は -20、3日目以降は -25（累進）
- **影響するもの**
  - **起床時刻**：リズム 70以上＝6:30起床、40〜69＝7:30起床、40未満＝9:00起床（寝坊）
  - **余剰時間**：起床が遅れるとその日の余剰時間が -2h
  - **体調不良の発生確率**：
    - リズム < 60 + 夜更かし：翌朝 30% で体調不良（体力 -15、余剰時間 -2h）
    - リズム < 30：毎朝 15% で体調不良
- **設計意図**
  - 「夜更かししてでも遊びたい」誘惑を演出しつつ、**中長期で見ると損をする** ジレンマを作る
  - 特に育児期・社会人フェーズで **リズム維持のコスト** が重く効く
  - 大人ほど夜更かしによる余剰時間の価値が高いが、翌日の体力/余剰時間が失われ**ジョウネツの継続投資が難しくなる** 設計

### 7.5 友人数（Friends）
- **範囲**：0〜∞（上限は緩やかな体感上限あり / 年齢に応じた減衰あり）
- **意味**：一緒に遊ぶ相手の多さ。交流系の遊びを成立させる前提条件
- **増加要因**
  - ランダムイベント「新しい友達ができた」（+1）
  - 学校入学・クラス替え・進学・就職などライフイベント時に一括増加
  - 交流系の遊びを一定回数行う（Lv アップ報酬）
- **減少要因**
  - 引っ越し・進学・就職などで自然減（-30%〜-60%）
  - 長期間交流系の遊びを選ばないと徐々に減少（疎遠化）
- **影響するもの**
  - **遊びの解禁条件**：以下のような遊びは一定人数の友人がいないと実行できない
    - 鬼ごっこ：友人 1人以上
    - ごっこ遊び：友人 1人以上
    - サッカー・野球：友人 4人以上
    - ボードゲーム会：友人 3人以上
    - 飲み会・バーベキュー（大人）：友人 2人以上
    - 結婚式二次会参加：友人 5人以上
  - **経験値ブースト**（交流系中心に）
    - 例：鬼ごっこでは「友人 1人ごとに交流系 +3exp」のように友人数が経験値に直接影響
    - 複数人で遊ぶ遊びほど、友人数による経験値倍率が高い
  - **ジョウネツへの補正**
    - 交流系の遊びを友人数2人以上で実行すると、ジョウネツ +2 ボーナス
- **設計意図**
  - 「一人で遊ぶ楽しさ」と「みんなで遊ぶ楽しさ」の両軸を用意
  - 友人を作るための交流系投資が、結果的に **解禁される遊びの幅を広げる** 循環構造
  - 大人になるほど友人が減りがちになる現実をシミュレートし、**中年以降に意識的な交流投資が必要** という体験を作る

- **音を出さなくてもわかるUI**（通勤時プレイを想定）

- エンディングは **人生アルバム** を年表形式でスクロールさせ、プレイヤーの選択を振り返らせる

## 12. リスクと対策

| リスク | 内容 | 対策 |
|---|---|---|
| コンテンツ量不足 | 1人開発で遊びマスタ200種は重い | JSONでデータ分離、生成AIで初期案を量産→手動調整 |
| ビジュアル制作の壁 | 1人でイラスト制作は困難 | 絵本調＋生成AI＋ベクターイラストで省力化 |
| ゲームバランス | 資源ジレンマが厳しすぎると萎え、緩いと刺さらない | Phase 5のβで複数パラメータセットをA/Bテスト |
| プレイ時間の長さ | 100歳まで通しプレイは長い | ステージ飛ばし周回 / 中断再開を厚く実装 |
| コンセプトの伝達 | 「遊びと人生」が初見で伝わらない可能性 | オンボーディングで「未就学児→エンディング」を5分で体験させるモード |
| 開発疲弊 | 1人開発でのモチベ低下 | Phase単位で "遊べる状態" を作る、SNSで開発ログ発信 |
