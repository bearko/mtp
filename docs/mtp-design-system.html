§1  デザイン憲章 — Design Charter
⏱
30秒ループ原則
1ターンは30秒以内で完結する。 情報は「読む」ではなく「見る」で把握できること。 スキマ時間に自然に手が伸びる軽快さを最優先する。 タップ数は常に最小化し、1アクション=1インテント。
👁
情報は「見せる」
HUDは3列グリッドで情報を分離。 体力ゲージを太く主役に置き、数値に頼らず ひと目で状態が分かる。グレード表示（A〜G）は 色だけで良否が判断できるよう設計する。
✨
ゴーストプレビュー
選択前にコストと効果を可視化する。 体力ゲージには半透明の赤で消費量を重ね、 素養カードには「+N」デルタを並列表示。 「選んで驚く」ではなく「選ぶ前から分かる」設計。
🌙
ダーク基調 × 季節演出
深夜ダークパープルをベースに置き、 ゴールド（カネ）・グリーン（獲得）・レッド（コスト）で ゲームの文脈を色で語る。 季節ごとに背景グラデーションのアクセントが変化する。
📓
絵本 × ゲームのハイブリッド
HUD・ドックは機能的ゲームUI。 S3結果画面・S10連絡帳は絵本的温かみ。 ゲームっぽさと物語っぽさを画面の役割で使い分ける。 描写コメントは斜体で「子どもの一人称」に統一。
🔇
音なしでも伝わる
通勤・電車プレイを想定。 結果の種別（獲得/コスト）は色と記号で表現。 SEなしでも「うまくいった」「消費した」が 直感的に分かる視覚言語を徹底する。

§3  現状の課題 → 改善案
✕
旧 HUD — 情報圧縮
体力・リズム・時計・ライフステージが横一行に詰め込まれ、瞬時の状態把握が困難。特に体力ゲージが細く、緊急度を感じにくい。
✓
新 HUD — 3列グリッド化
左：ステージ/年齢/残時間、中央：日付/リズム/太い体力ゲージ、右：金/友人/場所。役割別に分離し、体力ゲージを中央エリアの主役に据える。
✕
旧 S2 — 選択前に情報なし
遊びをタップして初めてコスト・効果が分かる設計。不確実なまま選ばなければならず、やり直しができないゲーム性と相性が悪い。
✓
新 S2 — ゴーストプレビュー
dock でアイコンをタップするとプレビューモードに。体力ゲージに赤ゴーストが重なり、素養カードに「+N」デルタが現れる。もう一度タップで確定。
✕
旧 S2 — delta バッジの可読性
素養カードの「+N」が実数値の上に position:absolute で重なり、「10」の上に「+10」が被って読めない状態（SPEC-034 §6.3で指摘済）。
✓
新 — grid 5列で並列表示
icon | label | grade | delta | value の5列gridに変更。deltaセルを独立させ「+12」と「82」を同時に確認できる。空の場合は空セルで位置を固定。
✕
旧 S3 — 3ボタンの誤タップリスク
「もう一度遊ぶ / 次の遊びを選ぶ / 今日おわる」の3ボタン配置で、スキマ時間プレイ中に誤タップしやすく、巻き戻しができないゲームで致命的。
✓
新 S3 — 2ボタン + 自動就寝
「もう一度 / 次を選ぶ」の2ボタンに整理。余剰時間ゼロで自動的に「おやすみ」が出るため、能動的な「今日おわる」は不要と判断（SPEC-034 §6.1）。
✕
旧 S10 — 連絡帳の世界観が薄い
日サマリが汎用的なリスト表示で終わり、「保育園の連絡帳」という世界観演出がコードの文字列だけに留まっている。没入感に欠ける。
✓
新 S10 — 連絡帳フレーム + チップ
ゴールドボーダーの連絡帳ヘッダーで世界観を演出。各遊びの結果を色分けチップ（身体/知性/感性/社交）で可視化。今日の情熱スコアを大きく表示して達成感を演出。

§5  デザイン憲章（詳細版）— Full Design Charter
① 体験原則
11ターンは30秒以内。タップ数は最小化し、スキマ時間でも自然に続けられる軽快さを最優先する。
2情報は「読む」ではなく「見る」で把握できること。色・グレード・ゲージで状態を瞬時に伝える。
3選択前に必ずコストと効果を可視化する（ゴーストプレビュー原則）。「選んで後悔」の体験を排除する。
4音がなくても遊べること。SEに依存せず、色・動き・テキストのみで情報を完結させる。
② ビジュアル言語
5深夜ダークパープルをベースに置き、明度の高いアクセントで重要情報を際立たせる。白背景は使わない。
6獲得は緑（#4ade80）、コストは赤（#f87171）、カネ/時間は金（#f5c842）に統一。ゲーム文脈を色で語る。
7グレード（A〜G）は文字と色の組み合わせで即時判断できること。数値は補助情報として小さく添える。
8季節ごとに背景グラデーションのアクセント色を変化させ、ライフステージの空気感を演出する。
③ 画面別ルール
9HUDは純粋な機能UI。ゲームHUD的な情報密度で設計し、世界観演出は行わない。
10S3結果・S10連絡帳は絵本的温かみを持たせる。コメントは子どもの一人称・斜体で統一。
11ドックの遊びボタンは常に全候補を横スクロールで見渡せること。隠れた選択肢は作らない。
12モーダル・オーバーレイはz-indexで厳密に管理し、ゲームフローを遮断しない設計にする。
④ コンポーネント規約
13素養カードは5列grid（icon/label/grade/delta/value）。deltaと実数値を重ねない。空deltaはempty spanで固定。
14体力ゲージは中央エリアの主役として最低6px高さを確保。3段階カラー（緑/黄/赤）で危険度を表現。
15全てのDOM参照にnullガードを実装すること（SPEC-034 §6.4教訓）。UIリデザイン後は参照箇所の全検索必須。
16フォントはM PLUS Rounded 1cのみ使用。見出し900/ラベル700/本文400の3ウェイトで構成する。
