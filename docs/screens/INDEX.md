## ドキュメントの 2 階層構造

```
docs/
├─ DEVELOPMENT_RULES.md        開発ルール（仕様駆動開発の大原則）
├─ screens/                    画面設計書
│   ├─ INDEX.md                 本ファイル（画面索引）
│   └─ SCREEN-DESIGN.md         画面ワイヤーフレーム・遷移図
└─ specs/                      機能仕様書
    ├─ SPEC-INDEX.md            仕様書索引
    ├─ SPEC-001-*.md            機能別仕様（1機能=1ファイル）
    └─ …
```

### 画面設計書（Screens）が担当するもの
- 全画面のワイヤーフレーム（ASCIIアート）
- 画面間の遷移図
- 画面ID（S1, S2, …）の命名と振り分け
- 共通HUD・共通レイアウトの外観

### 機能仕様書（Specs）が担当するもの
- ゲームロジックの詳細（計算式・閾値・挙動）
- データモデル
- UIコンポーネントの挙動（タップで何が起きるか）
- 画面横断の機能（体力上限、コアタイムなど）

## 相互参照のルール

- 画面設計書 → 仕様書：各画面のセクションで `対応仕様：SPEC-XXX` を明記
- 仕様書 → 画面設計書：`## 6. UIへの反映` セクションで画面IDを参照
- コード側：`@spec` 注釈でいずれのドキュメントへも紐付ける（[`../DEVELOPMENT_RULES.md`](../DEVELOPMENT_RULES.md) §3.1）
