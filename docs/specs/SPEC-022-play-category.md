# SPEC-022: 遊びカテゴリ

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-022 |
| 機能名 | 遊びカテゴリ（Play Category） |
| 対応ファイル | `prototype/game.js` の `CATEGORIES`, `PLAYS[*].categories` |
| 関連仕様 | SPEC-002, SPEC-003, SPEC-005, SPEC-011, SPEC-023, SPEC-024 |
| ステータス | Active（保育園分のみ実装） |
| 最終更新 | 2026-04-18 |

> カテゴリマスタ `CATEGORIES` は **SPEC-028** により `data/categories.json` から動的に読み込まれる。

## 1. 目的

遊びに **「その遊びを通じて人が学べる・得られる要素」** をタグ付けし、  
遊びツリー（SPEC-023）やスキル（SPEC-024）の土台となる共通語彙を提供する。

例：
- 滑り台 → 「遊具」「公園」「運動」
- 絵本を読む → 「読書」「文字」「感受性」
- 砂場遊び → 「遊具」「公園」「工作」「感受性」

子どもが遊ぶものほど **人間性の基礎となるカテゴリ**（感受性・運動・社交など）が多く、  
大人になるほど **専門性の高いカテゴリ**（経営・技能・文化など）が現れる設計方針。

## 2. 用語定義

- **カテゴリ（Category）**：遊びに付与するタグ。1 遊び につき 1〜5 個程度が付与される。
- **カテゴリID**：プログラム内識別子（例 `outdoor_toy`, `reading`, `movement`）
- **カテゴリラベル**：UI に表示する日本語名（例 「遊具」「読書」「運動」）
- **カテゴリグループ**：上位分類。プロト段階では 6 グループに整理（身体／創作／探求／交流／生活／文化）

## 3. 入力（Input）

- 遊びマスタ `PLAYS[*].categories`：カテゴリID の配列

## 4. 出力（Output）

- 遊び選択画面の詳細プレビューで **タグとして表示**
- 遊びツリー画面（S7、SPEC-023）で **絞り込みキー** として機能
- スキル画面（SPEC-024）で **経験値配分** のキーとして機能

## 5. 挙動（Behavior）

### 5.1 カテゴリマスタ（プロト段階のカタログ）

6 グループ × 合計 18 カテゴリを初期定義する。プロト段階で使用するのは保育園で発生する 10 種程度。

| グループ | カテゴリID | 日本語ラベル | 代表的な遊び例 |
|---|---|---|---|
| 身体 | `movement` | 運動 | 鬼ごっこ、トランポリン、公園 |
| 身体 | `outdoor_toy` | 遊具 | 滑り台、砂場、ブランコ |
| 身体 | `water` | 水遊び | プール、水風船 |
| 身体 | `nature` | 自然 | 虫取り、雪遊び、散歩 |
| 創作 | `crafting` | 工作 | 粘土、折り紙、絵 |
| 創作 | `music` | 音楽 | 歌、楽器、リズム |
| 創作 | `imagination` | 想像 | ごっこ遊び、おままごと |
| 探求 | `reading` | 読書 | 絵本、図鑑、本 |
| 探求 | `literacy` | 文字 | ひらがな、数字、学習 |
| 探求 | `science` | 観察 | 虫の観察、天体、実験 |
| 交流 | `social_play` | 共遊 | みんなで遊ぶ、協調 |
| 交流 | `friendship` | 交友 | 友達づくり、会話 |
| 生活 | `daily_life` | 生活 | 買い物、料理、片付け |
| 生活 | `sensitivity` | 感受性 | 絵本、音楽、四季 |
| 文化 | `tradition` | 伝統 | 祭り、季節行事 |
| 文化 | `tech` | 技術 | プログラミング、機械いじり |
| 文化 | `competition` | 競技 | スポーツ大会、ゲーム対戦 |
| 文化 | `travel` | 旅 | 旅行、遠足、冒険 |

### 5.2 遊びマスタへのカテゴリ付与

各遊びは `categories: ["cat1", "cat2", ...]` を持つ。プロト段階の保育園向け遊びのタグ例：

| 遊び | カテゴリ |
|---|---|
| 砂場遊び | `outdoor_toy`, `crafting`, `nature` |
| 鬼ごっこ | `movement`, `social_play` |
| プール | `water`, `movement` |
| 粘土遊び | `crafting`, `imagination` |
| トランポリン | `movement`, `outdoor_toy` |
| 雪遊び | `nature`, `movement`, `tradition` |
| 絵本を読む | `reading`, `literacy`, `sensitivity` |
| 滑り台 | `outdoor_toy`, `movement` |
| 虫取り | `nature`, `science` |
| ごっこ遊び | `imagination`, `social_play`, `friendship` |
| 大きな滑り台（解禁） | `outdoor_toy`, `movement`, `competition` |

### 5.3 カテゴリの役割

| 使い先 | 内容 |
|---|---|
| SPEC-003 結果画面 | 遊び結果に「獲得カテゴリ」を表示（どのカテゴリが成長したか可視化） |
| SPEC-005 原体験Lv | 従来通り 5 カテゴリ（physical等）へ加算。**カテゴリとは別枠**で併存 |
| SPEC-024 スキル | 付与カテゴリごとに 1:1 でスキル経験値が加算 |
| SPEC-023 遊びツリー | カテゴリでフィルタして遊びを分類表示 |

### 5.4 将来拡張

- プレイヤーが特定カテゴリを「お気に入り」として固定する機能
- カテゴリ毎の累計ポイントからパーソナリティ診断（「あなたは工作タイプ」）
- 発見イベントでの新カテゴリ解禁

## 6. UIへの反映

- S2 遊び選択プレビュー：プレビューのコスト欄下に「🏷 タグ: 遊具 / 公園 / 運動」の行を追加
- S3 結果画面：獲得スキル行に **カテゴリ名のチップ** を表示
- S7 遊びツリー画面（SPEC-023）：タブで **カテゴリ絞り込み** を提供

## 7. 想定される利用シナリオ

- 2歳が砂場遊びを選ぶ → 3 カテゴリ（遊具・工作・自然）に経験値が入り、対応する 3 スキルがレベルアップ余地を得る
- 遊びツリー画面でカテゴリ「運動」だけを絞り込むと、鬼ごっこ／トランポリン／プール／滑り台／大きな滑り台 が一覧される

## 8. 例外・エッジケース

- カテゴリが 1 つも付かない遊びは禁止（最低 1 つ）
- 同じカテゴリを重複指定しても加算は 1 回扱い
- 将来カテゴリを増やす場合は索引（本文 §5.1）とコード `CATEGORIES` を同時に更新する
