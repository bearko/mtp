# SPEC-008: 就寝と翌日への遷移

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-008 |
| 機能名 | 就寝 / 翌日遷移 |
| 対応ファイル | `prototype/game.js` の `goSleep`, `sleep`, `nextDay` |
| 関連仕様 | SPEC-001, SPEC-006 |
| ステータス | Active |
| 最終更新 | 2026-04-18 |

## 1. 目的
1日の締めくくりを演出し、就寝モードの選択によって **生活リズム**（SPEC-006）を変動させる。  
1日の成果サマリを表示し、プレイヤーに達成感を与える。

## 2. 用語定義
- **就寝モード**：early / normal / latenight。
- **日付繰り上げ**：`player.day += 1`。季節・年齢の境界判定を行う。

## 3. 入力（Input）
- 1日の成果（遊び回数・ジョウネツ増分・体力・リズム・友人数）。
- 就寝モードの選択。

## 4. 出力（Output）
- 翌日のプレイヤー状態（起床時刻・余剰時間・体調不良フラグ）。
- 起床画面への遷移。

## 5. 挙動（Behavior）

### 5.1 就寝画面への遷移条件
- 余剰時間が 0 になった時：SPEC-003 から自動遷移。
- 遊び選択画面で「今日をおわる」を押下した時。

### 5.2 就寝モード選択
| モード | リズム | 体力 | 備考 |
|---|---|---|---|
| early | +10 | +25 | 規則的 |
| normal | ±0 | +15 | 標準 |
| latenight | -15 | ±0 | 夜更かし枠。翌日の余剰時間 +1h |

### 5.3 翌日の状態決定
- SPEC-006 に従って起床時刻・余剰時間・体調不良フラグを決定。
- 日付 +1、30日ごとに季節更新、120日ごとに年齢 +1。

### 5.4 日次カウンタのリセット
- `dailyPlays = 0`
- `consecutiveCategoryCount` はリセットしない（ジョウネツの没頭ボーナスを日を跨いで維持）。

## 6. UIへの反映
- 画面：S5 就寝（`#screen-sleep`）
- 画面設計書：`docs/screens/SCREEN-DESIGN.md` §2.5

## 7. 想定される利用シナリオ
- 遊び3回、ジョウネツ+12獲得した日の夜：サマリ表示 → 早寝を選択 → 翌朝リズム90, 体力フル。

## 8. 例外・エッジケース
- 余剰時間が残っていても「今日をおわる」で強制就寝できる。
