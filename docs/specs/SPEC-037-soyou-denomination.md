# SPEC-037: 素養ポイントのデノミネーション（ライフステージ遷移時）

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-037 |
| 機能名 | ライフステージ変更時の素養ポイント 1/4 デノミ |
| 対応ファイル | `prototype/game.js` `applySoyouDenomination()`, `nextDay()`（誕生日処理） |
| 関連仕様 | SPEC-001（ライフステージ）, SPEC-033（素養モデル）, SPEC-038（英霊システム） |
| ステータス | **Draft（未実装）**。幼稚園移行時から実装予定 |
| 最終更新 | 2026-04-19 |

## 1. 目的

素養値はライフステージを重ねるごとに累積していくため、**後のステージの 1 プレイで得られる +N** が過去の累計から見て「誤差レベル」になってしまう。例えば保育園を 2 年遊び切ると素養は 500+ になり、幼稚園の 1 プレイ（+10）が 2% の変化にしかならずテンポが悪い。

そこで、**ライフステージが変わるタイミングで素養値を 1/4 にデノミ**する。プレイヤーが体感する「成長実感」は保たれ、新ステージでも +10 が無視できないインパクトを持つ。

Issue #14 の幼稚園パートで提起された仕様。

## 2. 用語定義
- **デノミネーション（Denomination）**：本仕様では「素養の実数値をステージ境界で 1/4 に切り詰める」操作を指す
- **消費優先順位**：英霊召喚（SPEC-038）は **デノミ前の素養** から消費する（§4.3）

## 3. 前提
- SPEC-033 で `player.soyou = {body, intellect, sensitivity, social, passion}` が実数値で管理されている
- SPEC-001 `nextDay()` 内で年齢繰り上げを行っている。ステージ境界は `resolveLifeStage(player.age)` の id が変わる瞬間
- 英霊システム（SPEC-038）はデノミ前の素養を消費してから呼び出しを処理する

## 4. 挙動

### 4.1 タイミング
- `nextDay()` で `hasBirthday === true` かつ **新しいステージ ID が直前のステージ ID と異なる** 瞬間
- 具体的には：
  - age 4→5（保育園 → 幼稚園）
  - age 6→7（幼稚園 → 小学校）
  - age 12→13（小学校 → 中学校）
  - age 15→16（中学校 → 高校）
  - age 18→19（高校 → 大学）
  - age 22→23（大学 → 社会人）
  - age 65→66（社会人 → 老後）

### 4.2 演算
```js
function applySoyouDenomination() {
  for (const k of SOYOU_KEYS) {
    player.soyou[k] = Math.floor((player.soyou[k] || 0) / 4);
  }
}
```
- 小数点以下は切り捨て
- グレード（SPEC-033 §4）は自動で再計算される（実数値が下がるため）

### 4.3 英霊召喚とのインタラクション（SPEC-038）
- 新しいステージに入る直前に、プレイヤーは **英霊召喚画面** でその時点の素養を消費できる
- 消費後に **デノミを実行** する
- つまりフローは：
  1. ステージ境界到達
  2. 英霊召喚画面（任意）
  3. デノミ実行（1/4）
  4. 新ステージ 1 日目スタート

### 4.4 UI 表示
- デノミ実行時にトースト：「新しいステージに入った！素養値が整理されました」
- 素養カードは再描画されるので、グレードが視覚的に下がる場合もある（例：S から A に戻るなど）
- 「デノミ前の値」を懐かしむのはあえて避け、新しい値から再出発する

## 5. テスト観点

- age 4→5 のタイミングで soyou.intellect が 100 → 25 に切り詰められる
- 小数点以下切り捨て（85 → 21、84 → 21）
- グレード B（115）が 28 に切り詰められ F（20-39）になる
- 複数ステージまたぐ遷移が短時間で来ても 1 回ずつ適用される（たとえば大学卒業後すぐ社会人）
- 英霊召喚（SPEC-038）が完了してからデノミが実行されること
