## 1. 目的

プレイヤーの `player.money`（所持金）を、ライフステージごとに **意味のあるリソース** として扱うための総則。保育園・幼稚園期は親負担のため関係なし、小学校以降で具体的に意味を持ち始める。

## 2. 用語定義
- **お小遣い（Allowance）**：親から定期的に与えられる金銭（小〜高校）
- **給与（Salary）**：就労で得る金銭（バイト・社会人）
- **支出（Expense）**：遊び・食事・学費・家賃で出ていく金銭
- **貯蓄（Savings）**：残った金額。明示的に `player.money` で管理
- **価格体系**：遊び・アイテム・サービスの定価リスト

## 5. 貯蓄と投資

### 5.1 貯金口座
- 当面は `player.money` が現金＋預金の総合

## 7. 実装メモ

### 7.1 grantAllowance 関数
```js
function grantAllowance() {
  const age = player.age;
  const dow = fakeDateForDay(player.day).getUTCDay();
  if (age >= 7 && age <= 9 && dow === 1) {
    player.money += 500;
  } else if (age >= 10 && age <= 12 && dow === 1) {
    player.money += 1000;
  } else if (age >= 13 && age <= 15 && player.day % 30 === 1) {
    player.money += 8000;
  } else if (age >= 16 && age <= 18 && player.day % 30 === 1) {
    player.money += 10000;
  } else if (age >= 19 && age <= 22 && player.day % 30 === 1) {
    player.money += 15000;
  }
  // 社会人は SPEC-045 で別途
}
```

### 7.2 payExpense 関数
- 遊び選択時、`moneyCost > player.money` なら選択不可（disabled）
- トースト：「お金が足りない…」

### 7.3 money 非表示（幼少期）
- SPEC-034 §4.4 ：age ≤ 6 は HUD で所持金非表示

## 8. テスト観点
- 小学校 1 年生の月曜日に 500 円が加算される
- 残高 100 円で 500 円の遊びは選択 disabled
- バイトを 1 時間すると 1000 円（高校）／1200 円（大学）が加算
- 6 月 1 日に月給 × 2 のボーナスが加算される（社会人）
