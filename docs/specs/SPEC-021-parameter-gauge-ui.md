# SPEC-021: パラメーターゲージUI

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-021 |
| 機能名 | パラメーターゲージ表示（体力・時間・経験値） |
| 対応ファイル | `prototype/game.js`（`renderGauge`, `renderHUD`, `renderResultPanel`, `renderCoreTimeGain`） / `prototype/styles.css`（`.gauge`, `.gauge-bar`） |
| 関連仕様 | SPEC-001, SPEC-003, SPEC-005, SPEC-009, SPEC-011, SPEC-019 |
| ステータス | Active |
| 最終更新 | 2026-04-18 |

## 1. 目的

プレイヤーに **「あとどれだけ余裕があるか」** を一目で伝える。  
これまで数値表示だったパラメーターをゲージ化し、限界と伸びしろを視覚化する。

- 体力 → 現在値 / 上限（上限も可変なため、伸び縮みを視覚化）
- 余剰時間 → その日の残り / 基準時間（例：4h）
- 経験値 → 現在Lv / 次のLvまでの進捗率

## 2. 用語定義

- **ゲージ（Gauge）**：水平の帯状のプログレスバー。左が 0、右が 100%。
- **ゲージラベル**：ゲージ下に表示される「現在値 / 上限」の数値テキスト。
- **ゲージカラー**：値の健康度によって色を変える（緑=余裕、橙=注意、赤=危険）。
- **デルタ表示**：ゲージ内に差分（+/-）を短時間アニメーションで重ねる（オプション、初期はシンプル）。

## 3. 入力（Input）

| ゲージ種別 | 現在値 | 上限 |
|---|---|---|
| 体力 | `player.stamina` | `player.staminaCap`（SPEC-019） |
| 生活リズム | `player.bioRhythm` | 100（固定） |
| 余剰時間 | `player.spareHours` | `player.spareHoursMax`（1日ベース、プロトは 4） |
| 原体験Lv（各5カテゴリ） | `exp − (lv² × 10)` | `((lv+1)² × 10) − (lv² × 10)` |

## 4. 出力（Output）

- ゲージ DOM（`.gauge > .gauge-bar`）
- ラベルテキスト（現在値 / 上限）
- 色（緑・橙・赤）

## 5. 挙動（Behavior）

### 5.1 ゲージのDOM構造
```
<div class="gauge" data-kind="stamina">
  <div class="gauge-track">
    <div class="gauge-bar" style="width: 52%"></div>
  </div>
  <div class="gauge-label">
    <span>体力</span>
    <span>15 / 30</span>
  </div>
</div>
```

### 5.2 表示配置
| 画面 | ゲージ |
|---|---|
| HUD（全画面共通） | 簡略化した細ゲージ 3 本（体力／リズム／余剰時間） |
| S1 起床 | フルサイズ ゲージ 4 本（体力／リズム／余剰時間／ジョウネツ） |
| S3 結果フェーズ | 変動したパラメーターのみゲージで表示。経験値5カテゴリはLvゲージ |
| S6 コアタイム | コアタイム終了後の獲得値表示で体力と経験値のLvゲージ |
| S5 就寝 | 体力と原体験Lvのゲージ（今日のまとめ） |

HUD の細ゲージは `height: 4px` で配置、数字表示と並走させる。

### 5.3 色ルール（体力／余剰時間）
| パーセント | 色 |
|---|---|
| ≥ 60% | 緑（`--good`） |
| 30〜60% | 橙（`--accent-2`） |
| < 30% | 赤（`--bad`） |

### 5.4 経験値Lvゲージ
- 現在Lvから次のLvまでの進捗を 0〜100% で表示。
- Lvアップ直後は 100% で一度満タン表示 → 次フレームで 0% へリセット（アニメ演出）。
- 色：常にオレンジ（`--accent`）固定。

### 5.5 アニメーション
- `.gauge-bar` は `transition: width 400ms ease-out` を付与し、値の変化をアニメーションで見せる。
- Lvアップ時は 1000ms で 100% → 0% → 新値の順に2段階アニメーション。

### 5.6 ゲージレンダリング関数
```
function renderGauge(container, { current, max, label, kind }):
  - container を .gauge 構造に置き換える
  - pct = clamp(current / max, 0, 1) * 100
  - color = thresholdColor(kind, pct)
  - bar.width = pct%
  - label に "currentの表示 / maxの表示" を描画
```
- `kind`：`stamina`, `time`, `rhythm`, `exp` など。

### 5.7 HUD への組み込み
HUD 2段目の体力・リズム・時間・ジョウネツを、数値の隣に細ゲージを並べる。  
幅が狭いため、**数値＋細ゲージ（width 48px）** のセットを1行に収める。

### 5.8 パフォーマンス
- 再描画はゲージ単位に局所化。画面全体を再構築しない。
- `requestAnimationFrame` 不要、CSS transition 任せ。

## 6. UIへの反映

- 画面設計書：`docs/screen-design.md` §1 共通HUD にゲージ追加
- 画面：S1, S3, S5, S6 の各 Scene 内にゲージを展開

## 7. 想定される利用シナリオ

- 2歳、体力上限 15 の状態で鬼ごっこ（-20）を選ぶ  
  → 実行後、体力ゲージが赤色でほぼ 0 を示す
- 原体験 身体Lv3 → Lv4 の Lvアップ時、ゲージが満タン演出後、新Lvの進捗 0% から再スタート
- 余剰時間ゲージが 1h を切るとオレンジ、0 になると赤でゼロ幅

## 8. 例外・エッジケース

- 上限が 0（例：上限 = 0 の老後末期の仮定）→ パーセント = 0 で固定。
- 現在値が上限を超える（異常系）→ 100% でクランプ、警告ログ。
- ダークモード：プロト未対応。将来カラートークン化する。
