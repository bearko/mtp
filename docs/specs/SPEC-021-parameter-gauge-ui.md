## 1. 目的

プレイヤーに **「あとどれだけ余裕があるか」** を一目で伝える。
パラメーターごとに最適な表現に分けて視覚化する：

- **体力** → 横長ゲージ（現在値 / 上限）。上限自体が年齢で可変なので「100%=その年齢の体力上限」が直感的に分かる。
- **余剰時間** → **24時間を1周とする時計円盤**。現在時刻に応じて扇形が時計回りに塗りつぶされる。  
  「ゲージは直感的でない」という評価を受けての改訂（v2）。
- **経験値** → 横長ゲージ（現在Lv内の進捗率）。

## 2. 用語定義

- **ゲージ（Gauge）**：水平の帯状のプログレスバー。左が 0、右が 100%。
- **ゲージラベル**：ゲージ下に表示される「現在値 / 上限」の数値テキスト。
- **ゲージカラー**：値の健康度によって色を変える（緑=余裕、橙=注意、赤=危険）。
- **時計円盤（Clock Dial）**：24時間を1周する円盤。現在時刻までが扇形で塗りつぶされる。
- **余剰ハイライト**：時計円盤上で「遊びに使える時間帯（朝・夜）」を別色で示すオプション表現。

## 3. 入力（Input）

| 表示種別 | 現在値 | 上限／基準 | 想定UI |
|---|---|---|---|
| 体力 | `player.stamina` | `player.staminaCap`（SPEC-019） | 横長ゲージ |
| 生活リズム | `player.bioRhythm` | 100（固定） | 横長ゲージ |
| **現在時刻** | `player.clockHour + player.clockMinute/60` | 24（固定） | **時計円盤** |
| 原体験Lv（各5カテゴリ） | `exp − (lv² × 10)` | `((lv+1)² × 10) − (lv² × 10)` | 横長ゲージ |

> 余剰時間の「残り ◯h」表示は時計円盤のラベルに併記する（例：下部に `12:00 / 残り 2h`）。

## 4. 出力（Output）

- 横長ゲージ DOM（`.gauge > .gauge-track > .gauge-bar`）
- 時計円盤 DOM（`.clock-dial`, SVG で実装）
- ラベルテキスト（現在値 / 上限 や 現在時刻 / 残り）

## 5. 挙動（Behavior）

### 5.1 体力ゲージ（横長）

#### 5.1.1 DOM構造
```html
<div class="gauge" data-kind="stamina">
  <div class="gauge-track">
    <div class="gauge-bar" style="width: 100%"></div>
  </div>
  <div class="gauge-label">
    <span>❤️ 体力</span>
    <b>15 / 15</b>
  </div>
</div>
```

#### 5.1.2 **初期表示の正しさ（重要）**
- **ゲーム起動直後は 100% 表示**（2歳なら `15/15` が満タンで描画される）。
- これまでの実装では HTML に `width: 0%` のみ記載し、JS による `renderHUD()` 呼び出しで CSS transition が 0→100% に補間される間、**一瞬 50% 程度に見えるバグ** があった（v1 で発生）。
- 対策：
  1. HTML の `<span id="hud-stamina">` / `<span id="hud-stamina-cap">` 初期値を `DEFAULT_PLAYER` と一致させる（例：`15` / `15`）。
  2. HUD のミニバーには **初期 width を CSS ではなく JS で同期的に設定**（`renderHUD()` を DOMContentLoaded 前に呼ぶ、もしくは HTML に `style="width: 100%"` を直書き）。
  3. `.gauge-bar` の `transition` は **初回描画をスキップ** するために、初期描画用のクラスを付けて `transition: none` にする運用を推奨。実装では JS 側で `applyGaugePercent` を初期化時に一度呼んでから HUD を表示することで視覚的なブレをなくす。

#### 5.1.3 色ルール（体力／リズム）
| パーセント | 色 |
|---|---|
| ≥ 60% | 緑（`--good`） |
| 30〜60% | 橙（`--accent-2`） |
| < 30% | 赤（`--bad`） |

### 5.2 時計円盤（Clock Dial）※ v2 で追加

#### 5.2.1 目的
時刻・余剰時間を **時計のアナロジー** で直感化する。  
「いま何時か」と「今日あとどれくらい遊べるか」が画面上の扇形で分かる。

#### 5.2.2 視覚仕様
- 円盤は **24時間で1周**（時計と同じ方向：12時=上、6時=右、18時=下、0時=左…ではなく、**24時間のアナログ時計的に、0時=上、6時=右、12時=下、18時=左、24時（=翌0時）=上に戻る**）。
- 中心から放射状に塗りつぶされる扇形は、**0:00 から現在時刻までの範囲**。
  - 例：
    - 0:00 → 円盤は真っ白（塗りつぶし 0%）
    - 8:00 → 1/3（=8/24）が塗りつぶされている
    - 12:00 → 半分
    - 18:00 → 3/4
    - 24:00 → 全部

#### 5.2.3 DOM（SVG）構造
```html
<div class="clock-dial" data-kind="time">
  <svg viewBox="0 0 100 100" class="dial-svg">
    <!-- 外枠 -->
    <circle cx="50" cy="50" r="46" class="dial-ring"/>
    <!-- 塗りつぶし扇形（JSで path を更新）-->
    <path d="" class="dial-fill"/>
    <!-- 時刻目盛り（任意、4本=6時間毎） -->
    <g class="dial-ticks">
      <line x1="50" y1="4"  x2="50" y2="10" />  <!-- 0時 -->
      <line x1="90" y1="50" x2="96" y2="50" />  <!-- 6時 -->
      <line x1="50" y1="90" x2="50" y2="96" />  <!-- 12時 -->
      <line x1="4"  y1="50" x2="10" y2="50" />  <!-- 18時 -->
    </g>
    <!-- 時刻テキスト -->
    <text x="50" y="48" class="dial-time"  text-anchor="middle">07:00</text>
    <text x="50" y="62" class="dial-sub"   text-anchor="middle">残り 2h</text>
  </svg>
</div>
```

#### 5.2.4 扇形パスの計算
中心 `(cx, cy) = (50, 50)`、半径 `r = 40`、開始角度は真上 `-90°`、時刻 `t` （0〜24）で：
```
angle(t) = -90 + (t / 24) * 360  // degree
x(t) = cx + r * cos(angle(t) * PI / 180)
y(t) = cy + r * sin(angle(t) * PI / 180)
```
扇形パス：
```
M cx,cy                     // 中心
L x(0), y(0)                // 0時の外周点
A r,r 0 largeArc,1 x(t), y(t)  // 円弧（時計回り）
Z                           // 中心へ戻る
```
`largeArc = t > 12 ? 1 : 0`。

#### 5.2.5 色ルール（時計円盤）
- 塗りつぶし色：
  - 0〜9時（朝・日中の前半）：淡い青系（`--clock-morning`）
  - 9〜18時（コアタイム・日中）：グレー系（`--clock-core`）※ コアタイムで自由時間が少ない時間帯であることを示唆
  - 18〜24時（夜）：オレンジ系（`--clock-night`）
- ただしプロト段階では **1色（`--accent` 相当）固定** で構わない（将来拡張）。
- 円盤の外枠：常に淡いグレー。

#### 5.2.6 アニメーション
- `path` 要素の `d` 属性は JS で直接更新する。CSS transition 非対応のため、**値変更時にフレーム間のスムーズアニメは任意**。プロトでは即時更新で十分。
- 必要に応じて fade 演出（`opacity`）を入れる。

#### 5.2.7 設置場所
| 画面 | 配置 |
|---|---|
| HUD | 小サイズ時計円盤（直径 28px）を HUD 右端に1つだけ配置。テキスト表示は時刻のみ |
| S1 起床 | 中サイズ時計円盤（直径 120px）を画面中央付近に表示 |
| S3 結果フェーズ | 「時間の流れ」カード内に小サイズ（直径 64px）を2つ（before / after）並べる |
| S5 就寝 | 就寝時刻（19:00 や 23:00）の円盤を表示して「もう夜」であることを視覚化 |

### 5.3 経験値Lvゲージ（横長）
- 現在Lvから次のLvまでの進捗を 0〜100% で表示。
- Lvアップ直後は 100% で一度満タン表示 → 次フレームで 0% へリセット（アニメ演出）。
- 色：常にオレンジ（`--accent`）固定。

### 5.4 アニメーション
- `.gauge-bar` は `transition: width 400ms ease-out` を付与し、値の変化をアニメーションで見せる。
- Lvアップ時は 1000ms で 100% → 0% → 新値の順に2段階アニメーション。
- 時計円盤は JS 同期更新（transition なし、瞬時反映）。

### 5.5 ゲージ／円盤レンダリング関数
```
function renderGauge(container, { current, max, label, kind, delta }):
  - container を .gauge 構造に置き換える
  - pct = clamp(current / max, 0, 1) * 100
  - color = thresholdColor(kind, pct)
  - bar.width = pct%
  - label に "現在値 / 上限" を描画

function renderClockDial(container, { clockHour, clockMinute, spareHours }):
  - container を .clock-dial 構造に置き換える（SVG）
  - t = clockHour + clockMinute/60
  - path.d = fanPath(t) // §5.2.4 の式
  - text.dial-time = "HH:MM"
  - text.dial-sub  = "残り Xh"
```
- `kind`：`stamina`, `rhythm`, `exp` など。

### 5.6 パフォーマンス
- 再描画はゲージ/円盤単位に局所化。画面全体を再構築しない。
- SVG の `path` 更新は DOM 操作 1回のみ。`requestAnimationFrame` 不要。

## 7. 想定される利用シナリオ

- **起動直後**：2歳プレイヤー → HUD の体力ゲージは `15 / 15` で 100% 緑満タン。時計円盤は 07:00 で `1/4`（=7/24）弱塗りつぶし。
- **朝の遊び終了**：絵本×2 で時計円盤が 09:00 まで進む（3/8 塗りつぶし）。体力ゲージは 15→11 で緑のまま。
- **保育園帰宅直後**：円盤は 16:00（2/3 塗りつぶし）、体力 6/15 で赤。
- **就寝（19:00）**：円盤は 19:00（= 19/24 ≒ 79% 塗りつぶし）、ほぼ真っ白なのは右上の 5/24 だけ。

## 8. 例外・エッジケース

- 上限が 0（例：上限 = 0 の老後末期の仮定）→ パーセント = 0 で固定。
- 現在値が上限を超える（異常系）→ 100% でクランプ、警告ログ。
- 時刻が 24:00 を超える（翌日への跨ぎ）→ 実質的に 24:00 でクランプ。`nextDay()` が呼ばれて新しい日で円盤がリセットされる。
- ダークモード：プロト未対応。将来カラートークン化する。
