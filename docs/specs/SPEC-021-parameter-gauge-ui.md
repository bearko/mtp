# SPEC-021: パラメーター表示UI（体力ゲージ・時計円盤・経験値ゲージ）

| 項目 | 内容 |

| 対応ファイル | `prototype/game.js`（`renderGauge`, `renderClockDial`, `renderHUD`, `renderResultPanel`, `renderWakeupGauges`） / `prototype/styles.css`（`.gauge`, `.gauge-bar`, `.clock-dial`） |
| 関連仕様 | SPEC-001, SPEC-003, SPEC-005, SPEC-009, SPEC-011, SPEC-019, SPEC-020 |

## 1. 目的

プレイヤーに **「あとどれだけ余裕があるか」** を一目で伝える。
パラメーターごとに最適な表現に分けて視覚化する：

- **体力** → 横長ゲージ（現在値 / 上限）。上限自体が年齢で可変なので「100%=その年齢の体力上限」が直感的に分かる。
- **余剰時間** → **24時間を1周とする時計円盤**。現在時刻に応じて扇形が時計回りに塗りつぶされる。  
  「ゲージは直感的でない」という評価を受けての改訂（v2）。
- **経験値** → 横長ゲージ（現在Lv内の進捗率）。

## 2. 用語定義

- **ゲージ（Gauge）**：水平の帯状のプログレスバー。左が 0、右が 100%。
- **ゲージラベル**：ゲージ下に表示される「現在値 / 上限」の数値テキスト。
- **ゲージカラー**：値の健康度によって色を変える（緑=余裕、橙=注意、赤=危険）。
- **時計円盤（Clock Dial）**：24時間を1周する円盤。現在時刻までが扇形で塗りつぶされる。
- **余剰ハイライト**：時計円盤上で「遊びに使える時間帯（朝・夜）」を別色で示すオプション表現。

## 3. 入力（Input）

| 表示種別 | 現在値 | 上限／基準 | 想定UI |
|---|---|---|---|
| 体力 | `player.stamina` | `player.staminaCap`（SPEC-019） | 横長ゲージ |
| 生活リズム | `player.bioRhythm` | 100（固定） | 横長ゲージ |
| **現在時刻** | `player.clockHour + player.clockMinute/60` | 24（固定） | **時計円盤** |
| 原体験Lv（各5カテゴリ） | `exp − (lv² × 10)` | `((lv+1)² × 10) − (lv² × 10)` | 横長ゲージ |

> 余剰時間の「残り ◯h」表示は時計円盤のラベルに併記する（例：下部に `12:00 / 残り 2h`）。

## 4. 出力（Output）

- 横長ゲージ DOM（`.gauge > .gauge-track > .gauge-bar`）
- 時計円盤 DOM（`.clock-dial`, SVG で実装）
- ラベルテキスト（現在値 / 上限 や 現在時刻 / 残り）

## 5. 挙動（Behavior）

### 5.1 体力ゲージ（横長）

#### 5.1.1 DOM構造
```html
<div class="gauge" data-kind="stamina">
  <div class="gauge-track">
    <div class="gauge-bar" style="width: 100%"></div>
  </div>
  <div class="gauge-label">
    <span>❤️ 体力</span>
    <b>15 / 15</b>
  </div>
</div>
```

#### 5.1.2 **初期表示の正しさ（重要）**
- **ゲーム起動直後は 100% 表示**（2歳なら `15/15` が満タンで描画される）。
- これまでの実装では HTML に `width: 0%` のみ記載し、JS による `renderHUD()` 呼び出しで CSS transition が 0→100% に補間される間、**一瞬 50% 程度に見えるバグ** があった（v1 で発生）。
- 対策：
  1. HTML の `<span id="hud-stamina">` / `<span id="hud-stamina-cap">` 初期値を `DEFAULT_PLAYER` と一致させる（例：`15` / `15`）。
  2. HUD のミニバーには **初期 width を CSS ではなく JS で同期的に設定**（`renderHUD()` を DOMContentLoaded 前に呼ぶ、もしくは HTML に `style="width: 100%"` を直書き）。
  3. `.gauge-bar` の `transition` は **初回描画をスキップ** するために、初期描画用のクラスを付けて `transition: none` にする運用を推奨。実装では JS 側で `applyGaugePercent` を初期化時に一度呼んでから HUD を表示することで視覚的なブレをなくす。

#### 5.1.3 色ルール（体力／リズム）
| パーセント | 色 |
|---|---|
| ≥ 60% | 緑（`--good`） |
| 30〜60% | 橙（`--accent-2`） |
| < 30% | 赤（`--bad`） |

### 5.2 時計円盤（Clock Dial）※ v2 で追加

#### 5.2.1 目的
時刻・余剰時間を **時計のアナロジー** で直感化する。  
「いま何時か」と「今日あとどれくらい遊べるか」が画面上の扇形で分かる。

#### 5.2.2 視覚仕様
- 円盤は **24時間で1周**（時計と同じ方向：12時=上、6時=右、18時=下、0時=左…ではなく、**24時間のアナログ時計的に、0時=上、6時=右、12時=下、18時=左、24時（=翌0時）=上に戻る**）。
- 中心から放射状に塗りつぶされる扇形は、**0:00 から現在時刻までの範囲**。
  - 例：
    - 0:00 → 円盤は真っ白（塗りつぶし 0%）
    - 8:00 → 1/3（=8/24）が塗りつぶされている
    - 12:00 → 半分
    - 18:00 → 3/4
    - 24:00 → 全部

#### 5.2.3 DOM（SVG）構造
```html
<div class="clock-dial" data-kind="time">
  <svg viewBox="0 0 100 100" class="dial-svg">
    <!-- 外枠 -->
    <circle cx="50" cy="50" r="46" class="dial-ring"/>
    <!-- 塗りつぶし扇形（JSで path を更新）-->
    <path d="" class="dial-fill"/>
    <!-- 時刻目盛り（任意、4本=6時間毎） -->
    <g class="dial-ticks">
      <line x1="50" y1="4"  x2="50" y2="10" />  <!-- 0時 -->
      <line x1="90" y1="50" x2="96" y2="50" />  <!-- 6時 -->
      <line x1="50" y1="90" x2="50" y2="96" />  <!-- 12時 -->
      <line x1="4"  y1="50" x2="10" y2="50" />  <!-- 18時 -->
    </g>
    <!-- 時刻テキスト -->
    <text x="50" y="48" class="dial-time"  text-anchor="middle">07:00</text>
    <text x="50" y="62" class="dial-sub"   text-anchor="middle">残り 2h</text>
  </svg>
</div>
```

#### 5.2.4 扇形パスの計算
中心 `(cx, cy) = (50, 50)`、半径 `r = 40`、開始角度は真上 `-90°`、時刻 `t` （0〜24）で：
```
angle(t) = -90 + (t / 24) * 360  // degree
x(t) = cx + r * cos(angle(t) * PI / 180)
y(t) = cy + r * sin(angle(t) * PI / 180)
```
扇形パス：
```
M cx,cy                     // 中心
L x(0), y(0)                // 0時の外周点
A r,r 0 largeArc,1 x(t), y(t)  // 円弧（時計回り）
Z                           // 中心へ戻る
```
`largeArc = t > 12 ? 1 : 0`。

#### 5.2.5 色ルール（時計円盤 / 3 レイヤー構造）※ v3

「残り時間のうち **遊びに使えるのはどれだけか** を直感的に知らせる」ため、  
時計円盤は **未経過部分を固定で 3 色に塗り分ける**（経過部分は上から別扇形で覆う）。

| レイヤー | 時間帯 | 色トークン | 意味 |
|---|---|---|---|
| 就寝帯（Sleep Band） | `0:00 → 起床時刻` と `就寝時刻 → 24:00`（年齢別 SPEC-020） | `--clock-sleep`（暗い紫グレー `#3a3240`） | 眠っていて遊べない時間 |
| コアタイム帯（Core Band） | `coreTime.startHour → coreTime.endHour`（例：保育園 9〜16） | `--clock-core`（グレー `#c3bfb0`） | 学び／仕事で遊べない時間 |
| 遊べる帯（Free Band） | 上記以外すべて（= 朝の遊び・夜の遊び時間） | `--clock-free`（淡いオレンジ `#f2cfa0`） | プレイヤーが自由に遊べる時間 |
| 経過塗り（Elapsed Fill） | `0:00 → 現在時刻` | `--accent`（濃いオレンジ） | もう過ぎた時間。上記の 3 帯の上から重ねて塗る |

- 描画順序：Sleep → Core → Free（3 色レイヤーを重ね描き）→ Elapsed Fill（現在時刻までを上から塗りつぶし）。
- プレイヤーには **「これから先の灰色帯の長さ＝拘束時間」「これから先のオレンジ帯＝遊べる時間」** として見える。
- 老後（`coreTime === null`）：コアタイム帯は描画しない。Sleep と Free のみ。
- 10歳以上（SPEC-020 の固定スケジュールなし）：Sleep 帯は `0:00-6:00` と `23:00-24:00` 等、暫定値で描画してよい（将来改善）。

#### 5.2.5b DOM（SVG）構造（v3）
```html
<svg viewBox="0 0 100 100" class="dial-svg">
  <circle cx="50" cy="50" r="46" class="dial-ring"/>
  <!-- 帯レイヤー（順番に重ねる）-->
  <path class="dial-band dial-band-free"  d="..."/>   <!-- Free 全体の背景 -->
  <path class="dial-band dial-band-sleep" d="..."/>   <!-- Sleep 帯（弧状）-->
  <path class="dial-band dial-band-core"  d="..."/>   <!-- Core 帯（弧状）-->
  <!-- 経過塗り：0:00 → 現在時刻 -->
  <path class="dial-fill" d="..."/>
  <!-- 目盛り、時刻テキスト -->
</svg>
```
実装詳細：
- `dial-band-free` は「円盤全体の色」として内側全体を塗る（実質 24:00 = 全周の扇形）。
- `dial-band-sleep` と `dial-band-core` はその上にパイ型の扇形を重ねる。複数区間ある場合は複数 `<path>` を並べる（例：Sleep が `0-7` と `19-24` の 2 区間なら 2 つの `<path>`）。
- `dial-fill`（経過塗り）は従来通り中心から現在時刻までの扇形。濃色で上書きすることで「すでに過ぎた時間」を視覚化。

#### 5.2.5c 例：2歳・07:00 起床／19:00 就寝／保育園 09:00〜16:00・現在 07:30
```
時間帯       色
0:00-07:00   Sleep（暗色）
07:00-09:00  Free（淡オレンジ）
09:00-16:00  Core（グレー）
16:00-18:00  Free（淡オレンジ）
18:00-19:00  食事バッファ（Free でOK）※ 18:00-19:00 は実質遊べないが、プロト段階では Free 扱い
19:00-24:00  Sleep（暗色）
重ね塗り     0:00-07:30 を Elapsed（濃オレンジ）で上書き
```
→ 結果、朝の遊び（07:00-09:00）の残り 1h30m がオレンジで視認、保育園帯（グレー）7h、夜の遊び（16:00-18:00）2h がオレンジで見える。

#### 5.2.6 アニメーション
- `path` 要素の `d` 属性は JS で直接更新する。CSS transition 非対応のため、**値変更時にフレーム間のスムーズアニメは任意**。プロトでは即時更新で十分。
- 必要に応じて fade 演出（`opacity`）を入れる。

#### 5.2.7 設置場所
| 画面 | 配置 |
|---|---|
| HUD | 小サイズ時計円盤（直径 28px）を HUD 右端に1つだけ配置。テキスト表示は時刻のみ |
| S1 起床 | 中サイズ時計円盤（直径 120px）を画面中央付近に表示 |
| S3 結果フェーズ | 「時間の流れ」カード内に小サイズ（直径 64px）を2つ（before / after）並べる |
| S5 就寝 | 就寝時刻（19:00 や 23:00）の円盤を表示して「もう夜」であることを視覚化 |

### 5.3 経験値Lvゲージ（横長）
- 現在Lvから次のLvまでの進捗を 0〜100% で表示。
- Lvアップ直後は 100% で一度満タン表示 → 次フレームで 0% へリセット（アニメ演出）。
- 色：常にオレンジ（`--accent`）固定。

### 5.4 アニメーション
- `.gauge-bar` は `transition: width 400ms ease-out` を付与し、値の変化をアニメーションで見せる。
- Lvアップ時は 1000ms で 100% → 0% → 新値の順に2段階アニメーション。
- 時計円盤は JS 同期更新（transition なし、瞬時反映）。

### 5.4a 差分ハイライト付きゲージアニメ（v4 追加）

結果画面（SPEC-003）で経験値を表示するときは、**「元のゲージ」→「差分が緑色のハイライトで足されていく」→「ハイライトが一色に馴染む」** のアニメーションで、
**「今回どれだけ経験値が増えたか」を視覚的に伝える**。

#### 5.4a.1 DOM 構造（ハイライト付きゲージ）

```html
<div class="gauge gauge-with-delta">
  <div class="gauge-track">
    <!-- before 時点までの黒（通常色）-->
    <div class="gauge-bar-base"  style="width: {beforePct}%"></div>
    <!-- before から after までの差分（緑ハイライト）-->
    <div class="gauge-bar-delta" style="left: {beforePct}%; width: {deltaPct}%;"></div>
  </div>
  <div class="gauge-label">
    <span>ラベル</span>
    <b>値表示</b>
  </div>
</div>
```

- `.gauge-bar-base` は通常の `--accent`（オレンジ）色
- `.gauge-bar-delta` は緑（`--good` / `#4caf7a` 相当）＋左端スタート＋アニメーション対象

#### 5.4a.2 アニメーションシーケンス

##### (a) 通常プレイ（Lvアップしない）
1. **0ms**：before 状態で描画（base 幅 = beforePct%、delta 幅 = 0%）
2. **150ms 後**：delta に `width: deltaPct%` を設定 → CSS transition で 600ms かけて緑ハイライトが右方向に伸びる
3. **約 900ms 後（任意）**：ハイライトを base 色にフェード（透明度でマージ） ※プロトでは未実装

##### (b) Lvアップ時（必ず 2 段階アニメ）
**重要**：Lvアップ時は **必ずゲージの満タン演出** を挟む。たとえ新Lv 区間で **余剰経験値が 0**（exp がちょうどLv基準値ぴったり）であっても、満タン→リセットのアニメは省略しない。これは「Lvアップした手応え」を視覚的に伝えるための演出。

フェーズ：
1. **フェーズA（0〜600ms）「前Lv満タンアニメ」**
   - ベース = 前Lv の beforePct%
   - 差分 = `100 − beforePct` % を緑ハイライトで伸ばす（常に右端＝100%まで）
   - ラベル：前Lv の表示のまま
2. **フェーズB（600〜900ms）「リセット」**
   - ベース = 0%、差分 = 0% に切替
   - ラベルを **新Lv の表記に更新**（`Lv2 ⬆ Lv UP!` 等）
   - `.lv-up-flash` の軽いフラッシュ（200ms）
3. **フェーズC（900〜1500ms）「新Lv の余剰分」**
   - 差分 = `overflowPct`%（新Lv 区間内に食い込んだ exp 分）を緑で伸ばす
   - 余剰 0 の場合は 0% のまま（アニメは発火しない）。**ただしフェーズ A/B で満タン→リセットの演出はすでに済んでいる**

##### (c) 複数Lvアップ（将来拡張）
- 現状は 1 プレイで +1 Lv までしか発生しない想定のため、複数Lvアップ時は末尾Lvだけに追従する。

#### 5.4a.3 呼び出し API

```
renderGaugeWithDelta(container, {
  label: "身体系 Lv3",
  beforeExp: 90,   // Lv3 到達の最低 exp 90
  afterExp:  108,  // 加算後
  lv:        3,
  lvUp:      false,
  kind:      "exp",  // "exp" / "skill"
  delta:     18,     // 表示用差分
  displayValue: "18/70",
});
```

- 内部で「現在Lv の exp 下限」「次のLv の exp 下限」から `beforePct` と `afterPct` を計算
- lvUp が true の場合は §5.4a.2 の 2 段階アニメを実行

#### 5.4a.5 非対応時のフォールバック
- JS の `requestAnimationFrame` が利用できない環境では、単にアニメーション無しで最終値をセットする（視覚効果がなくなるだけで機能は同じ）。

### 5.5 ゲージ／円盤レンダリング関数
```
function renderGauge(container, { current, max, label, kind, delta }):
  - container を .gauge 構造に置き換える
  - pct = clamp(current / max, 0, 1) * 100
  - color = thresholdColor(kind, pct)
  - bar.width = pct%
  - label に "現在値 / 上限" を描画

function renderClockDial(container, { clockHour, clockMinute, spareHours }):
  - container を .clock-dial 構造に置き換える（SVG）
  - t = clockHour + clockMinute/60
  - path.d = fanPath(t) // §5.2.4 の式
  - text.dial-time = "HH:MM"
  - text.dial-sub  = "残り Xh"
```
- `kind`：`stamina`, `rhythm`, `exp` など。

### 5.6 パフォーマンス
- 再描画はゲージ/円盤単位に局所化。画面全体を再構築しない。
- SVG の `path` 更新は DOM 操作 1回のみ。`requestAnimationFrame` 不要。

## 7. 想定される利用シナリオ

- **起動直後**：2歳プレイヤー → HUD の体力ゲージは `15 / 15` で 100% 緑満タン。時計円盤は 07:00 で `1/4`（=7/24）弱塗りつぶし。
- **朝の遊び終了**：絵本×2 で時計円盤が 09:00 まで進む（3/8 塗りつぶし）。体力ゲージは 15→11 で緑のまま。
- **保育園帰宅直後**：円盤は 16:00（2/3 塗りつぶし）、体力 6/15 で赤。
- **就寝（19:00）**：円盤は 19:00（= 19/24 ≒ 79% 塗りつぶし）、ほぼ真っ白なのは右上の 5/24 だけ。

## 8. 例外・エッジケース

- 上限が 0（例：上限 = 0 の老後末期の仮定）→ パーセント = 0 で固定。
- 現在値が上限を超える（異常系）→ 100% でクランプ、警告ログ。
- 時刻が 24:00 を超える（翌日への跨ぎ）→ 実質的に 24:00 でクランプ。`nextDay()` が呼ばれて新しい日で円盤がリセットされる。
- ダークモード：プロト未対応。将来カラートークン化する。
