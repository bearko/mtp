## 1. 目的

S2「遊び選択」画面の情報重複を整理し、**情報は読ませるのではなく見せる** デザインに改修する。画面中央の既存情報（起床ヘッダー、時計円盤、リズムゲージ等）は HUD に統合し、5 素養は HUD 直下の **共通パラメーターバー** に固定表示する。遊び選択時のコスト・効果はヘッダー／パラメーターバーの **ゴーストプレビュー** で可視化する。

参考：ウマ娘プリティーダービーのトレーニング画面（左：状態、中央：キャラクターと効果プレビュー、右：パートナー一覧）。

## 2. 用語定義
- **ゴーストプレビュー（Ghost Preview）**：プレビュー中、現在値の隣／内部に減少／増加後の値を半透明で重ねる視覚表現
- **共通パラメーターバー（Param Strip）**：5 素養（SPEC-033）を、上段「パラメーター名」＋下段「グレード＋数値」の 2 行×5 列で表示する固定バー
- **リズム 5 段階**：`絶不調 / 不調 / 普通 / 好調 / 絶好調`（`player.bioRhythm` 1-100 を 20 刻みでマッピング）

## 3. 前提
- 旧 HUD は単一行に age/stage/date/clock と 5 指標を圧縮していた
- 旧 S2 中央は 起床ヘッダー（scene-art / scene-title + clock-dial + 体力/リズム + "何して遊ぶ？"）で情報過多だった
- SPEC-033 により内部パラメータは `player.soyou.*` に統合される

## 4. HUD 再設計（3 エリア構成）

### 4.1 レイアウト（モバイル 縦 375×）
```
┌────────────────────────────────────────┐
│  [左エリア]     [中央エリア]    [右エリア] │
│  ライフステージ   年月日（曜日）   💰所持金 │
│  あと 〇時間     リズム（絵文字）   👥友人  │
│                 ❤️ 体力ゲージ(太)   🔥情熱  │
└────────────────────────────────────────┘
```

### 4.2 左エリア
- 1 行目：ライフステージ名（例「保育園1年目」）
- 2 行目：年齢「1歳」
- 3 行目：「あと 〇時間」（`player.spareHours` の残、時計円盤を廃止）

### 4.3 中央エリア
- 1 行目：日付「2026年4月1日（水）」（SPEC-001 §5.7）
- 2 行目：リズム（絵文字 + ラベル）
  - 絶不調（😵 / 1-20）
  - 不調（😞 / 21-40）
  - 普通（😐 / 41-60）
  - 好調（😊 / 61-80）
  - 絶好調（🤩 / 81-100）
  - **幼少期（age ≤ 6）は常に「絶好調」固定**（SPEC-001 §4 の低年齢リズム保護と整合）
- 3 行目：**❤️ 大きな体力ゲージ**（横幅は中央エリアの 80% 以上、高さ 12-14px、3 段階カラー：赤 20% / 黄 50% / 緑）
  - 現在値 / キャップ を数値で右端に併記
  - **ゴーストプレビュー**：遊び選択時は `staminaCost` 分を右から半透明の赤で差し引いた箇所を重ねる

### 4.4 右エリア
- 💰 所持金：`player.money` 円（**幼少期 age≤6 は非表示**）
- 👥 友人：`player.friends`
- 🔥 情熱：`player.soyou.passion` の **グレード + 数値**（旧ゲージから簡素化）
- **📍 現在の場所（v3 で追加 / SPEC-047 §6）**：`player.location` の icon + shortName

#### 4.4.1 v3 改訂：情熱表示 → 場所表示
情熱（passion）は素養カード 5 枚の 1 つに既に表示されているため、HUD の右エリアに単独表示するのは冗長。SPEC-047 §6 により **情熱表示を場所表示に置き換え** た。
- 旧 `#hud-passion-row` は DOM として残すが `hidden` 属性で非表示
- 新 `#hud-location` に location の icon + shortName を表示（例：🏠 自宅 / 🌳 近公園 / 🦒 動物園）

### 4.5 レスポンシブ
- 375px 幅：3 エリアを grid 4:5:3 で配置
- 360px 未満：各エリアの 2 行目以降のラベルを省略

## 5. メインエリア（画面中央）

### 5.1 レイアウト
```
┌──────────────────────────┐
│ [HUD]                    │
├──────────────────────────┤
│ 身体 知性 感性 社交 情熱  │ ← 共通パラメーターバー上段
│ G0  G0  G0  G0  G0       │ ← 共通パラメーターバー下段
├──────────────────────────┤
│ きょうのめあて             │
│ （ここにスキルタグ）        │ ← プレビュー時のみ
└──────────────────────────┘
```

### 5.2 共通パラメーターバー（`#param-strip`）
- 表示位置：HUD の直下。`S2 / S3 / S4 / S10 / S9` で同じ位置に表示する
- 上段：`身体 / 知性 / 感性 / 社交 / 情熱`
- 下段：上に大きいグレード、下に数値を縦積みする。グレード変化時も数値が見切れないよう、同じ列内で改行表示する
- プレビュー時：該当素養に `+N` の小バッジを数値の下へ添える。グレードアップ時は Before/After 横並びではなく `↑B` のように上昇記号 + 到達グレードだけを表示する
- 旧 S2 の 5 行表（`#soyou-list`）は DOM 互換のため残すが hidden とし、描画内容は空にする

### 5.3 ゴースト表現
- **増加予測**：共通パラメーターバー下段に `+10` を緑バッジで添える
- **グレードアップ**：グレード値を `↑F` のように表示
- **コスト**：ヘッダー体力ゲージに「減少後」の位置を半透明赤で重ねる。残り時間も「-〇h」を右に添える

### 5.4 スキルタグ表示
- 共通パラメーターバーの下、S2 の本文上部にプレビュー中のみ表示
- 「#reading #literacy #sensitivity」のようにハッシュタグ風に並べる
- 獲得済みスキルは白、新規は黄色アクセント

### 5.5 将来拡張
- 画面最上段に「あと〇ターン」の表示を置くエリアを確保しておく（ウマ娘参考）。
- 今回は `data-turn-hint` 属性だけ用意し、表示は空のまま

## 6. ドック（v6：遊び専用 + 右下サブメニュー）
- 画面最下段のドックには遊びアイコンだけを並べる
- 固定フッターの「1h休む / 今日おわる / 遊ぶ」は表示しない
- 遊びは「選択 → 同じアイコン再タップ」で確定する
- 遊び以外の `1h休む / ツリー / 今日おわる / きろく / 完走` は右下固定ハンバーガーメニューにまとめる
- 遊びアイコンは 78×72px に拡大し、絵文字と 1 行ラベルを併記する

## 7. プレビュー制御フロー

1. プレイヤーが dock の遊びアイコンをタップ → `selectPlay(id)`
2. `renderPlayPreview(play)` を呼ぶ
   - ヘッダー体力ゲージに `staminaCost` のゴースト
   - 共通パラメーターバーの該当キーに `(+N)` / グレードアップ矢印
   - スキルタグリストを表示
3. もう一度タップ → `startPlay(play)`
4. キャンセル（dock で×ボタン） → `clearPlayPreview()`

## 8. 互換性
- 旧要素（`#wakeup-header`, `#hud-stage`, `#hud-clock` 等）の一部は新 HUD から撤去する
- ただし **DOM 削除ではなく display:none** にしてリスク回避：まず見せなくして、後の SPEC で削除
- `renderHUD()` は新要素 / 旧要素の両方を更新する（旧要素は HUD レイアウト外で、見えないが値は書き込む）

## 9. 画面外の影響
- S10 日サマリや S9 週ハイライトは本 SPEC の対象外
- 素養キーの変更は SPEC-033 で扱う。表示粒度のグレード等も SPEC-033 §4 を参照

## 10. テスト観点
- 起動時の HUD 表示：左「保育園1年目 1歳 あと8時間」／中央「2026年4月1日（水） 😐普通 ❤️15/15」／右「👥2 🔥G(0)」（幼少期なので所持金なし）
- 絵本をタップするとヘッダー体力ゲージに -4 ゴースト、共通パラメーターバーの知性に `+10`
- もう一度タップすると遊び開始
- 幼少期（age≤6）はリズム常に絶好調
- 素養値 20 到達で F、40 で E、66 で D、80 で C、110 で B、140 で A

## 6. S3 結果画面・S9・S10 のボタン整理（v2 追加）

### 6.1 S3 結果画面から「今日おわる」ボタンを廃止（誤タップ防止）
- 旧：「🔁 もう一度遊ぶ / 次の遊びを選ぶ / 今日おわる」の 3 ボタン
- 新：**「🔁 もう一度遊ぶ / 次の遊びを選ぶ」の 2 ボタン**
- 余剰時間が尽きた場合は `actions-result-sleep` が自動で切り替わり「🌙 おやすみ」が出るので、プレイヤーから能動的に「今日おわる」を押す必要はない

### 6.2 S9 / S10 の「手動に切り替え」を自動モード解禁後のみ表示
- Phase 0（Day 1〜7）はそもそも自動モード封印（SPEC-026 §5.2）なので、このボタンは意味を持たない
- 自動モードが有効（`player.autoMode === true`）かつ `tutorialPhase(player.day) !== "phase0"` のときのみ表示
- 実装：
  - `showHighlight()`（S9 ウィークリーハイライト）で `#btn-highlight-switch-manual` の `hidden` を切り替え
  - S10 日サマリでは手動切替を表示しない。操作は `次の日へ` と `週末までスキップ` に絞る（SPEC-025 §7.1.3）。

### 6.3 素養カード `(+N)` バッジと実数値の視認性改善
- 旧：`.soyou-delta` を `position: absolute` で value の上に重ね、opacity 0.4 で value を薄くしていた → 「10」に「+10」が重なって読みにくい
- 新：grid に専用列を追加し、delta バッジと value を **並列表示**
  - 5 列：`icon | label | grade | delta-badge | value`
  - value 列は「0 → 10」と現→後の遷移表示を維持、delta 列は独立した `+10` ピル
  - 空のときは delta セルを空の span にして grid のブレを防ぐ

## 6.4 v3：旧 DOM 参照の null ガード（自動モード起動不能バグ修正）

### 症状
Phase 1 到達 → 情熱プロファイル決定 → 自動モード ON、あるいは dock-mode トグルで手動→自動に切り替えた際、**自動進行が一切動かない** 現象が発生していた。

### 原因
`startAutoLoop()` と `stopAutoLoop()` が、SPEC-034 v1 で index.html から削除された旧要素（`#wakeup-header`, `#choose-prompt`, `#preview-placeholder`, `#preview`）に対して `byId(...).hidden = true` を呼んでいた。`byId()` が null を返すので `.hidden = true` で **`TypeError: Cannot set properties of null`** が発生し、`runAutoTurn()` まで辿り着けなかった。

手動モードのときは `renderChooseScreen()` 内で同じ参照を既に null ガード付きで書き換え済みだったため問題が出ず、**自動モードだけ壊れている** 状態だった。

### 修正
`startAutoLoop` / `stopAutoLoop` 内のすべての `byId(...)` 参照を、**存在する要素だけを触る null ガード付き** に書き換える。

```js
function startAutoLoop() {
  stopAutoLoop();
  const wh = byId("wakeup-header"); if (wh) wh.hidden = true;
  const pv = byId("preview");       if (pv) pv.hidden = true;
  const pp = byId("preview-placeholder"); if (pp) pp.hidden = true;
  const cp = byId("choose-prompt"); if (cp) cp.hidden = true;
  const ap = byId("auto-progress"); if (ap) ap.hidden = false;
  const cb = byId("btn-confirm-play");
  if (cb) { cb.disabled = true; cb.textContent = "自動進行中…"; }
  runAutoTurn();
}
```

### 教訓
UI リデザインで DOM を削除した際、`byId()` で直接参照しているすべての呼び出し箇所を検索して null ガードを入れる（あるいは参照ごと削除する）必要がある。本件は `renderChooseScreen()` 側は修正済みだったが、`startAutoLoop()` / `stopAutoLoop()` の同種参照が見落とされていた。

### 6.5 v5：共通パラメーターバー固定表示
- S2 中央の 5 行素養表を廃止し、HUD 直下に `#param-strip` を追加する。
- `#param-strip` は `S2 / S3 / S4 / S10 / S9` で同じ位置に表示し、常に 2 行×5 列で 5 素養を見せる。
- 画面固有のスキル・イベント本文・連絡帳は、その下の `.scene` 内に配置する。
