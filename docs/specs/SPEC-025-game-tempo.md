# SPEC-025: ゲームテンポ設計（自動進行 + ステージ依存の時間流速）

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-025 |
| 機能名 | ゲームテンポ設計（プレイヤー駆動のスキップ + ライフステージ依存の時間流速） |
| 対応ファイル | `prototype/game.js` の `PASSION_PROFILES`, `tempoDaysPerTurn()`, `runSkipBatch()`, `setPassionProfile()` / `prototype/index.html` の `#screen-passion-profile`, `#screen-skip-summary` |
| 関連仕様 | SPEC-001, SPEC-002, SPEC-003, SPEC-010, SPEC-011, SPEC-020, SPEC-022, SPEC-023, SPEC-024 |
| ステータス | Active（Option B 実装中、Option A は別ブランチで検討） |
| 最終更新 | 2026-04-18 |

> 本仕様は Issue #7「ゲームのテンポを良くしたい」に対応する。  
> 論点：1日×100年＝36,500 ターン、1日3判断とすると 10万タップ超でゲームテンポが悪い。  
> プレイヤーに **「人生全体」を 30 〜 60 分で体験させる** ためには、タップ数とテンポの最適化が必須。

---

## 1. 目的

- プレイヤーの **意思決定タップを 1 プレイあたり 200〜400 タップ** に圧縮する
- **「まとめて進めて、まとめて振り返る」** 体験を提供し、プレイヤーが能動的に時間を操れる感覚
- **幼少期はゆっくり、社会人以降は速く** 時間が進むことで、人生のリアリティと時間感覚を再現
- 判断機会を **年齢フェーズに応じて最適な粒度** で提供する

## 2. 用語定義

- **情熱プロファイル（Passion Profile）**：プレイヤーが最初に（または節目で）設定する「遊びの好み」。例：公園派、インドア派、読書家
- **スキップ（Skip）**：プレイヤーが「◯日分まとめて進める」ボタンを押して、指定の日数分を自動処理する操作
- **スキップ単位（Skip Unit）**：1 回のスキップで進める日数。年齢に応じて変化（1日/3日/1週間/1ヶ月など）
- **介入イベント（Interruption Event）**：スキップ中でもプレイヤーのタップを求める出来事（Lv アップ、友人が増えた、解禁遊び出現、ライフステージ変化、体力ゼロ）
- **スキップサマリ（Skip Summary）**：スキップ完了後に表示される「過ぎた期間の出来事」まとめ
- **時間流速（Tempo）**：1 スキップ単位でゲーム内何日／何週間／何ヶ月進むかの単位

## 3. 基本方針（Issue #7 のB+Cハイブリッド）

- 案C「自動進行・自動選択」を **プレイヤー駆動のスキップ** として採用（能動的なテンポコントロール）
- 案B「ライフステージ依存の時間流速」を **スキップ単位** に反映
- 案A「週単位」は週タームをスキップの基本単位として扱うが、ステージによって 1〜30 日の可変単位にする

## 4. ライフステージ別のスキップ単位（Tempo Table）

`tempoDaysPerTurn(age)`：**1 回のスキップで経過する日数** を返す関数。年齢別：

| 年齢帯 | ライフステージ | スキップ単位 | 理由 |
|---|---|---|---|
| 1〜4 | 保育園 | **1 日** | 最初の学びは丁寧に体験 |
| 5〜6 | 幼稚園 | **3 日** | 数日ぶんまとめて |
| 7〜12 | 小学校 | **1 週間（7日）** | 平日×5＋休日×2 |
| 13〜15 | 中学校 | **1 週間** | 部活動リズム |
| 16〜18 | 高校 | **1 週間** | 同上 |
| 19〜22 | 大学 | **2 週間（14日）** | 長期休みが多い |
| 23〜35 | 社会人（若手） | **1 ヶ月（30日）** | 忙しく毎日の記憶が薄い |
| 36〜50 | 社会人（中堅） | **1 ヶ月** | |
| 51〜65 | 社会人（熟練） | **2 ヶ月（60日）** | 毎年があっという間 |
| 66〜 | 老後 | **1 ヶ月** | 再び時間を味わう |

### 4.1 プロト段階の扱い

- **保育園（1-4歳）だけ** 実装されているため、スキップ単位は「1日」
- ただしプレイヤーが 1 日単位でタップするのは退屈なので、**「今日はおまかせで進める」ボタン** を「1日スキップ」として扱う
- 幼稚園以降の実装時に週/月単位に自動昇格する

## 6. スキップ操作の骨組み

### 6.1 「◯日（単位）まとめて進める」ボタン

遊び選択画面（S2）の **起床ヘッダーの右側** に、ステージに応じた単位で「まとめて進める」ボタンを配置する。

- ラベル例（保育園）：`⏩ 今日はおまかせ`
- ラベル例（幼稚園以降）：`⏩ 3日まとめて`, `⏩ 今週まとめて`, `⏩ 今月まとめて`

タップすると以下の処理が走る：

```
function runSkipBatch(days):
  for d in 0..days:
    while プレイヤーの余剰時間 > 0:
      play = pickAutoPlay()
      autoFinalizePlay(play)
      if 介入イベント発生:
        モーダル表示 → [続ける] で次のイテレーションへ
    if コアタイムに到達 && 未消化:
      S6 自動消化（省略演出）
    次の日へ進む（nextDay）
  スキップサマリ画面を表示
```

### 6.2 スキップ中の画面演出
- スキップ中は **S2 を薄くグレーアウト** し、中央に **「⏩ N日後までまとめて進めているよ…」** のオーバーレイを表示
- 介入イベント時のみモーダルで一時停止

### 6.3 手動モードとの切替
- ボタンを押さない限り、従来どおり 1 ターン手動プレイ
- いつでも「⏩ まとめて」ボタンを押せる

## 7. 介入イベント（Interruption Event）

スキップ中に以下のイベントが発生したら必ず停止してプレイヤーのタップを求める：

| イベント | 介入内容 |
|---|---|
| Lv アップ | モーダル「○○スキルがLv○に上がった！」 |
| 新しい遊びの解禁 | モーダル「○○を覚えた！」 |
| ライフステージ切替 | モーダル「幼稚園に入園した」 |
| 体力ゼロ | 強制睡眠／強制終了フロー（SPEC-019） |
| 友人増減（±2 以上） | モーダル通知 |
| ランダムイベント（SPEC-004） | そのまま S4 表示 |

介入後に「⏭ 続けて進める」ボタンで残りのスキップを再開。

## 8. スキップサマリ画面（新画面 S9）

スキップ完了時または介入途中時点で、**過ぎた期間の出来事** を俯瞰するサマリ画面に遷移する。

### 8.1 表示項目
- 期間（○日目〜○日目、○歳×月×日）
- 遊んだ回数（遊び別、降順 3 件まで）
- 伸びたスキル（Lv アップしたスキル）
- 発見（新しい遊び、友人増加など）
- 合計経験値・ジョウネツ

### 8.2 フッター
- `続ける` または `もう一度まとめて進める` → 再度同じスキップ単位でスキップ
- `手動で遊ぶ` → S2 遊び選択画面に戻る

## 10. Option A と Option B（実装パターン比較）

本 Issue #7 に対して、**B + C ハイブリッド** の解釈で 2 パターンの実装を用意する。

| 観点 | Option A：自動進行ダイジェスト型 | Option B：週／月スキップ型 |
|---|---|---|
| コア体験 | 1 ターンずつ自動で流れ、5 ターン毎にダイジェスト | **プレイヤーが「1週間進める」ボタンで一気にスキップ** |
| プレイヤーのタップ | 介入イベントでのみ（受動的） | ボタンを押して進める（能動的） |
| 時間感覚 | 「知らぬ間に時間が過ぎる」没頭感 | **「まとめて振り返る」俯瞰感** |
| 実装の軽さ | 自動進行ループを作るため少し重い | **ボタン 1 つで複数ターン処理、軽め** |
| 手動介入 | トグルでいつでも手動化 | **ボタンを押さなければ手動（デフォルト）** |
| 向いている層 | 放置系ゲーム・ダイジェスト好き | 能動的操作派・セッション型ゲーム好き |
| リスク | 退屈・飛ばしすぎ | 押下の手間・リズム感が単調 |

### 10.1 本ブランチ（cursor/tempo-option-b-f03c）の担当
- **Option B：週／月スキップ型** を実装
- プレイヤーが「⏩ まとめて進める」ボタンを押すと、指定日数分を一気に自動処理
- サマリ画面で「過ぎた期間の出来事」をまとめて表示
- 自動進行ループは無し（押したタイミングでのみ進む）

### 10.2 比較ブランチ（cursor/tempo-option-a-f03c）の担当
- **Option A：自動進行ダイジェスト型** を実装（別 PR）

## 11. 想定される利用シナリオ

### 11.1 Option B での 2歳プレイヤー
- 初回：「🌳 外が好きな子」を選択
- 起床 → S2 に 🌳 の遊びが並ぶ。右上に **`⏩ 今日はおまかせ`** ボタン
- そのボタンを押すと：朝 2h + 保育園 + 夜 2h が **3 秒程度で一気に進行** → S9 スキップサマリ画面
- プレイヤーが「発見：🎢 大きな滑り台」「遊具スキル Lv 1→2」「ジョウネツ+15」などをまとめて確認
- 「⏩ もう一度まとめて進める」で翌日も一気に進められる
- 気分が乗れば「🎮 手動で遊ぶ」で従来 UI に戻る

## 12. 例外・エッジケース

- **プロファイル未選択**：初回起動時は強制的に S8 選択画面を表示。キャンセル不可
- **スキップ中の介入**：モーダル表示中はスキップ停止。「続ける」で再開
- **スキップ中に体力ゼロ**：強制睡眠／強制終了（SPEC-019）へ。残りのスキップはキャンセル
