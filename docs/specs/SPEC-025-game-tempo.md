# SPEC-025: ゲームテンポ設計（自動進行 + ステージ依存の時間流速）

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-025 |
| 機能名 | ゲームテンポ設計（自動進行 + ライフステージ依存の時間流速） |
| 対応ファイル | `prototype/game.js` の `PASSION_PROFILES`, `tempoDaysPerTurn()`, `runAutoTurn()`, `setPassionProfile()` / `prototype/index.html` の `#screen-passion-profile` |
| 関連仕様 | SPEC-001, SPEC-002, SPEC-003, SPEC-010, SPEC-011, SPEC-020, SPEC-022, SPEC-023, SPEC-024 |
| ステータス | Active（Option A 実装中、Option B は別ブランチで検討） |
| 最終更新 | 2026-04-18 |

> 本仕様は Issue #7「ゲームのテンポを良くしたい」に対応する。  
> 論点：1日×100年＝36,500 ターン、1日3判断とすると 10万タップ超でゲームテンポが悪い。  
> プレイヤーに **「人生全体」を 30 〜 60 分で体験させる** ためには、タップ数とテンポの最適化が必須。

---

## 1. 目的

- プレイヤーの **意思決定タップを 1 プレイあたり 200〜400 タップ** に圧縮する
- **「没頭＝自動で楽しい時間が過ぎる」** 感覚を演出し、プレイヤーは「節目」や「大事な選択」だけに介入
- **幼少期はゆっくり、社会人以降は速く** 時間が進むことで、人生のリアリティと時間感覚を再現
- 判断機会を **年齢フェーズに応じて最適な粒度** で提供する

## 2. 用語定義

- **情熱プロファイル（Passion Profile）**：プレイヤーが最初に（または節目で）設定する「遊びの好み」。例：公園派、インドア派、読書家、スポーツ派、クリエイター…
- **自動ターン（Auto Turn）**：プレイヤーの介入なしにゲームが 1 ターン（定義による）を消化する処理
- **介入イベント（Interruption Event）**：自動ターン中でも **プレイヤーのタップが必要** な出来事。例：Lv アップ、友人が増えた、解禁遊び出現、ライフステージ変化、体力ゼロ
- **時間流速（Tempo）**：1 ターンでゲーム内何日／何週間／何ヶ月進むかの単位
- **ハイライト（Highlight）**：自動ターンの後にまとめて表示される「この期間の出来事」のダイジェスト

## 3. 基本方針（Issue #7 のB+Cハイブリッド）

- 案C「自動進行・自動選択」を **全体の骨組み** として採用
- 案B「ライフステージ依存の時間流速」を **自動進行の粒度** に反映
- 案A「週単位」は週タームを基本粒度として保持するが、ステージによって 1〜30 日の可変単位にする

## 4. ライフステージ別の時間流速（Tempo Table）

`tempoDaysPerTurn(age)`：**1 回の自動ターンで経過する日数** を返す関数。年齢別：

| 年齢帯 | ライフステージ | 1 ターンの日数 | 1 年あたりのターン数 | 理由 |
|---|---|---|---|---|
| 1〜4 | 保育園 | **1 日** | 約120 | 最初の学びは丁寧に体験させる |
| 5〜6 | 幼稚園 | **2 日** | 約60 | 少しずつ加速 |
| 7〜12 | 小学校 | **3 日** | 約40 | 学校生活の繰り返し |
| 13〜15 | 中学校 | **7 日（1週間）** | 約17 | 週単位の部活動リズム |
| 16〜18 | 高校 | **7 日** | 約17 | 同上 |
| 19〜22 | 大学 | **14 日（2週間）** | 約8〜9 | 長期休みが多く、月の半分単位 |
| 23〜35 | 社会人（若手） | **30 日（1ヶ月）** | 約4 | 忙しくて毎日の記憶が薄い |
| 36〜50 | 社会人（中堅） | **30 日** | 約4 | |
| 51〜65 | 社会人（熟練） | **60 日（2ヶ月）** | 約2 | 毎年があっという間 |
| 66〜 | 老後 | **30 日** | 約4 | 再び時間を味わいたくなる |

### 4.1 人生を通算したターン数

上記テーブルで 1〜100 歳までを走破すると **約 1,100〜1,200 ターン** になる。
1 ターン 1 タップ（＋介入イベント 10〜20%）なら、**総タップ数 1,300〜1,500** 程度に収まる。
30〜60 分プレイで完走可能なペース。

### 4.2 プロト実装の段階的適用

- v1（保育園のみ実装済み）：年齢 1〜4 歳は `1 日 / ターン`（従来通り）
- v2（本仕様で保育園に自動進行を導入）：保育園も **1 日 / ターン** のまま、**自動遊び選択** のモードを提供
- 将来：幼稚園以降の実装時に Tempo Table を適用して、ターン粒度を拡げる

## 5. 情熱プロファイル（Passion Profile）

### 5.1 データ構造

```ts
type PassionProfile = {
  id: string;
  label: string;
  icon: string;
  description: string;
  preferredCategories: string[];  // SPEC-022 のカテゴリ ID を優先度順に列挙
  color: string;
};
```

### 5.2 プロト段階のプロファイル 4 種（保育園向け）

子どもが持つ「好みの芽生え」を 4 種類に要約。将来はステージごとに拡充する。

| ID | ラベル | 好む要素 |
|---|---|---|
| `outdoor_kid` | 🌳 外が好きな子 | `movement`, `outdoor_toy`, `nature`, `water` |
| `creator_kid` | 🎨 ものづくりが好きな子 | `crafting`, `imagination`, `music` |
| `bookworm_kid` | 📚 じっくり派 | `reading`, `literacy`, `sensitivity`, `science` |
| `social_kid` | 🤝 友達といる子 | `social_play`, `friendship`, `imagination` |

### 5.3 選択タイミング

- **初回起動時**：必須。選ばないとプレイ開始できない
- **ライフステージ切替時**：将来実装。プロフィールをアップデートできる
- **プロト段階**：保育園開始時に一度だけ選択

### 5.4 自動選択アルゴリズム

```
function pickAutoPlay(availablePlays, profile):
  // 1) スキル×好みカテゴリスコアを計算
  scored = availablePlays.map(play => ({
    play,
    score: computeScore(play, profile)
  }))

  // 2) スコア上位 3 つの中から重み付き抽選（常に同じ遊びにならないように）
  topK = sortDesc(scored)[:3]
  return weightedPick(topK)
```

#### 5.4.1 スコア式

```
score(play, profile) =
    (profile.preferredCategories ∩ play.categories).length × 10
  + averageSkillLv(play.categories) × 0.5      // 熟練した遊びをしたい性向
  + (play.unlockRequired ? 5 : 0)              // 解禁されたばかりの遊びは試したい
  + randomJitter(0, 2)                         // 少しだけ揺らぐ
```

### 5.5 「自分で選ぶ」モードとの切替

- 自動進行モード中でも、プレイヤーが **「自分で選ぶ」ボタン** を押せば従来の S2 遊び選択画面に戻る
- 自動進行 ↔ 手動進行 の切替は **ドック最左端のトグル** にする（Option A の実装）

## 6. 介入イベント（Interruption Event）

自動進行中でも、以下の **重要イベントは必ずプレイヤーに見せて介入を求める**：

| イベント | 介入内容 |
|---|---|
| Lv アップ | 結果画面で通知、タップで次へ |
| 新しい遊びの解禁 | 「◯◯を覚えた！」のモーダル |
| ライフステージ切替 | 「幼稚園に入園した」のモーダル |
| 体力ゼロ（強制睡眠 / 強制終了） | 従来どおり（SPEC-019） |
| 友人増減（±2 以上の大きな変化） | モーダルで通知 |
| ランダムイベント（SPEC-004） | そのまま S4 表示 |
| **プレイヤーが手動介入を選択** | 「自分で選ぶ」ボタン押下 |

介入イベントが **発生しないターン** は、0.3〜0.8 秒でサッと 1 ターンが進み、サマリ（後述）が数秒表示される。

## 7. ハイライト／サマリ

自動ターン後は以下の順で表示する：

1. **即時ミニトースト**（0.8s）：「砂場で遊んだ」
2. **ゲージ差分**（SPEC-021 §5.4a のアニメ）：獲得経験値 / スキル
3. **ハイライト集約**：一定ターン数ごとに「この 5 ターンのまとめ」カードを出し、タップで次へ

集約タイミング：
- 保育園：5 ターンに 1 度（= 5日に 1 度）
- 幼稚園：5 ターンに 1 度（= 10日に 1 度）
- 以降は Tempo Table と同じ粒度

## 8. 選択肢の重要度による粒度切替（Option A の差別化）

- 自動ターンは **ほぼ全自動**
- プレイヤーが手動に切り替えると従来の S2 画面で 1 ターン介入
- 介入イベントは必ずプレイヤーを待つ

## 10. Option A と Option B（実装パターン比較）

本 Issue #7 に対して、**B + C ハイブリッド** の解釈で 2 パターンの実装を用意する。

| 観点 | Option A：自動進行ダイジェスト型 | Option B：週／月スキップ型 |
|---|---|---|
| コア体験 | **1 ターンずつ自動で流れ、5 ターン毎にダイジェスト** | **プレイヤーが「1週間進める」ボタンで一気にスキップ、直後にサマリ** |
| プレイヤーのタップ | 介入イベントでのみ（受動的） | ボタンを押して進める（能動的） |
| 時間感覚 | 「知らぬ間に時間が過ぎる」没頭感 | 「まとめて振り返る」俯瞰感 |
| 実装の軽さ | 自動進行ループを作るため少し重い | ボタン 1 つで複数ターン処理するだけ、軽め |
| 手動介入 | トグルでいつでも手動化 | ボタンを押さなければ手動（デフォルト） |
| 向いている層 | 放置系ゲーム・ダイジェスト好き | 能動的操作派・セッション型ゲーム好き |
| リスク | 退屈・飛ばしすぎ | 押下の手間・リズム感が単調 |

### 10.1 本ブランチ（cursor/tempo-option-a-f03c）の担当

- **Option A：自動進行ダイジェスト型** を実装
- 自動進行ループ（0.5〜0.8 秒/ターン）
- 介入イベントで一時停止
- 5 ターン毎のハイライト集約

### 10.2 比較ブランチ（cursor/tempo-option-b-f03c）の担当

- **Option B：週／月スキップ型** を実装
- プレイヤーが「1 週間進める」「1 ヶ月進める」ボタンを押すと複数ターンを一気に処理
- サマリ画面で「過ぎた期間」をまとめて表示
- 自動進行ループは無し（押したタイミングでのみ進む）

## 11. 想定される利用シナリオ

### 11.1 Option A での 2歳プレイヤー
- 初回：「🌳 外が好きな子」を選択
- 2歳・夏：自動モードで砂場・滑り台・砂場と 3 ターン（3日分）が 2 秒で流れる
- 4 ターン目：発見「🎢 大きな滑り台」→ 介入モーダル、プレイヤーがタップして次へ
- 5 ターン目：ハイライト「この 5 日間のまとめ」表示 → 続行
- 飽きたら「🔀 手動」に切替て従来のドック選択に戻る

### 11.2 Option B での 2歳プレイヤー
- 初回：プロファイル選択
- 起床画面 →「1 週間進める」ボタン → 5 ターン分の自動処理 → サマリ
- 気分が乗らなければ「手動で 1 ターン」で従来通り遊ぶ

## 12. 例外・エッジケース

- **プロファイル未選択**：初回起動時は強制的に S8 選択画面を表示。キャンセル不可
- **自動選択の結果、すべてロック**：手動モードに自動で切替、ドックにも「遊べるものがない」メッセージ
- **介入イベントが連続発生**：各イベントをキューイングし、1 つずつ順番にモーダル表示
