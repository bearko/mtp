# SPEC-023: 遊びツリー（S7 画面）

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-023 |
| 機能名 | 遊びツリー（解放条件をツリー形式で閲覧する画面） |
| 対応ファイル | `prototype/game.js` の `renderPlayTreeScreen()`, `buildTreeNodes()`, `formatUnlockRequirement()` / `prototype/index.html` の `#screen-tree` |
| 関連仕様 | SPEC-001, SPEC-002, SPEC-005, SPEC-007, SPEC-010, SPEC-019, SPEC-022, SPEC-024 |
| ステータス | Active（保育園分のみ実装） |
| 最終更新 | 2026-04-18 |

## 1. 目的

プレイヤーが **「この先どんな遊びがあるか」を能動的に探索** できる画面を提供する。  
ツリー構造で解放条件を可視化することで、「自分の選択次第で遊びの幅はいくらでも広がる」ことを自覚させる。

プロト段階ではライフステージ＝保育園の範囲のみ表示（先のステージの存在は「？？？」系ノードで示唆）。

## 2. 用語定義

- **遊びツリー**：遊びをノードとしたグラフ構造。各ノードは「解放条件」を持つ。
- **公開範囲**：現在のライフステージに対して表示されるノード範囲。
- **ノード状態**：以下の 4 状態のいずれか。
  - **遊び中（Playable）**：今すぐ遊べる
  - **手前（NearLock）**：条件を満たせば近いうちに遊べる（同ライフステージ内）
  - **先のステージ（Future）**：現ライフステージでは解放不可。シルエット表示。
  - **未開示（Hidden）**：ヒント級。ライフステージを跨いでもまだ見せない。

## 3. 入力（Input）

- プレイヤーの現状態（SPEC-001）
- 遊びマスタ `PLAYS`（カテゴリ／条件を含む）
- カテゴリマスタ（SPEC-022 §5.1）
- 解禁履歴 `player.unlockedPlays`

## 4. 出力（Output）

- 遊びツリー画面（S7）の DOM 描画
- ノードのタップで詳細情報を表示（パネル or トースト）

## 5. 挙動（Behavior）

### 5.1 画面遷移

- 遊び選択画面（S2）のドック右端に **遊びツリーアイコン**（🌳）を固定配置
- タップで S7 へ遷移
- S7 の戻るボタンで S2 へ戻る
- **遷移元／遷移先**：S2 ↔ S7 の双方向のみ（S3/S5/S6 からは遷移しない）

### 5.2 ライフステージ別の公開範囲

現ライフステージに応じて「表示対象」を 3 帯に分けて描画：

| 帯 | 表示内容 | ノード状態 |
|---|---|---|
| 上段「今のステージ」 | 現ライフステージの全遊び | Playable / NearLock / 解禁待ち |
| 中段「次のステージ」 | 次のライフステージの **見出し** のみ | Future（「もうすぐ幼稚園…」等の紹介） |
| 下段「もっと先」 | 遥か先の概念。**1 行だけのティザー** | Hidden |

プロト段階では保育園ステージなので：
- 上段：保育園ステージの全遊び
- 中段：「幼稚園（4歳〜）」の案内と代表遊び 2〜3 個（シルエット）
- 下段：「小学校より先…」の 1 行

### 5.3 ノードレイアウト

- **縦スクロール** の長いリスト（ツリーと呼ぶが、プロト段階では縦並びで十分）
- カテゴリごとに **セクション分け**（タブ風ヘッダー）：身体 / 創作 / 探求 / 交流 / 生活 / 文化
- ノード 1 件：アイコン + 名前 + 解放状態 + 解放条件の要約
- **「今遊べる」ノードは上に** 寄せる（SPEC-002 §5.5 と同じ並び順）

### 5.4 ノードの解放条件表示

`formatUnlockRequirement(play)` が生成する 1 行説明を右寄せで表示。プロト段階の文言例：

| 条件 | 表示テキスト |
|---|---|
| 年齢 `ageMin=3` | `3歳〜` |
| 年齢 `ageMax=10` | `〜10歳` |
| 季節 `seasons=[summer]` | `夏限定` |
| 所持金 `moneyCost=300` | `¥300必要` |
| 友人数 `minFriends=1` | `友人 1人以上` |
| スキル要求（将来） | `遊具Lv5` |
| 発見解禁 `unlockRequired` | `発見で解禁` |
| コアタイム入手 | `保育園で発見` |

複数条件は `/` 区切りで表示：例 `夏限定 / ¥300必要`。

### 5.5 ノード状態の判定

```
function classifyTreeNode(play, player):
  if play.lifeStage !== currentStage: return "Future"      // 先のステージ
  if play.unlockRequired && !unlocked: return "NearLock"   // 発見待ち（保育園で見つけるはず）
  avail = isPlayAvailable(play)
  if avail.ok: return "Playable"
  return "NearLock"                                        // 条件を満たせば遊べる
```

### 5.6 ノードのタップ挙動

- **Playable**：S2 相当のプレビューを上部ダイアログで表示、「遊ぶ」ボタンで S3 へ直行（将来実装）。プロトでは「戻る」→ S2 へ戻してドックで選ぶ導線に留める
- **NearLock**：条件の詳細をトースト or モーダルで表示（プロトではトースト）
- **Future**：「幼稚園ステージで遊べるようになります」のトースト
- **Hidden**：タップ不可

### 5.8 データソースの要約

- 遊びマスタからツリーを自動構築（手書きのノード配置は不要）
- ライフステージは `PLAYS[*].ageMin / ageMax` からキーアクセス
- プロト段階ではカテゴリごとのセクションのみで、縦並びのリスト形式。将来は視覚的なノードグラフも検討（未実装）

## 6. UIへの反映

- 画面：S7 遊びツリー（新規、`#screen-tree`）
- ドック：S2 の遊びアイコン列の **一番右** に `🌳` を固定（タップで S7）
- 画面設計書：`docs/screens/SCREEN-DESIGN.md` §2.7（新設）

## 7. 想定される利用シナリオ

- 2歳プレイヤーが保育園コアタイムで「大きな滑り台」を未解禁のとき、ツリー画面で 🎢 が「保育園で発見」でロック表示されている → 「保育園で何かを見つければ解放されるんだな」と理解
- 幼稚園ステージ紹介を見た親プレイヤーが「4歳になったら図鑑が読めるようになるらしい」と先の楽しみを知る
- 現時点で遊べる遊びを S2 で絞り込む（SPEC-002 §5.1 の非表示ルール変更）代わりに、遊びたい場合は ツリー経由で全体像を把握できる

## 8. 例外・エッジケース

- 遊びマスタが空の場合：「まだ遊びがありません」表示
- カテゴリセクションがすべて空：そのセクションの見出しごと非表示
- Future/Hidden のシルエット表示は画像アセットを持たないのでテキスト + 疑問符アイコンで代替
