# SPEC-003: 遊びの描写と結果統合（コアループ）

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-003 |
| 機能名 | 遊びの描写・結果反映・時間経過の統合画面 |
| 対応ファイル | `prototype/game.js` の `startPlay`, `finalizePlay`, `renderResultPanel`, `renderResultDiscoveryCard`, `replayPlay`, `advanceClock` |
| 関連仕様 | SPEC-002, SPEC-004, SPEC-005, SPEC-007, SPEC-019, SPEC-020, SPEC-021 |
| ステータス | Active |
| 最終更新 | 2026-04-18 |

## 1. 目的
「遊び終了」「パラメーター反映」「時間経過」の3画面を **1画面に統合** し、タップ数を削減する。  
旧フロー：起床→遊ぶ→描写→イベント→**終了→反映→時間経過**→遊ぶ …（5タップ）  
新フロー：起床→遊ぶ→**描写（演出＋結果＋時間経過を含む）**→遊ぶ …（2タップ）

さらに、結果フェーズに **「もう一度遊ぶ（-◯h）」** アクションを追加し、**同じ遊びのリピート** を1タップで行えるようにする。

## 2. 用語定義
- **描写フェーズ**：選んだ遊びの状況文とプログレスバーを見せる期間（約2.2秒）。
- **結果フェーズ**：描写フェーズに続く同一画面内で、獲得経験値・Lvアップ・ステータス差分・時間経過を表示する期間。
- **ランダムイベント挿入**：確率で、描写フェーズの後に SPEC-004 の別画面が挟まる。
- **もう一度遊ぶ（Replay）**：結果フェーズのフッターから同じ遊びを再実行する導線。S2 を経由しない。
- **発見カード**：チュートリアル発見や新しい遊び解禁を、追加タップ不要で結果画面上部に表示するカード。

## 3. 入力（Input）
- 選択された遊び（SPEC-002 の結果）
- 現在のプレイヤー状態（SPEC-001）
- 友人数（SPEC-007）
- 体力上限（SPEC-019）

## 4. 出力（Output）
- 画面内に順次表示される要素：
  1. 描写文（遊びのフレーバー）
  2. プログレスバー
  3. 獲得原体験 / Lvアップ
  4. ステータス差分（体力・所持金・友人数・ジョウネツ）
  5. 時間経過（時刻 before → after、余剰時間の減少）
  6. **各種ゲージ**（SPEC-021）：体力ゲージ、経験値Lvゲージ、余剰時間ゲージ
- プレイヤー状態の永続的な変更。
- 必要に応じて **強制睡眠 / 強制終了** の発動（SPEC-019）。

## 5. 挙動（Behavior）

### 5.1 フェーズ遷移
```
[描写フェーズ] 2.2 秒
   │ プログレスバー 0→100%
   │ 描写文を1つ表示
   ▼
[イベント判定] 確率 35%
   │ Hit → S4 ランダムイベント画面（SPEC-004）へ遷移
   │ Miss → そのまま結果フェーズへ
   ▼
[結果フェーズ] 同一画面で下部にフェードインで表示
   │ ・獲得原体験 +○/+○（SPEC-005）
   │ ・Lvアップ演出（該当時、SPEC-021 ゲージ演出）
   │ ・ジョウネツ +○
   │ ・時計 06:30 → 07:30
   │ ・余剰時間 6h → 5h
   │ ・ステータス差分（体力/所持金/友人数） + 体力ゲージ（SPEC-021）
   ▼
[体力ゼロ判定 SPEC-019]
   │ 1〜12歳 体力=0 → 強制睡眠 2h → 結果フェーズ継続 or 就寝遷移
   │ 13歳以上 体力=0 → 強制終了 → 即 S5 就寝
   ▼
[次アクション：フッターボタン]
   │ ① もう一度遊ぶ（-◯h）← 再実行コスト（時間/体力/所持金）を満たしていれば有効
   │ ② 次の遊びを選ぶ    ← 余剰時間 > 0 なら有効
   │ ③ 今日おわる        ← いつでも可
   │
   │ 余剰時間 == 0 → 自動的に S5 就寝画面へ遷移（上記ボタンは使われない）
```

### 5.2 獲得経験値の計算
1. 基本値：`play.gain`（5カテゴリ）。
2. **友人数ボーナス（SPEC-007）**：`play.friendBonusPerPerson` がある場合、主カテゴリに `friendBonusPerPerson × 現在の友人数` を加算。
3. **没頭ボーナス（ジョウネツ）**：直前の遊びと主カテゴリが同じなら、ジョウネツ獲得量に連続回数を加算（`3 + 連続数`）。
   - **リピート（§5.7）時は連続数がそのまま維持されるため、没頭ボーナスが積み上がる。**
4. **熟練ブースト（SPEC-024 §5.4）**：遊びのカテゴリ（SPEC-022）で所持スキルLv の平均に応じた係数 `skillBoost ∈ [1.0, 2.0]` を掛ける。`physical/creative/…` の 5 カテゴリ値、友人数ボーナス、スキル加算量すべてに適用（没頭ボーナスには適用しない）。
5. **低体力プレイ（SPEC-002 §5.9）**：`stamina < staminaCost` で遊びを始めた場合、`lowStaminaMultiplier = stamina / staminaCost`（0〜1）を全経験値系（原体験・友人ボーナス・ジョウネツ・スキル）に掛ける。
6. **スキル経験値加算（SPEC-024 §5.3）**：遊びの `categories` ごとに、原体験5カテゴリ合計を `categories.length` で割って熟練ブースト＆低体力倍率を乗じ、`player.skills[cat].exp` に加算。

### 5.3 時間経過
- 時刻が `play.timeCost` 時間だけ進む（例：1.5h なら 1h30m）。
- 余剰時間も `play.timeCost` 分減少。

### 5.4 ステータス消費
- `player.stamina -= play.staminaCost`（下限0でクランプ）
- `player.money -= play.moneyCost`

### 5.5 ランダムイベント発火（SPEC-004）
- 描写フェーズ後、乱数 < 0.35 で S4 に遷移。
- S4 の「なるほど」押下で、結果フェーズに戻って合算表示する。

### 5.6 体力ゼロ時の挙動（SPEC-019 連携）
- 結果フェーズの値反映後、`player.stamina` が 0 になっていればライフステージ別の分岐を実行：
  - **1〜12歳**：強制睡眠 2h → 回復量 `floor(staminaCap × 0.30)` → トースト表示 → 余剰時間 -2h → 余剰ゼロなら自動で S5
  - **13歳以上**：強制終了 → 翌日 spareHours=0 フラグを立てて S5 自動遷移
  - ゲージ（SPEC-021）：体力は 0% の赤色で固定表示される

### 5.7 「もう一度遊ぶ（-◯h）」アクション
- **ボタンラベル**：`🔁 もう一度遊ぶ（-${play.timeCost}h）`
- **有効条件**（SPEC-002 と同じルールで再評価）：
  - `player.spareHours >= play.timeCost`
  - `player.stamina >= play.staminaCost`
  - `player.money >= play.moneyCost`
  - `play.seasons` / `ageMin/Max` / `minFriends` すべてOK
- **表示条件**（v2、追加）：
  - 上記の有効条件に加え、**次のアクションが「遊び選択」である** 場合のみ表示
  - すなわち以下の文脈では **非表示**：
    - コアタイム時間に達している（`now >= coreTime.startHour && !coreTimeDoneToday`）→ 次のアクションは **「🏫 保育園に行く」** 一択
    - その日の余剰時間が 0 になっている → 次のアクションは **「🌙 おやすみ」** 一択
    - 余剰時間は残っているが、リピートコスト（時間/体力/所持金）を満たさない → リピート disabled、次遊び選択を表示
- **挙動**：押下すると S2 を経由せず、**同じ遊びで `startPlay(play)` を再実行**。描写フェーズから再開。
- `player.consecutiveCategoryCount` / `lastPlayCategory` はリセットしない → 没頭ボーナスが積み上がる。

### 5.8 結果フェーズのフッター構成（v2：文脈別）

結果フェーズのフッターは **現在の文脈に応じて表示ボタンを変える**。
プレイヤーに提示するアクションが実質 1 択の場合は、ボタンも 1 つだけに絞る。

| 文脈 | 表示するボタン |
|---|---|
| **通常**（遊び選択画面に戻れる／リピート可） | `🔁 もう一度遊ぶ (-Xh)` ＋ `次の遊びを選ぶ` ＋ `今日おわる`（ghost） |
| **コアタイムに到達**（朝の遊び中に 09:00 等に達した） | **`🏫 {ステージ名}に行く` の 1 ボタンだけ** |
| **余剰時間が 0**（夜の遊び後など） | **`🌙 おやすみ` の 1 ボタンだけ** |
| リピート不可（体力／所持金／時間不足）だが余剰ある | `次の遊びを選ぶ` ＋ `今日おわる`（ghost） |

```
文脈 = 通常
┌──────────────────────────────────────────┐
│ [ 🔁 もう一度遊ぶ (-1h) ] [次の遊びを選ぶ]│
│ [ 今日おわる ]                           │
└──────────────────────────────────────────┘

文脈 = コアタイム到達
┌──────────────────────────────────────────┐
│ [ 🏫 保育園に行く ]                       │
└──────────────────────────────────────────┘

文脈 = 余剰時間 0
┌──────────────────────────────────────────┐
│ [ 🌙 おやすみ ]                           │
└──────────────────────────────────────────┘
```

### 5.8a スキップボタンの表示と連打防止（v2 追加）
- スキップボタンは **描写フェーズ中のみ** 表示される。
- **結果フェーズでは非表示**（スキップは演出をスキップするためのもので、結果確認や経験値反映をスキップさせる意図ではない）。
- **連打防止**：
  - 1回クリックされたら `skipArmed = false` にする防御フラグを立て、以降のクリックを無視する。
  - これにより `playRAF` のキャンセル → `finishDescription` → `finalizePlay` が **二重に実行されない**。
  - 二重実行されると **獲得経験値・時間経過・体力消費がすべて 2 回反映される**致命的なバグになるため、必ず実装する。
- DOM 側でも `btn-skip-play.disabled = true` に即座に切り替える（視認性＋二重クリック抑止の二重化）。

### 5.9 結果フェーズに表示するゲージ（SPEC-021）
- 体力ゲージ（現在値 / 上限）
- 主カテゴリの原体験Lvゲージ（Lvアップ時は満タン→リセット演出）
- **時計円盤**（before / after の2枚を並べて時刻の進みを視覚化。SPEC-021 §5.2）

### 5.10 結果フェーズに表示するスキル獲得（SPEC-024）
- 遊びに紐づく各カテゴリ（SPEC-022）のスキル経験値増分を 1 行ずつ表示
- **各スキルにもLvゲージを表示**（原体験Lv と同じ表現）
- 原体験 5 カテゴリとスキルの両方で、**差分ハイライト付きアニメ（SPEC-021 §5.4a）** を適用
  - 元の経験値までが通常色（オレンジ）
  - 今回増えた分が **緑色** で右方向に伸びるアニメーション
- Lvアップしたスキルには `⬆ Lv UP!` を併記、ゲージも 2 段階アニメで次Lvに繰り越し
- 体力不足プレイの場合は `× 倍率%` を末尾に併記

### 5.11 v4: 新しい遊びの発見表示
- 手動プレイで新しい遊びを発見した場合、従来の `#interrupt-overlay` モーダルは表示しない。
- `renderResultDiscoveryCard()` が、結果画面 `#result-panel` の最上部に `#result-discovery-card` を表示する。
- 表示内容はアイコン、タイトル、本文。複数件あれば縦に並べる。
- 目的は、発見を結果情報の一部として見せ、モーダルを閉じるための追加タップをなくすこと。
- 自動モード中の介入イベントは、従来どおりキュー表示を維持してよい。

- 画面：S3 遊びの描写（`#screen-playing`） … 描写 + 結果を 1画面で表示

## 7. 想定される利用シナリオ

### 7.1 砂場で集中プレイ
- 2歳・余剰4h・体力15。砂場遊び（-1h/-10）→ 結果フェーズで体力5→0…にはならないが、もう一度遊ぶ（-1h）で体力5。さらにもう一度→体力0→**強制睡眠 2h**で +4。

### 7.2 没頭ボーナスの積み上げ
- 粘土遊び連続 3回 → 創作系カテゴリで `consecutiveCategoryCount = 3` → 没頭ボーナスで `passion += 3 + 3 = 6`。

## 8. 例外・エッジケース
- スキップボタン：描写フェーズを即座にスキップ可能。結果フェーズの計算は同じ。
- 余剰時間が遊びの `timeCost` より短い場合：SPEC-002 でロックされるため発生しない。
- リピート中に所持金や時間が尽きた瞬間：ボタンが自動的に disabled に切り替わる。
