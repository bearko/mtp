# SPEC-024: スキル

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-024 |
| 機能名 | スキル（カテゴリ 1:1 のLv 1〜100 成長パラメーター） |
| 対応ファイル | `prototype/game.js` の `player.skills`, `gainSkillExp()`, `skillLvFromExp()`, `skillBoostMultiplier()` |
| 関連仕様 | SPEC-003, SPEC-005, SPEC-022, SPEC-023 |
| ステータス | Active（保育園分のみ実装） |
| 最終更新 | 2026-04-18 |

## 1. 目的

遊びを積み重ねた成果を **細やかに可視化** する。  
「同じ遊びを繰り返すことで要領を掴んで、もっと楽しめるようになる」というリアリティを体感させる。

既存の **5 カテゴリ原体験Lv**（SPEC-005）はプレイヤー全体の成長軸、  
**スキル**（SPEC-024）は カテゴリ（SPEC-022）ごとの **細かな熟練度** を表す別枠のパラメーター。

## 2. 用語定義

- **スキル（Skill）**：プレイヤーが所持する成長パラメーター。各カテゴリと 1:1 対応。
- **スキル経験値（Skill EXP）**：スキルごとの累積経験値。
- **スキルレベル（Skill Lv）**：1〜100。経験値から算出。
- **熟練ブースト**：スキルLv が高いほど、対応する遊びで得られる経験値（原体験／スキル両方）が上昇する係数。

## 3. 入力（Input）

- 遊びの完了（SPEC-003 `finalizePlay()`）
- 遊びの `categories`（SPEC-022）
- 現在のスキルLv（ブースト計算のため）

## 4. 出力（Output）

- `player.skills[categoryId] += gainedExp`
- スキルLv の更新
- S3 結果画面にスキル獲得行を表示
- 以降の `finalizePlay()` 計算で熟練ブースト係数を適用

## 5. 挙動（Behavior）

### 5.1 スキル一覧
- スキルのID／ラベルは **カテゴリと完全一致**（SPEC-022 §5.1）。
- 初期値：全スキル `exp = 0`, `Lv = 1`（プロト段階では未習得でも Lv1 とみなし、経験値累積で 2 以上に成長）

### 5.2 スキルレベル算出式

原体験Lv（SPEC-005）と同じ曲線を使うが、**上限は 100**：
```
skillLvFromExp(exp) = min(100, max(1, floor(sqrt(exp / 10)) + 1))
```
- exp=0 → Lv1
- exp=10 → Lv2（Lv1 区間は 10 exp で卒業）
- exp=40 → Lv3
- exp=90 → Lv4
- exp=98010 → Lv99
- exp=100000 → Lv100（理論上の最大、プロトでは到達困難）

### 5.3 遊び実行時のスキル経験値獲得

```
function awardSkills(play, baseMultiplier):
  for cat in play.categories:
    // 遊びの「主カテゴリ寄与」を (play 全体の gain 合計) × (1 / カテゴリ数) で近似
    baseExp = totalExpOf(play)  // play.gain の合計（SPEC-005 の経験値合計）
    expForCategory = round(baseExp / play.categories.length * baseMultiplier)
    player.skills[cat] = (player.skills[cat] || 0) + expForCategory
```

- プロト段階では **`baseExp = play.gain の5カテゴリ合計`**、`baseMultiplier = 1` で扱う。
- 体力不足で遊ぶ場合（SPEC-002 §5.9 低体力プレイ）は `baseMultiplier < 1` の形で配分される。

### 5.4 熟練ブースト

スキルLv が高いほど、次にその遊びをしたときの獲得経験値が上がる。

```
skillBoostMultiplier(play) =
  1 + (average(player.skills[cat].lv for cat in play.categories) − 1) × 0.02
  // 上限 2.0（= Lv51 で 2 倍）
```

例：
- 全 Lv1（初プレイ）：係数 1.00
- 全 Lv5 平均：係数 1.08
- 全 Lv26 平均：係数 1.50
- 全 Lv51 以上：係数 2.00（上限クランプ）

#### 5.4.1 ブーストの適用先

`finalizePlay()` 内の計算に以下を反映：

| 加算対象 | ブースト適用 |
|---|---|
| 原体験経験値（5カテゴリ）(SPEC-005) | ✅ あり |
| 友人数ボーナス (SPEC-007 §5.2) | ✅ あり（主カテゴリに乗る時点で一緒に係数倍） |
| ジョウネツ（没頭ボーナス）(SPEC-003 §5.2) | ❌ なし |
| スキル経験値（本仕様 §5.3） | ✅ あり（自己強化ループ） |
| イベント効果 (SPEC-004) | ❌ なし |

### 5.5 スキル成長の速度感（目安）

- 保育園レベルの遊び 1 回で `gain 合計 ≒ 15` exp → カテゴリ数 3 → 1 カテゴリに +5 exp
- 10 回繰り返すと +50 exp → Lv3 相当
- Lv10 到達は同じ遊びを 100 回近く繰り返したのと同等（プロトでは現実的でないので、将来的にはカテゴリの共通加算で素早く伸びる設計）

### 5.6 スキルとカテゴリ指標の併存

- 従来の **5 カテゴリ原体験Lv**（SPEC-005）は引き続き有効。**ライフステージ／人生全体の成長** を表す。
- **スキル**（本仕様）は **遊びの要素別熟練度** を表す。
- 両者は用途が異なるため **別枠で管理**。混ぜてはいけない。

### 5.7 データ構造

```
player.skills = {
  movement:      { exp: 0, lv: 1 },
  outdoor_toy:   { exp: 0, lv: 1 },
  ...
}
```

- 未取得のカテゴリでも「その遊びで初めて触れた瞬間に」作られる（遅延初期化）。

## 6. UIへの反映

- S2 遊び選択プレビュー：右上に「現在の関連スキルLv バッジ」を表示（例 「遊具Lv3 / 運動Lv2」）
- S3 結果画面：原体験Lv表示の次に「獲得スキル」カードを追加。スキル名＋経験値増分＋Lvアップ有無
- S7 遊びツリー（SPEC-023）：遊びのノードに「推奨スキル」を表示し、所持スキルLv が満たしていれば印を付ける
- HUD には載せない（情報過多になるため）

## 7. 想定される利用シナリオ

### シナリオA：砂場を何度も遊ぶ
- カテゴリ `outdoor_toy`, `crafting`, `nature` のスキル経験値が均等に貯まる
- 3 回目以降は `outdoor_toy` スキルLv が上がって熟練ブーストが効き、原体験経験値が少し増える

### シナリオB：違う遊びだが同じカテゴリ
- 砂場（`outdoor_toy`）→ 滑り台（`outdoor_toy`）→ 大きな滑り台（`outdoor_toy`）と繰り返すと、  
  `outdoor_toy` スキルが突出して伸び、以降の遊具系遊びすべてで熟練ブーストが効く

### シナリオC：スキル 1 と原体験Lv 1 の違い
- `outdoor_toy` スキルLv 1 → 1 体験以上の共通キュー
- `physical` 原体験Lv 1 → 「身体系」の総体験量（砂場でも鬼ごっこでもLv上昇する）

## 8. 例外・エッジケース

- ブースト係数が上限（2.0）に達しているカテゴリは、それ以上の加速はしない
- 同一遊びを連続プレイしてもスキル経験値は通常通り加算（没頭ボーナスはジョウネツのみに作用）
- スキルLv は 100 到達時点で以降の経験値加算は「溢れ値」として累積するだけでLv変動しない
