## 1. 目的

「スキップや自動モードでタップ数は減るが、**作業感が出て楽しくない**」というフィードバックを起点に、ゲームデザイン全体を **プランナー視点** で診断し、改善案を提示する。

本文書は「実装する SPEC」ではなく「**次に何の SPEC を起こすべきかを決める叩き台**」である。各提案に **期待効果・リスク・実装コスト** を示し、採否の判断材料とする。

## 2. 診断：なぜ作業感が出るのか

### 2.1 現状のプレイヤー体験マップ

プレイヤーの 1 日は（SPEC-020 固定睡眠サイクルから）：

```
[起床 07:00]
  │
  ├─ [S2 遊び選択]（ドックで遊びをタップ）
  │    │
  │    ├─ [S3 遊び中]（2 秒の演出）
  │    └─ [S3 結果]（素養/スキルの差分表示）
  │     └─ [S2 に戻る]
  │
  │ ... 余剰時間が尽きるまで繰り返す
  │
  ├─ [S6 コアタイム / 保育園]（自動消化）
  │
  ├─ [S2 夕方の遊び]（朝と同じループ）
  │
  └─ [S10 日サマリ（連絡帳）]
[就寝 19:00]
```

**手動モード**：毎日 3〜5 回この S2 → S3 → S2 のループを回す。
**自動モード**：プレイヤーが何もしなくても勝手に遊びが選ばれて進む。**見る要素しかない**。
**スキップ（週末まで／明日の夜まで）**：S10 を飛ばして日を一気に進める。**見ることすら飛ばす**。

### 2.2 「作業感」の正体

プレイヤーがタップ数削減系機能（自動 / スキップ）を使うと、以下が失われる：

| 失われるもの | 本来の役割 |
|---|---|
| **選択の緊張感** | 今日は何して遊ぶか決める瞬間 |
| **不確実性への期待** | 何が起きるかワクワクする |
| **結果への驚き** | A ランクが出るとか、思わぬ発見とか |
| **成長の実感** | 「今日はこれだけ伸びた」の主観的な達成 |

自動化／スキップはタップ数を減らすが、**減らしたタップの分、得られる体験も薄まる**。  
しかも現状：
- 自動モードは **プレイヤーの意志が反映されない一本道**（情熱プロファイルで傾向指定だけ）
- スキップは **見ないで数字だけ進む**
- 素養値・グレードが変動しても、**視覚的な見どころ（surprise）が薄い**

結果、自動モード／スキップで「数値稼ぎ」の作業感だけが残る。

### 2.3 根本的な問題：なぜ毎日同じことを繰り返させているのか？

現在のコアループは **S2 → S3 → S2 の 2 画面の往復** が支配的。保育園期だけでも Day 1-1440 の 4 年間を回すので、1000 回以上の同じ往復が発生する。

人生ゲームは「本質的に繰り返し」が避けられない。しかし **同じ繰り返しでも「同じに見えない」演出** が足りていない：
- 季節・場所の変化（SPEC-047 で場所は導入されたが、遊び体験自体は同じ）
- 遊びの中で **事件・選択肢・偶発性** が少ない
- NPC・友達との **関わりの変化** が薄い（SPEC-041 も未実装）
- 長期的な **目標** がプレイヤーに見えない

## 3. 「楽しい」の分類：どの感情を設計したいか

### 3.1 ゲームで経験しうる 8 つの快感（Marc LeBlanc「8 Kinds of Fun」を参考）

1. **Sensation**（感覚）：視覚・音・触感の心地よさ
2. **Fantasy**（空想）：物語世界に没入
3. **Narrative**（物語）：先を知りたい感覚
4. **Challenge**（挑戦）：難しいことを克服
5. **Fellowship**（仲間）：他者との交流
6. **Discovery**（発見）：未知に触れる
7. **Expression**（表現）：自分のスタイルを作る
8. **Submission**（娯楽化）：無思考で浸る

### 3.2 本作の現状配分（プランナー主観）

| 快感 | 現状の強度 | 現状の演出 |
|---|---|---|
| Sensation | ★★☆☆☆ | 絵文字・ドット絵・簡易アニメ |
| Fantasy | ★★★☆☆ | 転生導入・連絡帳の世界観 |
| Narrative | ★☆☆☆☆ | **ほぼ無い。1日単位で閉じる** |
| Challenge | ★☆☆☆☆ | 低体力プレイ程度、明確な挑戦が無い |
| Fellowship | ★☆☆☆☆ | 友人数だけ、個別 NPC 未実装 |
| Discovery | ★★★☆☆ | 発見プール、遊び解禁、場所拡大 |
| Expression | ★★☆☆☆ | 情熱プロファイル選択くらい |
| Submission | ★★★★☆ | 自動モードで放置できる |

### 3.3 問題点と方向性

- **Submission（放置）だけが強すぎる** → だから作業感になる
- **Narrative（物語）と Challenge（挑戦）が弱い** → 選ぶ動機が弱い
- **Fellowship（仲間）と Expression（表現）の伸び代が大きい** → NPC・プロファイル進化で回収可能

## 4. プレイヤーの感情曲線（再整理）

SPEC-040 §8 でライフステージごとの感情曲線を設計目標として定義済み：

```
  強い没頭 ─▲ 好奇心 ─▲ 達成感 ─▲ 選択の重み ▲ 充実感 ▲ 懐かしさ
保育園    幼稚園   小学校   中学校    高校    大学 社会人   老後
```

**問題**：現在の実装は「保育園が 4 年」あり、この「強い没頭」期が長すぎて飽きる。本来「強い没頭」は 3-6 ヶ月で切り替わるべき。

## 5. 改善提案（アイデア集）

以下、短期 / 中期 / 長期 に分けて提示。**すべて採用する必要はない**。プランナーで採否を判断する。

### 5.1 【短期】✨ 小さな surprise を増やす

**問題**：毎日同じ遊びを選ぶと、S3 の結果画面も毎回ほぼ同じ。

**提案**：
- **遊び中の「小イベント」**：遊び実行中に 20% の確率で `flavorEvent` が発動。例：「絵本で新しい言葉を覚えた（intellect +5）」「砂場で友達と協力できた（social +5）」
- **ランダムフレーバーテキスト**：同じ遊びでも S3 の descriptions を 10 種類に増やす
- **天気・気分システム**：その日の天気（晴/曇/雨/雪）や気分（ウキウキ/穏やか）で素養倍率が ±10% 変動、フレーバーも変わる
- **クリティカル表現**：10% で「特別な一日！」演出、素養 +50%

**メリット**：実装軽、既存ループを壊さない、毎日が少し違って見える
**リスク**：乱数依存なので「意味」を感じにくい可能性

### 5.2 【短期】🎯 デイリーミッション / ウィークリーミッション

**問題**：毎日何を目指すのかが曖昧。

**提案**：
- 毎朝ランダムで「今日のチャレンジ」を 1 つ提示
  - 例：「今日は 3 回以上絵本を読もう」「大きな公園に行ってみよう」「新しい遊びを 1 つ発見しよう」
- 達成時に **素養ボーナス + 連絡帳に特別コメント**
- 週次チャレンジ：「今週は 15 回遊ぼう」「3 箇所以上の場所に行こう」
- **達成率の可視化**：HUD に小さくプログレス表示

**メリット**：1 日ごとに目的ができる、Challenge の快感が上がる
**リスク**：ミッションが単調だと逆に作業化

### 5.3 【短期】💡 アンロックのテンポを上げる

**問題**：保育園 4 年間で遊びのバリエーションが少ない状態が続く。発見頻度も高くない。

**提案**：
- 毎週 1 つ以上は **新しい遊び** が解禁されるように発見プールを拡充
- 解禁イベントを **セレモニー化**：「新しい遊びを覚えた」モーダルを盛大に（紙吹雪・効果音）
- 3 つ以上解禁で「遊び図鑑」が開く → NPC 風にコレクション要素
- 未解禁遊びのシルエット表示（SPEC-023 既にある）をもっと目立たせ、ワクワク感を演出

**メリット**：Discovery の快感が最大化
**リスク**：コンテンツ追加が重い

### 5.4 【中期】🎭 NPC との関係を「物語」に

**問題**：現状 `player.friends` は整数。誰と仲良いのか、どう変化しているのか見えない。

**提案**（SPEC-041 の実装前倒し）：
- 保育園から親友候補 2-3 人固定配置（ユウタ、マヤ、タケル 等）
- その日の遊びで「今日は誰と遊んだ」を記録
- 関係値が一定を超えると **小さなエピソード** 発生
  - 「ユウタと一緒に大きな滑り台に挑戦してみたけど怖くて泣いた」
  - 「マヤがお気に入りの絵本を貸してくれた」
- 連絡帳（SPEC-027）に NPC 名が入って **人格化された日記** として読める
- 将来ステージに NPC を引き継ぐ（幼稚園・小学校でも同じ子が成長して再登場）

**メリット**：Fellowship と Narrative が一気に強化、「物語としての人生」になる
**リスク**：エピソードコンテンツが大量に必要

### 5.5 【中期】🧭 長期目標の可視化（ライフ・アンビション）

**問題**：ゲーム全体の目的が明示されない。「100 年過ごす」以外に何を目指す？

**提案**：
- 転生直後に **「どんな人生を歩みたい？」** の選択肢を提示
  - 芸術家：sensitivity 中心
  - 学者：intellect 中心
  - スポーツマン：body 中心
  - リーダー：social 中心
  - 情熱家：passion 中心
- 選んだ目標に対して **素養の目標値** が設定される（例：sensitivity を A に）
- HUD or プロファイル画面で **目標に対する進捗バー** を常時表示
- 達成するとエンディング分岐（特別な称号・エピローグ）
- **年齢ごとに中間目標**：「5 歳までに〇〇」のようなマイルストーン

**メリット**：Expression と Challenge が一気に強化、プレイに方向性ができる
**リスク**：自由度が下がる可能性（目標に縛られる）、選択肢を慎重に設計すべき

### 5.6 【中期】🎲 「選択」を増やす：分岐イベント

**問題**：現状の選択は「どの遊びをするか」の 1 種類だけ。

**提案**：遊び中や日常的な場面で **分岐選択モーダル** を発生させる：
- 公園で：「知らない子が転んで泣いている。どうする？」
  - 助ける（social +10, passion +5）
  - ママを呼ぶ（intellect +3, social +5）
  - 見なかったことにする（passion -3）
- 保育園で：「今日の発表会で主役を頼まれた」
  - 引き受ける（passion +15, sensitivity +20）
  - 他の子に譲る（social +10）
- 各選択に **短期効果 + 長期影響** を持たせる（記録は SPEC-041 の history に）

**メリット**：Expression と Challenge、Narrative 全部が上がる
**リスク**：分岐ブランチの組み合わせ管理が複雑、コンテンツ量が必要

### 5.7 【中期】🎬 自動モードの「ダイジェスト化」

**問題**：自動モード中、プレイヤーは **何が起きたか覚えていない**（ただ数値だけが変化する）。

**提案**：自動モードを「完全自動」ではなく「**要所だけ見せるダイジェスト**」に変える。
- 自動ループ中の遊びは **ダイジェストカード** に変換（1 秒で「絵本 ×3・公園 ×2」と絵文字で見せる）
- ただし **ハイライト機会**（グレードアップ、新発見、ミニイベント）だけは **自動で手動モーダル** を挟む
  - Lv アップ：モーダルで「読書 Lv 5 になった！」
  - 新発見：「大きな滑り台を発見した！」
  - 分岐イベント：プレイヤー選択を促す
- **自動モードでも「重要な瞬間」は必ずプレイヤーの目と手を介する**

**メリット**：自動モードが「放置」ではなく「要所だけ関わる」プレイになる、物語を取りこぼさない
**リスク**：介入頻度の調整が難しい、頻度が高すぎると逆に煩わしい

### 5.8 【中期】🎵 遊び自体を「体験」に：ミニゲーム化

**問題**：S3 の 2 秒間アニメはあってないようなもの。

**提案**：一部の遊びに **簡易ミニゲーム** を組み込む。
- 絵本：「これから読む絵本を 3 択から選ぶ」→ 選んだ絵本で獲得素養が微妙に違う
- 滑り台：「滑るコースを選ぶ」（早い / 曲がる / 逆走）→ 結果にバリエーション
- 砂場：「何を作る？」（山 / お城 / トンネル）→ カテゴリ毎に異なる素養
- お歌：「4 曲から選んで歌う」→ 歌うたびに累積で「持ち歌」記録

**メリット**：Sensation と Expression が上がる、**同じ遊びでも「自分なりの」遊び方**が作れる
**リスク**：実装コスト大

### 5.9 【長期】🌱 「あなたの人生」の記録機能

**問題**：長く遊ぶほど「自分が何をしてきたか」が不明瞭になる。

**提案**：
- **人生年表**（タイムライン画面）：主要イベント・出会った人・発見したことを時系列で表示
- **思い出の瞬間**：ゲーム中に手動で「この瞬間をブックマーク」できる機能。老後の回想モードでランダムに再生
- **連絡帳アーカイブ**：過去の連絡帳（SPEC-027）をすべて閲覧可能
- **スクリーンショット共有**（将来）：好きな瞬間を画像化してシェア

**メリット**：Narrative と Fantasy が長期的に強化、ゲームに「重さ」が生まれる
**リスク**：実装コスト大、プレイ時間が長くなりすぎる可能性

### 5.10 【長期】🎪 ライフステージ別のテーマ特化

**問題**：保育園 4 年間が同じ遊びの繰り返しで長い。

**提案**：**ライフステージを「フェーズ」にさらに分ける**：
- 保育園 **前期**（1-2 歳）：感覚遊び中心、2 歳で「言葉」が開く
- 保育園 **中期**（3 歳）：社会性が開花、友達が中心
- 保育園 **後期**（4 歳）：自我の形成、「なぜ？」期
- 各フェーズで **新しい遊びのカテゴリ** が大量に解禁
- フェーズ切り替え時に **盛大なお祝いモーダル** で節目を演出

**メリット**：同じステージ内でも変化が続く、飽きにくい
**リスク**：SPEC-011 などの設計変更が必要

### 5.11 【長期】🎨 プレイヤーが「遊びを作る」

**問題**：用意された遊びを選ぶだけ。

**提案**：高度な仕組みとして、プレイヤーが **独自の遊びを編集・作成できる** システム。
- 既存の遊びを組み合わせて「絵本を読みながら散歩」のような複合遊びを作る
- 名前を付けて、ドックに登録
- 他の子（NPC）に教える → その NPC が連絡帳で言及
- SNS 的に将来的に他プレイヤーとシェア

**メリット**：Expression が最大化、唯一無二の体験
**リスク**：実装超重、バランス調整困難

## 6. 個人的おすすめ（プランナー視点の優先度）

以下、筆者推奨の **採用優先度**：

### Tier S（最優先、次 PR 候補）
- **5.1 小さな surprise を増やす**：最小コストで最大効果。フレーバー 10 種化、天気システム、クリティカル演出。これだけで毎日が違って見える
- **5.7 自動モードのダイジェスト化**：自動 = 放置ではなく「要所だけ関わる」に変える。現状の自動モードの致命的な作業感を解消する
- **5.4 NPC との関係を物語に**（SPEC-041 の実装前倒し）：保育園から親友 3 人固定、連絡帳に名前が入るだけで感情的重みが変わる

### Tier A（中期、S の反応を見て着手）
- **5.6 分岐イベント**：選択肢の質・量を増やし「人生ゲーム感」を取り戻す
- **5.5 長期目標の可視化**：人生全体の方向性を可視化
- **5.2 デイリーミッション**：短期の目的を作る

### Tier B（長期、世界観・コンテンツが充実してから）
- **5.9 人生年表・記録機能**
- **5.10 フェーズ細分化**：保育園前期/中期/後期
- **5.3 アンロックテンポ向上**：コンテンツ追加が前提
- **5.8 ミニゲーム化**

### Tier C（実験的、将来検討）
- **5.11 プレイヤーが遊びを作る**

## 9. プランナー・デベロッパー双方への問い

本文書の結論として、プランナー側に以下を確認したい：
