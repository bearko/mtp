# SPEC-006: 生活リズム

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-006 |

## 1. 目的
「夜更かし誘惑 vs 翌日の体調リスク」というジレンマを作る。  
短期的な余剰時間の獲得（夜更かし +1h）と、中長期の体調悪化・寝坊リスクを天秤にかけさせる。

## 2. 用語定義
- **生活リズム（bioRhythm）**：0〜100の数値。睡眠時刻の規則性を表す。
- **寝坊**：リズムが低い状態で発生し、起床時刻が遅れる。
- **体調不良**：翌朝の体力が減少し、余剰時間も減少する状態。

## 3. 入力（Input）
- 就寝時に選ばれた睡眠モード（`early` / `normal` / `latenight`）。
- 現在の生活リズム。

## 4. 出力（Output）
- 生活リズムの増減。
- 翌朝の起床時刻、余剰時間、体調不良フラグ。

## 5. 挙動（Behavior）

### 5.1 就寝モードによる生活リズム変化
| モード | 就寝時刻 | リズム変化 | 体力変化 |
|---|---|---|---|
| early（早寝） | 21:00 | +10 | +25 |
| normal（普通） | 23:00 | ±0 | +15 |
| latenight（夜更かし） | 25:00 | -15 | ±0（その日+10のボーナスと翌日-10で相殺） |

### 5.2 起床時刻の決定（翌朝）
| 翌朝のリズム | 起床時刻 |
|---|---|
| 70 以上 | 06:30 |
| 40〜69 | 07:30 |
| 40 未満 | 09:00（**寝坊**） |

### 5.3 翌日の余剰時間の計算
```
base = 6h
if sleepMode == "latenight": base += 1   // 深夜遊び枠
if wake >= 9:                base -= 2   // 寝坊ペナルティ
if sickness:                 base -= 2   // 体調不良
if stamina < 30:             base -= 1   // 低体力
spareHours = max(0, base)
```

### 5.4 体調不良の発生条件
- `sleepMode == "latenight"` かつ `リズム < 60` の状態で、翌朝 30% で発動。
- 発動時：体力 -15、余剰時間 -2h、HUDに「朝から体調がすぐれない…」のトースト表示。

## 6. UIへの反映
- 起床画面（S1）にメッセージで反映（「寝坊した…」「スッキリだ！」等）。
- 就寝画面（S5）で選択肢と効果を表示。
- HUDに生活リズム（🌙 アイコン）を常時表示。

## 7. 想定される利用シナリオ
- リズム80の状態で早寝 → 翌朝リズム90で6:30起床、体力フル、余剰時間6h。
- リズム50の状態で夜更かし → 翌朝リズム35で9:00起床、30%で体調不良、余剰時間が2〜3hに。

## 8. 例外・エッジケース
- 連続夜更かし（3日以上）でリズムが大きく下がり続けるが、下限は0でクランプ。
- 将来的にはライフステージによる減衰感度の変更を検討。
