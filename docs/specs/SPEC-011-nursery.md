## 1. 目的

1〜3歳のプレイヤーは、平日 **09:00〜16:00** を保育園で過ごす。  
保育園では「遊びの基礎」を学び、公園の遊具や室内遊具の遊び方を覚えることで、帰宅後の余剰時間に使える **新しい遊びを解禁** する。

コンセプト：
- 時間を拘束されるが、**そこで得る発見が後の遊びを豊かにする**。
- 保育園に行くこと自体が原体験ポイントとなり、一定確率で発見が起こる。

連携する他の仕様：
- **SPEC-019**：年齢別体力上限（1歳=10, 2歳=15, 3歳=30, 4歳=50）、体力ゼロ時の強制睡眠。
- **SPEC-020**：固定起床07:00／就寝19:00、朝の遊び時間（07:00〜09:00）、夜の遊び時間（16:00〜18:00）。
- **SPEC-021**：体力ゲージ（起動時は満タン表示）、**時計円盤**（24h を 1周、現在時刻までを扇形で塗りつぶし）。
- **SPEC-022**：保育園の各遊びに `categories` を付与（砂場=遊具/工作/自然 など）。
- **SPEC-023**：保育園ステージ全遊びを S7 遊びツリーに公開。幼稚園以降はシルエットで予告。
- **SPEC-024**：各カテゴリに対応するスキルが 1:1 で成長する。
- **SPEC-026**：**Day 1〜7 は保育園を休園** し、昼寝枠（13:00〜15:00）以外を自由遊びにするチュートリアル仕様。
- **SPEC-027**：日の終わりサマリ（S10）を **連絡帳形式**（先生コメント＋家庭コメント）で演出。
- **SPEC-028**：遊び・イベントマスタは `data/*.json` で外部管理。

## 2. 用語定義

- **保育園コアタイム**：09:00〜16:00 の時間帯。
- **生活活動時間枠**：13:00〜15:00 の時間帯で、日替わりで **室内遊び** または **公園遊び** が行われる。
- **発見（Discovery）**：保育園での学びや経験で得られるもの。
  - 新しい遊びの解禁
  - 基礎パラメーター（身体系／社会系）の加算
  - 友人数の増加

## 3. 入力（Input）

- プレイヤー状態（SPEC-001）：年齢が 1〜3 であること。
- 日付（`player.day`）：日替わりで「室内 / 公園」を決めるために使用。

## 4. 出力（Output）

- 時刻の進行：`clockHour = 16`（コアタイム終了時刻）。
- 原体験・パラメーターの加算（下記 §5.3）。
- 友人数の増加（確率）。
- 新しい遊びの解禁通知（該当時）。
- 画面：S6 コアタイム画面（保育園仕様）の描画。

## 5. 挙動（Behavior）

### 5.1 発動条件
- プレイヤーの年齢が 1〜4 歳（※SPEC-019 テーブルで 4歳を保育園に含めて扱う。ただし発見内容・セリフは未就学児共通とする）。
- 起床直後、S1 → S2 に進もうとするタイミングではなく、**朝の遊び時間終了後、09:00 到達時に自動発火**。
- 朝の遊び時間（07:00〜09:00）は自由に遊べる（SPEC-020）。時刻が 09:00 に達した時点で S6 コアタイム画面に強制遷移。

### 5.1.1 遷移タイミング詳細（v2：起床画面統合）
1. 07:00 固定起床（SPEC-020）。**朝の遊び選択画面（S2）に直接出現**。画面上部に起床ヘッダー（時計円盤・体力ゲージ・挨拶）が表示される（SPEC-002 §1.1）。余剰時間 = 朝2h + 夜2h = **4h**（実運用は朝2h→コアタイム→夜2hに分けて消費）
2. 朝の遊びで時計が 09:00 に達した瞬間（または朝の余剰 2h を使い切った瞬間）、S3 結果フェーズのフッターが **「🏫 保育園に行く」1 ボタン** に切り替わる（SPEC-003 §5.8）。押下で S6 保育園コアタイムへ遷移
3. コアタイム終了 16:00 で S2（夜の遊び）へ戻る。この時の起床ヘッダーは圧縮表示（SPEC-002 §5.2.1）
4. 夜の遊びで 18:00 到達 または余剰0で、S3 結果フェーズが **「🌙 おやすみ」1 ボタン** に切り替わる
5. 「おやすみ」押下で S5 就寝画面へ。1〜9歳はモード選択なしの「おやすみ」1 ボタンのみ（SPEC-020 §5.5.1）
6. 19:00 就寝 → 翌 07:00 起床、S2 朝の遊び選択画面へ

### 5.2 1日の流れ（保育園内）

```
09:00  登園、先生と挨拶、自由遊びで保育園生活スタート
10:00  朝の会、みんなで絵本
11:00  クラス活動（うた、てあそび）
12:00  給食、昼食休憩
13:00  生活活動時間（偶数日:室内遊び / 奇数日:公園遊び）※ §5.4 参照
15:00  おやつ、自由遊び
16:00  降園（帰宅）
```

プロト実装では、これらを **1画面の中で時系列で描写** して見せる（アニメーション）。

### 5.3 獲得値（確定分）

| 項目 | 値 |
|---|---|
| 経験値 physical | +5 |
| 経験値 social | +3 |
| ジョウネツ | +1 |
| 体力消費 | 年齢に応じて可変（下表） |
| 時刻 | 09:00 → 16:00 |

#### 5.3.1 年齢別の体力消費
体力上限（SPEC-019）に対して約半分を消費するバランスで設計。

| 年齢 | 体力上限 | 保育園コアタイム消費 |
|---|---|---|
| 1 | 10 | -5 |
| 2 | 15 | -8 |
| 3 | 30 | -12 |
| 4 | 50 | -20 |

#### 5.3.2 昼寝による体力回復（1〜2歳のみ）
- 1〜2歳は 13:00 の昼寝時に `floor(staminaCap × 0.20)` の体力回復。
- 3歳以降は **昼寝なし**（SPEC-019 §5.1 テーブルの注記）。

> 起床時刻が 09:00 未満なら、一旦 `clockHour = 9` に揃えてから保育園を開始する。
> 起床が 09:00 を超えていた（寝坊・体調不良）場合は、`clockHour` をそのまま 16:00 まで進める。

### 5.4 生活活動時間（13:00〜15:00）

日によって内容が変わる。プロト段階のルール：
- `player.day % 2 === 0`（偶数日）：**室内遊び**
- `player.day % 2 === 1`（奇数日）：**公園遊び**

内容による差分：
| 種別 | 追加経験値 | 備考 |
|---|---|---|
| 室内遊び | creative +3 | 工作・粘土・絵本など |
| 公園遊び | physical +5 | 滑り台・砂場・追いかけっこ |

### 5.5 発見（Discovery）の抽選

毎日、コアタイム消化中に **確率 70%** で以下のプールから 1 件を抽選（重み付け）する。

| ID | テキスト | 重み | 効果 |
|---|---|---|---|
| `meet_new_friend` | 新しいお友達と仲良くなった！ | 10 | 友人数 +1, social +5 |
| `learn_slide` | 大きな滑り台の上手な滑り方を教えてもらった！ | 8 | **「大きな滑り台」解禁**（遊び `big_slide`）, physical +3 |
| `learn_sandcastle` | 砂場で先生にお城の作り方を教わった！ | 8 | creative +5, physical +3（既存「砂場遊び」効果ブースト） |
| `nap_refresh` | お昼寝でぐっすり眠った。夢の中で冒険した。 | 6 | 体力 +10, 探求 +3 |
| `song_time` | 「はらぺこあおむし」の歌を覚えた！ | 6 | 探求 +3, creative +3 |
| `cry_then_smile` | 友達とおもちゃを取り合って泣いた…最後は仲直り。 | 4 | social +4, ジョウネツ +2 |
| `park_adventure` | 公園で小さな虫を見つけて観察した！ | 5 | 探求 +4, physical +2 |

- 発見が起こらなかった日は、「平穏な一日だった」のテキストのみを表示。
- 同じ遊びが二重に解禁されないよう、**既に解禁済みの遊びは発見プールから除外** する。

### 5.6 解禁される遊び

保育園の発見で解禁される遊びのプロト実装：

| 遊びID | 名前 | アイコン | 初期状態 | 解禁条件 |
|---|---|---|---|---|
| `big_slide` | 大きな滑り台 | 🎢 | ロック | `learn_slide` を発見済み |

- それ以外の解禁遊び（例：手押し車、三輪車）は将来拡張として Draft 扱い。
- 解禁された遊びは `player.unlockedPlays` 配列に保存し、S2 遊び選択画面の候補に含める。

### 5.7 UI上の描写順（S6 保育園画面）

1. 大きなイラスト（朝の保育園の風景）と「🏫 保育園に行ってきた」タイトル
2. タイムラインカード：1日のざっくりしたログ（09:00 登園…16:00 降園）
3. **発見カード**（発見があった場合のみ）：テキスト + 効果差分
4. **獲得したもの**カード：経験値差分・体力差分・解禁遊び
5. 「家に帰る」ボタンで S2 へ遷移

### 5.8 スキップ可否
- **スキップ不可**（SPEC-010 §5.7）。
- プレイヤーがコアタイムをスキップすると「遊びの発見」機会を失う設計。

## 8. 例外・エッジケース

- 起床が 9:00 過ぎの場合：`clockHour` をそのまま 16:00 に進める（遅刻の演出は将来）。
- **自動モード中（SPEC-025 §6.2）**：S6 保育園画面の「家に帰る」ボタンは自動でタップされる（画面遷移のみ。プレイヤーの操作不要）。
- **余剰時間の計算（更新 SPEC-020 準拠）**：
  - 朝の遊び時間：07:00 起床 〜 09:00 コアタイム開始 → **2h**
  - 夜の遊び時間：16:00 コアタイム終了 〜 18:00（食事バッファ前） → **2h**
  - 合計余剰時間ベース（SPEC-021 分母用）：**4h**
- 体力 0 時：**SPEC-019 §5.4.1 強制睡眠**が発火（2時間仮眠 + 体力を `staminaCap × 0.30` へ回復）。
  - 仮眠がコアタイム（09:00〜16:00）と重なった場合は、**コアタイム時間が仮眠を吸収** して夜の余剰時間は保護される（SPEC-019 §5.4.1a）。例：朝 08:00 に体力 0 → 仮眠 08:00-10:00 → 10:00 はコアタイム中なので、夜の余剰 2h はそのまま残る。
- 発見による解禁が全て済んでいる場合、`meet_new_friend` 等の「効果のみ」発見のみが残る。

## 10. 保育園「貯める」ギミック（Issue #14 反映 v2）

Issue #14 のライフステージごとの基本コンセプトで、保育園は **「貯める」遊び** と定義された。遊びの選択肢は少なめだが、没頭して素養（SPEC-033）をどんどん貯める期間。以下の 3 つのギミックを仕様として追加する（順次実装）。

### 10.1 親遣い遠出イベント（Parental Outing）
- 1日の始まり（起床直後）に、**親の意思で特別な遊びが発動** することがある
- 発生タイミング：
  - 土日祝日（`dayOfWeek` が土日）：確率 40%
  - 平日：確率 2%（稀）
- ネガティブではなく **特別な遊び**。経験値は通常の 1.5〜2.0 倍
- 例：`trip_zoo`（動物園）・`trip_aquarium`（水族館）・`trip_park_big`（大きな公園）
- 発動したその日は **他の遊びは不可**。遠出の 1 発だけ遊んで **直接 S10 日の終わり** へ
- 親の支出：`moneyCost` は親が負担するため `player.money` からは引かない（幼少期なので関係ない）

**データ構造**（SPEC-028 マスタに追加）:
```json
{
  "id": "trip_zoo",
  "scope": "parental_outing",
  "icon": "🦒",
  "name": "動物園",
  "lifeStageTag": "nursery",
  "gain": { "intellect": 30, "sensitivity": 20, "social": 10, "passion": 8 },
  "text": "おとうさん・おかあさんに動物園に連れて行ってもらった！"
}
```

### 10.2 アイテムバフ（Item Buff）
- 遠出系イベント（または稀な「お買い物」イベント）の後、**親が新しい絵本やおもちゃを買ってくれる** ことがある
- 購入されたアイテムは `player.items[]` に追加される
- アイテムごとに「{カテゴリ}系の遊びで獲得経験値 +N%」のバフが付く
- 例：
  - `item_new_picturebook`：読書カテゴリの遊びで +15%
  - `item_sandbox_set`：クラフト系の遊びで +20%
  - `item_ball`：運動系の遊びで +15%
- バフは `finalizePlay()` の gainBoost に乗算で反映

### 10.3 感染症イベント（Infection）
- ネガティブイベント。保育園で感染症にかかる可能性がある
- 発生条件：
  - 月別確率：**4-6月**（春）・**11-2月**（冬）は高確率 15%／日
  - 7-10月（夏秋）は低確率 3%／日
- 発動すると：
  - 3〜5 日間、朝の余剰時間がゼロ（病欠）
  - 体力が毎朝 `-20%`（発熱）
  - 体力は毎朝 0.5 倍しか回復しない
  - 連絡帳に「お休みのお知らせ」が表示される
- 3 日目以降に自然回復する

### 10.4 実装ステップ
1. イベントマスタ（`events.json`）に `scope: "parental_outing"` / `scope: "infection"` を追加
2. `nextDay()` 冒頭で確率抽選し、親遣い遠出 or 感染症を発火
3. 発火した日は `player._specialDayMode = "outing" | "infection"` フラグを立てて S2 の dock を隠す
4. `renderSoyouResultList` は既存の描画でそのまま動く
5. アイテムは `player.items = [{ id, buffs: { category: multiplier } }]` で持ち、`skillBoostMultiplier()` と並列に適用
