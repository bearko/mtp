# SPEC-011: 保育園（1〜3歳）コアタイム

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-011 |
| 機能名 | 保育園ライフステージ・コアタイム実装 |
| 対応ファイル | `prototype/game.js` の `LIFE_STAGES.nursery`, `NURSERY_DISCOVERIES`, `playCoreTime()`, `renderNurseryScreen()` |
| 関連仕様 | SPEC-001, SPEC-002, SPEC-007, SPEC-010 |
| ステータス | Active（実装対象） |
| 最終更新 | 2026-04-18 |

## 1. 目的

1〜3歳のプレイヤーは、平日 **09:00〜16:00** を保育園で過ごす。  
保育園では「遊びの基礎」を学び、公園の遊具や室内遊具の遊び方を覚えることで、帰宅後の余剰時間に使える **新しい遊びを解禁** する。

コンセプト：
- 時間を拘束されるが、**そこで得る発見が後の遊びを豊かにする**。
- 保育園に行くこと自体が原体験ポイントとなり、一定確率で発見が起こる。

## 2. 用語定義

- **保育園コアタイム**：09:00〜16:00 の時間帯。
- **生活活動時間枠**：13:00〜15:00 の時間帯で、日替わりで **室内遊び** または **公園遊び** が行われる。
- **発見（Discovery）**：保育園での学びや経験で得られるもの。
  - 新しい遊びの解禁
  - 基礎パラメーター（身体系／社会系）の加算
  - 友人数の増加

## 3. 入力（Input）

- プレイヤー状態（SPEC-001）：年齢が 1〜3 であること。
- 日付（`player.day`）：日替わりで「室内 / 公園」を決めるために使用。

## 4. 出力（Output）

- 時刻の進行：`clockHour = 16`（コアタイム終了時刻）。
- 原体験・パラメーターの加算（下記 §5.3）。
- 友人数の増加（確率）。
- 新しい遊びの解禁通知（該当時）。
- 画面：S6 コアタイム画面（保育園仕様）の描画。

## 5. 挙動（Behavior）

### 5.1 発動条件
- プレイヤーの年齢が 1〜3 歳。
- 起床直後、S1 → S2 に進もうとするタイミング。
- `player.clockHour <= 9`（朝に到着する想定）。

### 5.2 1日の流れ（保育園内）

```
09:00  登園、先生と挨拶、自由遊びで保育園生活スタート
10:00  朝の会、みんなで絵本
11:00  クラス活動（うた、てあそび）
12:00  給食、昼食休憩
13:00  生活活動時間（偶数日:室内遊び / 奇数日:公園遊び）※ §5.4 参照
15:00  おやつ、自由遊び
16:00  降園（帰宅）
```

プロト実装では、これらを **1画面の中で時系列で描写** して見せる（アニメーション）。

### 5.3 獲得値（確定分）

| 項目 | 値 |
|---|---|
| 経験値 physical | +5 |
| 経験値 social | +3 |
| ジョウネツ | +1 |
| 体力消費 | -15 |
| 時刻 | 09:00 → 16:00 |

> 起床時刻が 09:00 未満なら、一旦 `clockHour = 9` に揃えてから保育園を開始する。
> 起床が 09:00 を超えていた（寝坊・体調不良）場合は、`clockHour` をそのまま 16:00 まで進める。

### 5.4 生活活動時間（13:00〜15:00）

日によって内容が変わる。プロト段階のルール：
- `player.day % 2 === 0`（偶数日）：**室内遊び**
- `player.day % 2 === 1`（奇数日）：**公園遊び**

内容による差分：
| 種別 | 追加経験値 | 備考 |
|---|---|---|
| 室内遊び | creative +3 | 工作・粘土・絵本など |
| 公園遊び | physical +5 | 滑り台・砂場・追いかけっこ |

### 5.5 発見（Discovery）の抽選

毎日、コアタイム消化中に **確率 70%** で以下のプールから 1 件を抽選（重み付け）する。

| ID | テキスト | 重み | 効果 |
|---|---|---|---|
| `meet_new_friend` | 新しいお友達と仲良くなった！ | 10 | 友人数 +1, social +5 |
| `learn_slide` | 大きな滑り台の上手な滑り方を教えてもらった！ | 8 | **「大きな滑り台」解禁**（遊び `big_slide`）, physical +3 |
| `learn_sandcastle` | 砂場で先生にお城の作り方を教わった！ | 8 | creative +5, physical +3（既存「砂場遊び」効果ブースト） |
| `nap_refresh` | お昼寝でぐっすり眠った。夢の中で冒険した。 | 6 | 体力 +10, 探求 +3 |
| `song_time` | 「はらぺこあおむし」の歌を覚えた！ | 6 | 探求 +3, creative +3 |
| `cry_then_smile` | 友達とおもちゃを取り合って泣いた…最後は仲直り。 | 4 | social +4, ジョウネツ +2 |
| `park_adventure` | 公園で小さな虫を見つけて観察した！ | 5 | 探求 +4, physical +2 |

- 発見が起こらなかった日は、「平穏な一日だった」のテキストのみを表示。
- 同じ遊びが二重に解禁されないよう、**既に解禁済みの遊びは発見プールから除外** する。

### 5.6 解禁される遊び

保育園の発見で解禁される遊びのプロト実装：

| 遊びID | 名前 | アイコン | 初期状態 | 解禁条件 |
|---|---|---|---|---|
| `big_slide` | 大きな滑り台 | 🎢 | ロック | `learn_slide` を発見済み |

- それ以外の解禁遊び（例：手押し車、三輪車）は将来拡張として Draft 扱い。
- 解禁された遊びは `player.unlockedPlays` 配列に保存し、S2 遊び選択画面の候補に含める。

### 5.7 UI上の描写順（S6 保育園画面）

1. 大きなイラスト（朝の保育園の風景）と「🏫 保育園に行ってきた」タイトル
2. タイムラインカード：1日のざっくりしたログ（09:00 登園…16:00 降園）
3. **発見カード**（発見があった場合のみ）：テキスト + 効果差分
4. **獲得したもの**カード：経験値差分・体力差分・解禁遊び
5. 「家に帰る」ボタンで S2 へ遷移

### 5.8 スキップ可否
- **スキップ不可**（SPEC-010 §5.7）。
- プレイヤーがコアタイムをスキップすると「遊びの発見」機会を失う設計。

## 6. UIへの反映

- 画面：S6 コアタイム画面（保育園バージョン）
- HUD：ライフステージ名として「🏫 保育園」を表示。
- 画面設計書：`docs/screen-design.md` §2.6（追加）

## 7. 想定される利用シナリオ

### シナリオA（解禁あり）
- 2歳・3日目（奇数日）
- 06:30 起床 → 「今日を始める」 → S6 保育園画面へ
  - 09:00 登園
  - 13:00 公園遊び
  - 14:00 **発見「大きな滑り台の上手な滑り方を教えてもらった！」**（`learn_slide`）
  - 16:00 降園
- 獲得：physical +10、social +3、creative +0、ジョウネツ +1、体力 -5（-15 +10 昼寝）
- 新規解禁：🎢 大きな滑り台
- [家に帰る] → S2 遊び選択へ。候補に「大きな滑り台」が追加されている。

### シナリオB（発見なし）
- 1歳・4日目（偶数日）
- 06:30 起床 → S6 保育園画面
  - 13:00 室内遊び（粘土）
  - 発見抽選：外れ（30%）
  - 平穏に1日終了
- 獲得：physical +5, social +3, creative +3, ジョウネツ +1, 体力 -15

## 8. 例外・エッジケース

- 起床が 9:00 過ぎの場合：`clockHour` をそのまま 16:00 に進める（遅刻の演出は将来）。
- 余剰時間：コアタイム消化後、`spareHours = 21:00 − 現在時刻 − 食事2h` で暫定計算（プロト）。
  - 例：16:00 終了 → 21:00 までの5h − 食事2h = **3h**
- 体力 0 時：体力消費は 0 でクランプ。
- 発見による解禁が全て済んでいる場合、`meet_new_friend` 等の「効果のみ」発見のみが残る。
