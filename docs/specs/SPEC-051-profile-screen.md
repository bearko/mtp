## 0. 本 SPEC のコンセプト（最重要、これが魂）

### 0.1 コンセプト
> 「スコア画面」ではなく、**「自分の宝箱」**。  
> 数字を並べて他人と比較するためではなく、**自分と大切な人の足跡を愛でるための場所**。

### 0.2 設計の柱：4 つの成長実感（SPEC-048 §12.1 再掲）

本画面はどのタブを開いても 4 つの軸のどれかを感じられる：
1. **できるようになった** → 称号タブ、ミッション達成履歴
2. **認められた** → 称号タブのフレーバーテキスト
3. **親が喜んでくれた** → 思い出タブの連絡帳アーカイブ
4. **親が褒められた** → 思い出タブの「周りからの声」

### 0.3 絶対原則：他人と比較しない

保育園期・幼稚園期は：
- **他プレイヤーの平均値**を出さない
- **クラスの順位**を出さない
- **「上位 X%」**などのパーセンタイル表示をしない

あるのは **「昨日までの自分」との比較**のみ。

## 1. 目的

プレイヤーが **振り返り・達成感・これから**を一望できる場所。
マイライフの「シーズン成績画面」に相当するが、**数字ではなくエピソード中心**に設計する。

## 2. 4 タブ構成

### 2.1 タブ構成の再設計（v1 からの変更）

v1 で提案した 4 タブを **コンセプトに合わせて改名** した：

| v1 | v2（本仕様） | 改名理由 |
|---|---|---|
| 🎯 今年のミッション | 🎯 **今やっている目標** | SPEC-050 の表現と統一、年単位でなくストーリー単位 |
| 🏆 称号 | 🏆 **できるようになったこと** | 「称号」ゲーム用語は白々しい。4 軸①の表現に直接 |
| 💫 ハマってる遊び | 💖 **だいすきな遊び** | 「ハマり」は中学生以降の表現。幼児向けの語に |
| 📜 人生通算記録 | 📖 **思い出のアルバム** | 「通算記録」はスポーツ的。幼児期はアルバム的 |

### 2.2 ドックへのアイコン追加

ドック右端に `📊 自分` アイコンを追加。アイコン絵文字は v1 から据え置き。

> Why `📊` か？：記録を見るという意味のシンボル。ただし UI テキストは「自分」として、ゲーム的な冷たさを避ける。

## 3. 🎯 今やっている目標（ミッションタブ）

### 3.1 表示内容
SPEC-050 のミッションバナーと同じ内容を、大きなカードで閲覧：

```
┌──────────────────────────┐
│ 🎯 今やっている目標          │
├──────────────────────────┤
│ 🧩 パズルにちょうせん！      │
│   発端：児童館で見かけた     │
│   触媒：「簡単なのから…」     │
│   今：4 ピースを練習中       │
│   進捗：[========    ] 75%   │
│                              │
│   【達成条件】               │
│   ・身体 30 以上（現 22）    │
│   ・知性 40 以上（現 35）    │
│   ・情熱 50 以上（現 42）    │
│   ・パズル 4 / 6 ピース完了   │
├──────────────────────────┤
│ 🎤 大きな声でうたう           │
│ ...                          │
└──────────────────────────┘
```

### 3.2 Why：達成条件を詳しく表示する
- ゲームの仕組み理解を助ける
- 「あと少しで達成できる」とプレイヤーが意識できる

### 3.3 Why not：達成率を 1 つの数字に集約
- 「75%」と示すと、プレイヤーは数字を追い始める
- 各条件を個別に示すことで、**どの活動が必要か** が自然に伝わる

## 4. 🏆 できるようになったこと（称号タブ）

### 4.1 表示内容

称号は「今までできなかったことができるようになった瞬間」の記念。

```
┌──────────────────────────┐
│ 🏆 できるようになったこと    │
│                              │
│  1 年目（1-2 歳）            │
│  ─────────────────          │
│  🎀 パパ・ママを呼べる        │
│     「2026 年 7 月 12 日」    │
│     『ママ、だっこ！』          │
│                              │
│  🎀 立って歩ける              │
│     「2026 年 9 月 3 日」     │
│     『よちよち、えへへ』       │
│                              │
│  2 年目（2-3 歳）            │
│  ─────────────────          │
│  🧩 パズル 9 ピース           │
│     「2027 年 5 月 18 日」    │
│     『できた！』＋ 母「えらいね〜」│
│                              │
│  🎤 発表会で歌えた            │
│     「2027 年 12 月 22 日」   │
│     先生「みんなに聞こえたね」 │
│                              │
│  （まだ）                    │
│  ─────────────────          │
│  🔒 ひとりで寝られる          │
│  🔒 お絵かきで家族を描く      │
│  ...                         │
└──────────────────────────┘
```

### 4.2 表示ルール

- **年齢別グルーピング**：1 年目・2 年目・3 年目…（他プレイヤーの平均と比較しない）
- **獲得日時を日付で明記**：「いつできるようになったか」が自分ごととして見える
- **フレーバーテキスト** 付き：達成時の主人公・先生・親のセリフから 1 つ引用
- **まだできないもの** は薄く表示、具体的な条件は伏せる（「お絵かきで家族を描く」のタイトルだけ）

### 4.3 Why「称号」と呼ばない
- 「称号」「メダル」はゲーム用語で、主人公（幼児）の素朴な成長記録と相性が悪い
- **「できるようになったこと」**という言葉に近い「花」絵文字 🎀 で飾る
- プレイヤーが見て「そうそう、これができた日だ」と過去を愛おしめる

### 4.4 Why not：グローバルランキング・友達との比較
- 「あなたはクラスで XX 番目に多く称号を獲得」などの表示は、本作のコアメッセージを損ねる
- 他プレイヤーのデータは一切参照しない

### 4.5 自動認定ロジック

SPEC-050 のミッション達成時に称号が自動で追加される。他に：
- **マイルストーン称号**：年齢到達時（1 歳、2 歳…）
- **累積称号**：同じ遊びを累計 N 回
- **全素養系**：5 素養すべてが F 以上、等

称号マスタは `prototype/data/titles.json` で管理。初期 40 件程度を用意：

```json
[
  {
    "id": "title_say_mama",
    "name": "パパ・ママを呼べる",
    "icon": "🎀",
    "lifeStageTag": "nursery",
    "trigger": { "type": "discoveryFound", "discovery": "disc_1_mama" },
    "flavorText": { "speaker": "player", "text": "『ママ、だっこ！』" }
  },
  {
    "id": "title_puzzle_9",
    "name": "パズル 9 ピース",
    "icon": "🧩",
    "lifeStageTag": "nursery",
    "trigger": { "type": "missionCompleted", "missionId": "m_puzzle_9" },
    "flavorText": { "speaker": "mother", "text": "えらいね〜、できたね！" }
  }
]
```

│ ... など                     │

### 5.2 Why 回数ベースか
- 「何回遊んだか」は自分との比較で完結
- 「クラスで何番目に好きか」などは出さない

### 5.3 NPC 情報の織り込み
- SPEC-041 の NPC 関係で「よく一緒に遊んだ子」を表示
- 「誰と」「どこで」の記録が、**大切な人と一緒にいた時間** を可視化

### 6.2 3 セクション

#### §6.2.1 両親・先生からの言葉（感情セクション）

- 連絡帳（SPEC-027）の **両親コメント・先生コメント** から、感情温度が高いものを自動ピックアップ
- 「嬉しい」「驚いた」「誇らしい」などのキーワード含むコメントを 10 件アーカイブ
- **プレイヤーが何度でも読み返せる**、マイライフの「ファンレター画面」に相当

#### §6.2.2 忘れられない日（エピソードセクション）

- fullday_trip で訪れた場所
- ミッション達成のドラマチックな日
- NPC との関係が大きく変わった日
- 日付順に時系列表示

#### §6.2.3 数字で見る（最小限）

- プレイ日数・行った場所数・読んだ絵本数などを控えめに表示
- 素養の現在値・グレードは表示しない（S2 HUD で見える）
- **「ちょっと」の前置き**で、数字は主役じゃない空気を作る

### 6.3 Why 感情軸か
- §4.1 の実感軸 ③④「親が喜んでくれる」「親が褒められる」を最も強く回収できる場所
- プレイヤーが落ち込んだ時、ここを見れば「自分の子のこと、たくさん褒めてもらえたなあ」と温かくなれる

### 6.4 Why not：詳細統計ダッシュボード
- グラフ・折れ線・ヒートマップは Excel っぽくなる
- 幼児期の記録に「可視化」は邪魔
- ただし通算記録数値は §6.2.3 で最小限だけ出す

### 8.2 自動ピックアップロジック

- 1 日ごとに `finalizePlay()` で連絡帳コメントが生成される際、emotion スコアを判定
- emotion スコア >= 7（10 段階）なら `renrakuchoHighlights` に追加
- 常に最新の 10 件を保持

`prototype/data/titles.json`：40 件の称号定義（§4.5）

## 10. Why not：以下の機能は導入しない

### 10.1 Why not：複数キャラ管理
- 本作は「1 人の人生を 100 年」がコア
- 複数キャラ切替はコンセプトを薄める

### 10.2 Why not：他プレイヤーとの比較・SNS シェア
- コアメッセージ「どの子も豊かに育つ」を損ねる
- プレイヤーは比較されることで安心するが、主人公は競争にさらされる

### 10.3 Why not：数字グラフ中心
- 数字は S2 HUD と S10 日サマリで見える
- プロファイル画面は **感情で振り返る** 場所

### 10.4 Why not：動的ニュース・お知らせ
- 「今月のおすすめ」「新着イベント」等は運営メッセージの色が濃く、ゲーム世界観を壊す
