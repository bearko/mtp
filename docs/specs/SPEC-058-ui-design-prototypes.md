## 1. 目的

PR #30 の HUD 案を本線として採用し、S2 を「情報が見える」「次の行動が選びやすい」ゲーム HUD 寄りのプロトタイプへ整理する。

同時に、S2「遊びを選ぶ」画面へ毎日表示の **今日の焦点カード** を追加し、プレイヤーが「次に何をするとよいか」を迷いにくくする。

## 2. 用語定義

- **ゲーム HUD 寄り**: 濃色 HUD・高コントラスト・バッジ・数値の読み取りやすさを優先する表現。
- **今日の焦点カード**: S2 の上部に毎日表示される「きょうのめあて」カード。
- **きろく**: SPEC-051 のプロフィール導線を、幼少期向けに言い換えた名称。

## 3. 入力（Input）

- 現在の `player` 状態。
- `player.activeMissions` / `MISSION_SCENARIOS`。
- 遊びマスタ `PLAYS` と各遊びの `gain` / `categories`。
- UI デザイントーン: `gamehud` 固定。

## 4. 出力（Output）

- `.phone-frame` に `theme-gamehud` クラスを固定付与する。
- S2 に `#daily-focus-card` を毎日表示する。
- ドック末尾に `🌱 きろく` ボタンを表示し、`#screen-record` へ遷移できる。

## 5. 挙動（Behavior）

### 5.1 デザイントーン

1. `prototype/index.html` の `.phone-frame` は `theme-gamehud` を固定で持つ。
2. タイトル画面と S2 にデザイン切替 UI は表示しない。
3. 絵本風テーマは今回のスコープから外す。

### 5.2 今日の焦点カード

S2 を表示するたびに `renderDailyFocusCard()` を呼ぶ。

優先順位:

1. 進行中ミッション（`state === "ACCEPTED"`）がある場合、そのミッションを最優先で表示する。
2. 進行中ミッションがない場合、現在の素養のうち最も伸びているものを見つけ、その素養を伸ばせる実行可能な遊びを最大 2 件提案する。
3. 実行可能な遊びがない場合、「今日はゆっくり休む」提案にする。

表示内容:

- ラベル: `きょうのめあて`
- 見出し: ミッション名または伸ばしたい素養
- 本文: 今日の選択理由
- 推奨遊び: 最大 2 件のピル

### 5.3 遊びアイコンの意味バッジ

S2 ドックの遊びアイコンに、必要に応じて小さなバッジを表示する。

| バッジ | 条件 |
|---|---|
| `めあて` | 今日の焦点カードで推奨された遊び |
| `好き` | 累計プレイ回数が 3 回以上 |
| `新しい` | 累計プレイ回数が 0 回 |
| `つかれる` | `avail.isLowStamina === true` |

### 5.4 きろく画面

`🌱 きろく` ボタン押下で `#screen-record` を表示する。

初期プロトタイプでは以下の 3 カードを表示する。

1. `できたこと`: 獲得済み称号数と最新称号。
2. `だいすきな遊び`: 累計回数上位 3 件。
3. `思い出`: 連絡帳ハイライトまたは忘れられない日。

## 6. UI への反映

- S0 タイトル: 通常の開始導線のみ表示。
- S2 遊び選択: 今日の焦点カード、遊びバッジ、`🌱 きろく` 導線を表示。
- S17 きろく: 記録を感情寄りに振り返る簡易画面。

## 7. 想定される利用シナリオ

### 7.1 S2 で今日の焦点を見る

S2 を開くと、`きょうのめあて` が常に表示される。進行中ミッションがあればミッション、なければ伸びている素養と関連遊びが表示される。

### 7.2 遊びバッジで選ぶ

ドックの `めあて` / `好き` / `新しい` / `つかれる` バッジを見て、遊びの意味を判断する。

### 7.3 きろくを見る

ドックの `🌱 きろく` を押すと、できたこと・好きな遊び・思い出を確認できる。

## 8. 例外・エッジケース

- 進行中ミッションが複数ある場合は `MISSION_SCENARIOS.priority` が高いものを優先する。
- 推奨遊びがロック中の場合は表示候補から外す。
- `player._playCounts` が空の場合、だいすきな遊びカードは「これから増えていく」と表示する。
- `titles` / `renrakuchoHighlights` / `memorableDays` が空の場合、きろく画面は空状態の文言を表示する。

## 9. Why not

### 9.1 Why not: 絵本風テーマを残す

ユーザー判断により、PR #30 の HUD 案で進行する。絵本風テーマは一旦オミットし、比較 UI も外す。

### 9.2 Why not: 今日の焦点カードをミッション中だけ表示する

ユーザー指示により、今回は毎日表示で作成する。ミッションがない日も「今日は何を大切にするか」を示すことで、S2 の迷いを減らす。

### 9.3 Why not: プロフィール導線を `アルバム` にする

ユーザー指示により、導線名は `きろく` とする。`きろく` は分析的すぎず、幼少期にも後年にも使いやすい中間表現である。

### 9.4 Why not: テーマ切替 UI を残す

本線が HUD 案に決まったため、切替 UI はプレイヤーに不要な選択肢を増やす。`theme-gamehud` 固定にして操作を単純化する。
