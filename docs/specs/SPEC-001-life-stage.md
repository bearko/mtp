# SPEC-001: ライフステージ / プレイヤー状態

| 項目 | 内容 |

## 1. 目的
1歳から100歳までの人生を、プレイヤーの「状態」として一元管理する。  
年齢・季節・時刻・体力・生活リズム・所持金・友人数・ジョウネツ・原体験Lvを持つ。

## 2. 用語定義
- **プレイヤー状態**：現在のプレイヤー全体の状態を保持するオブジェクト（`player`）。
- **ライフステージ**：年齢区分。未就学児（1〜6）・小学生（7〜12）・中学生（13〜15）・高校生（16〜18）・大学生（19〜22）・社会人（23〜29）・結婚／育児（30〜45）・中年（46〜60）・定年（61〜75）・老後（76〜100）。
- **余剰時間（spareHours）**：1日の中で遊びに使える時間。睡眠・食事・学業/仕事を除いた時間。

## 3. 入力（Input）
- ゲーム開始時：初期値（年齢5歳、夏、体力80、リズム90、所持金0、友人2、ジョウネツ0）。
- 日々の遊びの結果（SPEC-003, SPEC-005）。
- 就寝結果（SPEC-006, SPEC-008）。

## 4. 出力（Output）
- HUDへの描画（年齢 / 日付・季節 / 時刻 / 体力 / 生活リズム / 所持金 / 余剰時間 / 友人数 / ジョウネツ）。
- ライフステージ判定結果。

## 5. 挙動（Behavior）
### 5.1 プレイヤー状態の構造
```
age            年齢 1〜100
day            ゲーム内通算日
season         "spring" / "summer" / "autumn" / "winter"
clockHour      時（0-24）
clockMinute    分（0-59）
spareHours     余剰時間（h）
stamina        体力 0-100
bioRhythm      生活リズム 0-100
money          所持金（円）
friends        友人数（人）
passion        ジョウネツ（累積スコア）
exp            原体験5カテゴリの経験値
```

### 5.2 年齢進行
- プロト段階では **120日で +1歳**（年齢100で到達時にエンディング予定、未実装）。

### 5.3 季節の循環
- 30日ごとに次の季節へローテ（春→夏→秋→冬→春）。

### 5.4 時刻の表現
- 起床時刻は SPEC-006（生活リズム）によって決まる。
- 遊びの所要時間だけ時刻が進行する（SPEC-003）。

## 6. UIへの反映
- 共通HUD（全画面共通の上部固定バー）に表示：年齢 / 日付・季節 / 時刻 / 体力 / リズム / 所持金 / 余剰時間 / 友人 / ジョウネツ。
- 画面設計書：`docs/screen-design.md` §1.1

## 7. 想定される利用シナリオ
- 朝起きると HUDに「5歳 / 3日目(夏) 06:30」のように表示される。
- 遊びで1時間消費すると時刻が +1h 進み、余剰時間が -1h される。

## 8. 例外・エッジケース
- `spareHours` は 0 未満にならないようにクランプ。
- 100歳到達時はエンディング画面へ（未実装）。
