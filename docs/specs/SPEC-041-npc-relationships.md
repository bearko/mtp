## 1. 目的

従来 `player.friends` という **整数 1 つ** だけで友人数を管理していたが、世界観の充実と感情の起伏のために、**NPC ごとの関係値** を持つシステムに刷新する。

親・担任・親友・恋人・上司といった **主要 NPC** を名前と基調付きで登場させ、プレイヤーが「人と共に人生を歩んでいる」感覚を得られるようにする。

## 2. 用語定義

- **NPC（Non-Player Character）**：プレイヤー以外の人物（親・先生・友達・恋人・上司など）
- **関係値（Relationship Value）**：プレイヤーと NPC の親密度を 0〜100 で表す数値
- **関係性ラベル**：関係値に応じて付与される文字列（初対面／知り合い／親しい／親友／恋人／生涯の友、等）
- **NPC タイプ**：親族／教育者／友人／恋人・配偶者／同僚／上司／後進

### 3.2 プレイヤーオブジェクト

```js
player.relationships = {
  "mother":        { value: 90, history: ["出生時"] },
  "father":        { value: 80, history: [] },
  "teacher_sakura": { value: 60, history: ["入園日に笑顔で迎えてくれた"] },
  "friend_yuta":   { value: 55, history: ["公園で話しかけた"] }
};
```

- **value**：0〜100 の整数
- **history**：簡易エピソード文字列の配列（将来、連絡帳や回想画面で使う）

## 4. 関係値の扱い

### 4.1 関係性ラベル

| 範囲 | ラベル | 説明 |
|---|---|---|
| 0〜9 | 他人 | まだ出会っていない／関係が壊れた |
| 10〜29 | 顔見知り | 名前を知っている程度 |
| 30〜49 | 知り合い | たまに話す |
| 50〜69 | 友達 | 日常的に関わる |
| 70〜84 | 親しい | 悩みを相談できる |
| 85〜94 | 親友・家族 | 人生に大きな影響 |
| 95〜100 | かけがえのない人 | 一生の宝物 |

### 4.2 関係値の上下要因

| 要因 | 変化量 |
|---|---|
| 一緒に遊ぶ（同じ遊びを共有） | +2 |
| 相手のイベントに参加 | +5 |
| ケンカ・衝突イベント | −10 |
| 長期間会わない（60日以上） | 毎30日で -1 |
| 恋人化イベント成功 | +20、以降 90+ を維持しやすい |
| 結婚イベント | +10、以降 95+ 安定 |
| 死別 | 関係値固定で history に「永遠に記憶される」追加 |

### 4.3 日々の自動調整

```js
// nextDay() 内で全 NPC を走査
for (const id of Object.keys(player.relationships)) {
  const rel = player.relationships[id];
  const lastSeen = rel.lastSeenDay || 0;
  const daysSince = player.day - lastSeen;
  if (daysSince >= 60) {
    rel.value = Math.max(0, rel.value - 1);
  }
}
```

## 6. NPC の登場・退場フロー

### 6.1 登場
- ライフステージの切替時、`lifeStageRange[0]` に到達した NPC を `player.relationships` に追加
- 初期値は NPC マスタの `initialValue`
- 登場時にトースト：「[名前] と出会った」

### 6.2 退場
- `lifeStageRange[1]` を超えた NPC は **history に「別れのエピソード」を追加** し、関係値を保持したまま接触機会は消える
- 退場時にモーダル：「[名前] との思い出」（history を表示）
- 死別イベント（SPEC-046 §5）は独立した扱い

## 8. 遊びとの連動

### 8.1 一緒に遊ぶ遊び
- 一部の遊びには `coPlayNpcs: ["friend_yuta"]` のように NPC を紐付けて、その遊びをすると関係値 +2

### 8.2 一緒にいないと出来ない遊び
- デート・BBQ・結婚式など、特定 NPC の関係値が一定以上で解禁される

## 9. テスト観点

- 保育園入園時（Day 1）に mother / father / teacher_sakura の 3 人が `player.relationships` に登録されている
- 絵本を母と読む遊び `picturebook` をすると mother.value が +2 / +2 / +2 と段階的に上がる
- 卒園時に teacher_sakura の history に「卒園式で涙を流した」が追加される
- 中学校 以降 で friend_yuta の関係値が 70 を超えていると部活勧誘イベントが発火
