## 1. 目的

プレイヤーの現在状態（年齢・季節・所持金・友人数・体力・余剰時間）に応じて、  
「今、実行できる遊び / できない遊び」を明示し、意思決定の場を提供する。

UI は **画面下部の横スクロールできるアイコン行（ドック）** 形式を採用し、  
**アイコンタップ → 上部にイメージ・詳細表示 → 再タップで確定** の 2段タップで選ぶ。  
スマホ片手操作を前提とし、親指の届く範囲にアイコンを集めることで快適性を担保する。

### 1.1 起床画面（S1）との統合（v2）

旧フローでは `S1 起床 → [今日を始める] → S2 遊びを選ぶ` と 1 タップ余分に要していたが、  
**1日の最初の遊び画面に起床時のコンディション表示を埋め込み、S1 と S2 を 1 画面に合体** させる。
- 起床時のメッセージ・時計円盤・体力/リズムゲージを **S2 の上部に常時表示**
- 「今日を始める」ボタンは廃止。初期画面は直接 S2（朝の遊び選択画面）になる
- これにより 1日あたり 1 タップ削減

## 2. 用語定義

- **遊びマスタ（Play）**：遊び1種の定義データ。`PLAYS` 配列に格納。
- **実行可否（Availability）**：プレイヤーの現状況で遊びが実行できるかどうかのフラグと理由。
- **ドック（Dock）**：画面下部の横スクロール可能なアイコン群。
- **プレビュー（Preview）**：画面上部に表示される、選択中の遊びのイメージ・詳細・獲得予測・コスト。
- **確定タップ（Confirm Tap）**：プレビューが出ている状態で再度同じアイコンをタップする操作。これで SPEC-003 に進む。

## 3. 入力（Input）

- プレイヤー状態（SPEC-001）
- 遊びマスタ（`timeCost`, `moneyCost`, `staminaCost`, `seasons`, `ageMin`, `ageMax`, `minFriends`, `friendBonusPerPerson`, `gain`, `unlockRequired`）

## 4. 出力（Output）

- 画面下部のアイコン行（ドック）
- 画面上部のプレビューエリア（イメージ＋詳細＋獲得予測＋ロック理由）
- 「遊ぶ」ボタン（プレビュー表示中に明示的に押すルート）

## 5. 挙動（Behavior）

### 5.1 ロック判定ルール（いずれか1つでも該当でロック）
| 条件 | ロック理由の表示 |
|---|---|
| `play.unlockRequired` が true かつ `player.unlockedPlays` に未登録 | 非表示（ドックに出さない） |
| `play.seasons` に現在の季節が含まれない | 「○季は季節外」 |
| `player.age < play.ageMin` | 「○歳以上が必要」 |
| `player.age > play.ageMax` | 「○歳まで」 |
| `player.money < play.moneyCost` | 「所持金不足 (¥○必要)」 |
| `player.spareHours < play.timeCost` | 「時間不足」 |
| `player.stamina < play.staminaCost` | 「体力不足」 |
| `player.friends < play.minFriends` | 「友人○人以上必要」（SPEC-007） |

### 5.2 画面レイアウト（v2：起床ヘッダー付き）

#### 5.2.1 起床ヘッダー（Wakeup Header）の表示条件
- 「今日初めて S2 を表示するとき」＝ `player.coreTimeDoneToday === false` かつ `player.dailyPlays === 0` のときは **フル表示**（大きめ挨拶＋体力ゲージ＋時計円盤）
- それ以外（すでに遊んだ後の S2 再訪や、コアタイム後の夜の遊び選択画面）は **圧縮表示**（小さな時計円盤＋コンディションひとこと）
- どちらの場合も **ドック・プレビュー・固定フッター** は共通
- 「今日を始める」ボタンは **廃止**

### 5.3 インタラクションフロー
1. 画面遷移直後：**プレビューは空状態**（「遊びを選んでください」のメッセージ）
2. プレイヤーがドックのアイコンをタップ → プレビューが該当の遊びで埋まる
3. 別のアイコンをタップ → プレビューが切り替わる
4. **同じアイコンを再度タップ or フッターの「遊ぶ」ボタンを押す** → `confirmPlay()` で S3 へ遷移
5. ロック中のアイコン：プレビューに「⚠ ロック理由」を表示し、「遊ぶ」ボタンは disabled

### 5.4 アイコン行（ドック）の表示ルール
- **実行可能な遊びを左**、**ロック中を右** に並べる
- ロック中のアイコンは **グレー＋南京錠バッジ** 🔒
- 横スクロール可能（`overflow-x: auto`, `-webkit-overflow-scrolling: touch`）
- 1アイコンサイズ 48×48 px、間隔 8 px
- 選択中のアイコンは **リング（outline）で強調表示**

### 5.5 並び順詳細
ドック内は以下の優先度順：
1. 実行可能 + 今日未プレイ（最優先）
2. 実行可能 + 今日プレイ済み
3. ロック中（時間不足 / 体力不足）
4. ロック中（年齢・季節・所持金）
5. ロック中（友人不足）
6. 隠し（`unlockRequired` + 未解禁）は **表示しない**

### 5.6 友人数の表示（SPEC-007連動）
- `minFriends` が設定されている遊びは、プレビューに「👥 要友人 X人以上（1人ごと経験値+N）」を併記。

### 5.7 プレビューの「遊ぶ」ボタン
- フッターに **3つ目のボタン** として固定表示。
- プレビュー未選択：disabled（薄く）
- 選択中がロック：disabled（代わりにロック理由を表示）
- 実行可能：primary カラーで有効
- ラベル：「🎮 遊ぶ（-X h）」の形式で時間コストを明示

### 5.8 アクセシビリティ
- 各アイコンに `aria-label="砂場遊び, 1時間, 身体+10"` を付与（プロトは最低限）
- プレビューは `aria-live="polite"` でスクリーンリーダーに通知

## 6. UIへの反映

- 画面：S2 遊びを選ぶ（`#screen-choose`）
- 画面設計書：`docs/screen-design.md` §2.2（アイコン行ドック型に更新）

## 7. 想定される利用シナリオ

- 2歳・夏・朝：ドックに「🏖 🧱 🛝 📖 🦗」等が並ぶ。🏖 砂場遊びをタップ → プレビューに詳細表示 → 「遊ぶ」で S3。
- プレイ中に `learn_slide` を発見して帰宅 → ドックに 🎢 が追加で出現する。
- 所持金0円で 🏊 プールをタップ → プレビューに「⚠ 所持金不足 (¥300必要)」→ 「遊ぶ」は disabled。

## 8. 例外・エッジケース

- ドックが空（候補が全てロック）：ドック全体の代わりに「今できる遊びがありません」を表示し、「1h休む」「今日おわる」のみ使える。
- 選択中アイコンが途中で解禁／ロックになるケース（例：時間経過で時間不足に）→ プレビューを再評価して disabled に切り替え。
