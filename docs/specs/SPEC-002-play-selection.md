## 1. 目的

プレイヤーの現在状態（年齢・季節・所持金・友人数・体力・余剰時間）に応じて、  
「今、実行できる遊び / できない遊び」を明示し、意思決定の場を提供する。

UI は **画面下部の横スクロールできるアイコン行（ドック）** 形式を採用し、  
**アイコンタップ → 上部にイメージ・詳細表示 → 再タップで確定** の 2段タップで選ぶ。  
スマホ片手操作を前提とし、親指の届く範囲にアイコンを集めることで快適性を担保する。

> 遊びマスタ `PLAYS` は **SPEC-028** により `data/plays.json` から動的に読み込まれる。レコード追加だけで新しい遊びを増やせる。チュートリアル期間中は SPEC-026 §5.2 の段階解禁ルールで初期ドックが絞られる。

### 1.1 起床画面（S1）との統合（v2）

旧フローでは `S1 起床 → [今日を始める] → S2 遊びを選ぶ` と 1 タップ余分に要していたが、  
**1日の最初の遊び画面に起床時のコンディション表示を埋め込み、S1 と S2 を 1 画面に合体** させる。
- 起床時のメッセージ・時計円盤・体力/リズムゲージを **S2 の上部に常時表示**
- 「今日を始める」ボタンは廃止。初期画面は直接 S2（朝の遊び選択画面）になる
- これにより 1日あたり 1 タップ削減

## 2. 用語定義

- **遊びマスタ（Play）**：遊び1種の定義データ。`PLAYS` 配列に格納。
- **実行可否（Availability）**：プレイヤーの現状況で遊びが実行できるかどうかのフラグと理由。
- **ドック（Dock）**：画面下部の横スクロール可能なアイコン群。
- **プレビュー（Preview）**：画面上部に表示される、選択中の遊びのイメージ・詳細・獲得予測・コスト。
- **確定タップ（Confirm Tap）**：プレビューが出ている状態で再度同じアイコンをタップする操作。これで SPEC-003 に進む。

## 3. 入力（Input）

- プレイヤー状態（SPEC-001）
- 遊びマスタ（`timeCost`, `moneyCost`, `staminaCost`, `seasons`, `ageMin`, `ageMax`, `minFriends`, `friendBonusPerPerson`, `gain`, `unlockRequired`）

## 4. 出力（Output）

- 画面下部のアイコン行（ドック）
- 画面上部のプレビューエリア（イメージ＋詳細＋獲得予測＋ロック理由）
- 「遊ぶ」ボタン（プレビュー表示中に明示的に押すルート）

## 5. 挙動（Behavior）

### 5.1 ロック判定ルール（v3：絞り込みを強化）

**v3 方針：遊び選択画面には「今できる遊び」だけを並べる。** 「今できない遊び」はドックから除外し、遊びツリー画面（SPEC-023）で確認できるようにする。

| 条件 | ドック表示 | プレビュー表示 |
|---|---|---|
| `play.unlockRequired` が true かつ `player.unlockedPlays` に未登録 | 非表示 | — |
| `play.seasons` に現在の季節が含まれない | **非表示（v3）** | — |
| `player.age < play.ageMin` / `age > play.ageMax` | **非表示（v3）** | — |
| `player.money < play.moneyCost` | **非表示（v3）** | — |
| `player.spareHours < play.timeCost` | **非表示（v3）** | — |
| `player.friends < play.minFriends` | **非表示（v3）** | — |
| `player.stamina < play.staminaCost` | **表示（遊べる、§5.9 低体力プレイ）** | 経験値減衰警告 |
| 全条件クリア | 表示（通常） | 通常プレビュー |

※ 「非表示」は `dock.hidden` と同義で、アイコン自体がドックに描画されない。
※ 体力不足だけは唯一の例外で、**子どもらしい無茶** として許容する。詳細は §5.9。

### 5.1.1 ドックの最右端：遊びツリーアイコン（v3）

SPEC-023 の導入に合わせ、**ドックの一番右に固定で 🌳 遊びツリーアイコン** を配置する。
- ロック表現は不要（常に有効）
- タップで S7 遊びツリー画面へ遷移
- これにより、「今できない遊びはここで確認できる」導線が常に可視化される

### 5.1.2 遊びツリーアイコンの視覚差別化（v4 追加）

遊びツリーアイコンは **「遊び本体のアイコン」ではなく「画面遷移用のナビゲーションアイコン」** であることを一目で伝える必要がある。  
プレイヤーが「ただの遊びアイコン」と誤認しないよう、以下で明確に差別化する：

| 要素 | 遊びアイコン（通常） | 遊びツリーアイコン（ナビ） |
|---|---|---|
| 背景色 | 白カード `var(--card-bg)` | **濃い緑 `var(--nav-tree-bg, #3d6b4a)`** |
| 文字色 | なし（絵文字のみ） | 白（アイコン＋ラベル文字） |
| ボーダー | 薄いベージュ | なし or 濃い緑の濃淡 |
| 形状 | 52×52px の正方形・角丸 14px | **ピル型（52×52px、角丸 26px 相当 or 左右 24px）** |
| 区切り | — | 他の遊びアイコンとの間に **左側に薄い縦線 or 余白** で区切り |
| ラベル | なし | 「ツリー」などのミニテキストを併記（2 行構成）|
| 効果 | — | 軽く浮いたシャドウで「ボタン感」を強調 |

これにより、プレイヤーは直感的に「これは他とは違うボタン＝画面遷移する」と理解できる。

実装上の CSS クラス：
- `.dock-icon` …… 遊び用（既存）
- `.dock-icon.dock-tree` …… 遊びツリー専用（差別化スタイル）

本仕様は同一コンポーネントをクラス修飾子で差別化する方針。将来的にナビ系アイコンが増えた場合は `.dock-nav` 共通クラスに格上げする。

#### 5.2.1 起床ヘッダー（Wakeup Header）の表示条件
- 「今日初めて S2 を表示するとき」＝ `player.coreTimeDoneToday === false` かつ `player.dailyPlays === 0` のときは **フル表示**（大きめ挨拶＋体力ゲージ＋時計円盤）
- それ以外（すでに遊んだ後の S2 再訪や、コアタイム後の夜の遊び選択画面）は **圧縮表示**（小さな時計円盤＋コンディションひとこと）
- どちらの場合も **ドック・プレビュー・固定フッター** は共通
- 「今日を始める」ボタンは **廃止**

### 5.3 インタラクションフロー
1. 画面遷移直後：**プレビューは空状態**（「遊びを選んでください」のメッセージ）
2. プレイヤーがドックのアイコンをタップ → プレビューが該当の遊びで埋まる
3. 別のアイコンをタップ → プレビューが切り替わる
4. **同じアイコンを再度タップ or フッターの「遊ぶ」ボタンを押す** → `confirmPlay()` で S3 へ遷移
5. ロック中のアイコン：プレビューに「⚠ ロック理由」を表示し、「遊ぶ」ボタンは disabled

### 5.3.1 プレビュー表示時の起床ヘッダー置換（v3 追加）

**「起床ヘッダーと並列表示」ではなく、「プレビュー領域が起床ヘッダーを置換する」** 動作にする。  
スクロールを発生させず、スマホ片手操作で常に親指の近くに「遊ぶ」ボタンがあることを保証する。

| プレイヤーの状態 | 上部領域の表示 |
|---|---|
| まだ誰も選んでいない（初回 or 別のアイコン選択解除後） | **起床ヘッダー**（full or compact モード、SPEC-002 §5.2.1） |
| 何かのアイコンをタップ中（`selectedPlayId !== null`） | **プレビュー**（アイコン、タイトル、フレーバー、コスト、獲得予測、ロック理由） |
| アイコンから外をタップ／画面遷移 | 起床ヘッダーに戻る |

実装方針：
- 起床ヘッダー（`#wakeup-header`）とプレビュー（`#preview` + `#preview-placeholder`）を **排他表示** にする。
- `selectPlay()` 冒頭で `wakeup-header.hidden = true`、プレビューを表示。
- `renderChooseScreen()` では `wakeup-header` を再表示し、プレビューをプレースホルダーに戻す。
- プレースホルダー（「下のアイコンから…」）は、起床ヘッダーが出ているときは **不要** なため非表示にする。

これにより、2歳プレイヤーの iPhone SE（375×667）でも「起床挨拶 → アイコンタップ → コスト/獲得 → 遊ぶ」がスクロールなしで完結する。

### 5.4 アイコン行（ドック）の表示ルール
- **実行可能な遊びを左**、**ロック中を右** に並べる
- ロック中のアイコンは **グレー＋南京錠バッジ** 🔒
- 横スクロール可能（`overflow-x: auto`, `-webkit-overflow-scrolling: touch`）
- 1アイコンサイズ 48×48 px、間隔 8 px
- 選択中のアイコンは **リング（outline）で強調表示**

### 5.5 並び順詳細
ドック内は以下の優先度順：
1. 実行可能 + 今日未プレイ（最優先）
2. 実行可能 + 今日プレイ済み
3. ロック中（時間不足 / 体力不足）
4. ロック中（年齢・季節・所持金）
5. ロック中（友人不足）
6. 隠し（`unlockRequired` + 未解禁）は **表示しない**

### 5.6 友人数の表示（SPEC-007連動）
- `minFriends` が設定されている遊びは、プレビューに「👥 要友人 X人以上（1人ごと経験値+N）」を併記。

### 5.9 低リソースプレイ（v3 追加 → v4 拡張：時間不足も吸収）

子どもは自分の体力・時間を勘定せずにお構いなく遊ぶリアリティを再現するため、**体力不足 or 時間不足でも遊びを開始できる** 例外を設ける。  
ただし得られる経験値は **体力残量** と **時間残量** の比率の小さい方に応じて減衰する。

v4 改訂の背景（バグ報告）：
- 旧仕様では「時間不足」を `isHidden` としてドックから完全に消していたため、**余剰時間が残り 0.5h の状態では全ての遊びが消えて『今できる遊びがありません』と表示される** 問題が発生していた
- プレイヤー報告「体力 1 だと選べなくなる」は、実体は「体力 1 かつ余剰時間ほぼ 0 の状況」。時間も低リソースプレイの対象に含めることで解決する

#### 5.9.1 ドック表示と経験値減衰倍率（v4）
```
staminaMult = min(1.0, player.stamina     / play.staminaCost)    // 体力充足率
timeMult    = min(1.0, player.spareHours  / play.timeCost)       // 時間充足率
multiplier  = min(staminaMult, timeMult)                          // 不足している方に従う
```
- multiplier 0 の場合も遊び自体はできるが、原体験経験値・スキル経験値・ジョウネツはすべて 0 になる（＝泣きながら遊ぶ）
- 実体力・実時間はプレイ後にそれぞれ 0 にクランプ（時間 → そのまま夜へ、体力 → SPEC-019 強制睡眠）

#### 5.9.1a 例外：余剰時間が完全ゼロ（`player.spareHours <= 0`）のとき
- 時間比がゼロのままでは意味のある遊びにならないため、**全ての遊びは isHidden でドックから消す**
- プレイヤーは「今日おわる」or コアタイムへの遷移に誘導される
- これは「朝の遊び → 保育園 → 夜の遊び → 就寝」の 1 日サイクル（SPEC-020）に従う

#### 5.9.2 UI 表示
- プレビュー画面で **multiplier < 1.0** の場合、赤文字で `⚠ 体力不足 / 時間不足：経験値 X%` を表示
- ドックアイコンには `.low-stamina` クラスを付与（プロトでは △ バッジ相当の薄い警告枠）
- 「遊ぶ」ボタンは有効（子どもらしさを尊重）

#### 5.9.3 計算の適用先（SPEC-003 と連携）
`finalizePlay()` 内の経験値系すべてに `lowStaminaMultiplier` を乗せる：
- 素養 5 カテゴリ：✅ 減衰
- 友人数ボーナス：✅ 減衰
- ジョウネツ獲得：✅ 減衰（floor）
- スキル経験値：✅ 減衰
- イベント効果：❌ 減衰なし（運の要素）

#### 5.9.4 関数名について
実装上は `lowStaminaMultiplier()` / `isLowStamina` フラグを **そのまま流用**する（互換維持）。v4 以降は「低リソース」の意としてドキュメント側で読み替える。

#### 5.9.5 テスト観点
- stamina=1, staminaCost=4 → 倍率 0.25（25%）、遊び選択可
- stamina=4, staminaCost=4, spareHours=0.5, timeCost=2 → 倍率 0.25（時間不足側）、遊び選択可
- stamina=1, staminaCost=4, spareHours=0.5, timeCost=2 → 倍率 0.25（どちらでも同じ）、遊び選択可
- spareHours=0 → ドック全滅（`isHidden`、`今日の遊びは終わった`）
- stamina=0, staminaCost=4 → 倍率 0、選択はできるが経験値 0

### 5.7 プレビューの「遊ぶ」ボタン
- フッターに **3つ目のボタン** として固定表示。
- プレビュー未選択：disabled（薄く）
- 選択中がロック：disabled（代わりにロック理由を表示）
- 実行可能：primary カラーで有効
- ラベル：「🎮 遊ぶ（-X h）」の形式で時間コストを明示

### 5.8 アクセシビリティ
- 各アイコンに `aria-label="砂場遊び, 1時間, 身体+10"` を付与（プロトは最低限）
- プレビューは `aria-live="polite"` でスクリーンリーダーに通知

## 7. 想定される利用シナリオ

- 2歳・夏・朝：ドックに「🏖 🧱 🛝 📖 🦗」等が並ぶ。🏖 砂場遊びをタップ → プレビューに詳細表示 → 「遊ぶ」で S3。
- プレイ中に `learn_slide` を発見して帰宅 → ドックに 🎢 が追加で出現する。
- 所持金0円で 🏊 プールをタップ → プレビューに「⚠ 所持金不足 (¥300必要)」→ 「遊ぶ」は disabled。

## 8. 例外・エッジケース

- ドックが空（候補が全てロック）：ドック全体の代わりに「今できる遊びがありません」を表示し、「1h休む」「今日おわる」のみ使える。
- 選択中アイコンが途中で解禁／ロックになるケース（例：時間経過で時間不足に）→ プレビューを再評価して disabled に切り替え。
