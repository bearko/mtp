> 併存関係：**原体験Lv（本仕様）は人生全体の 5 軸評価**。**スキル（SPEC-024）はカテゴリごとの熟練度**。両者は混ぜずに併存させる。

## 1. 目的
遊びを通じて「身体系・創作系・探求系・交流系・競技系」の経験値を蓄積し、  
Lv アップによって **新しい遊びの解禁** と **没頭感** を演出する。

## 2. 用語定義
- **原体験カテゴリ**：5つのカテゴリ（physical / creative / explore / social / competitive）。
- **経験値（exp）**：カテゴリごとに蓄積される数値。
- **レベル（Lv）**：経験値から計算される段階。

## 3. 入力（Input）
- 遊びの結果（SPEC-003）
- ランダムイベントの効果（SPEC-004）

## 4. 出力（Output）
- `player.exp` の更新。
- 画面上の差分表示（Lv X → Lv Y）。

## 5. 挙動（Behavior）
### 5.1 Lv算出式
```
Lv = floor(sqrt(exp / 10))
```
- exp=10 で Lv1、exp=40 で Lv2、exp=90 で Lv3、exp=160 で Lv4 …
- Lv が上がるほど、次のLvに必要な経験値が大きくなる曲線。

### 5.2 経験値加算フロー
1. 遊びの `gain` による加算。
2. SPEC-007 による友人数ボーナスの加算（主カテゴリへ）。
3. SPEC-004 によるイベント効果の加算。

### 5.3 差分表示
- Lv アップした場合：`Lv3 → Lv4 ⬆ Lv UP` の表示を行う。
- Lv 据え置きの場合：`Lv3 (+5 exp)` の表示。

## 6. UIへの反映
- 統合画面（S3の結果フェーズ）で差分を表示。
- 画面設計書：`docs/screens/SCREEN-DESIGN.md` §2.3（結果フェーズ）

## 7. 想定される利用シナリオ
- 砂場遊び（身体+10 / 創作+5）を4回実施 → 身体exp=40 → 身体Lv2到達。

## 8. 例外・エッジケース
- 経験値は負にならない。
- 上限 Lv は 99（exp 98010）。到達後は exp のみ加算される。
