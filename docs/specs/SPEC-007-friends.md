# SPEC-007: 友人数

| 項目 | 内容 |
|---|---|
| 仕様ID | SPEC-007 |
| 機能名 | 友人数 |
| 対応ファイル | `prototype/game.js` の `isPlayAvailable`, `applyGain`, EVENTS |
| 関連仕様 | SPEC-002, SPEC-003, SPEC-004 |
| ステータス | Active |
| 最終更新 | 2026-04-18 |

## 1. 目的
「一人遊び vs みんなで遊ぶ」の選択軸を用意し、交流系の遊びを成立させる前提条件を定義する。  
友人数は遊びの解禁条件 / 経験値倍率 / ジョウネツ補正に影響する。

## 2. 用語定義
- **友人数（friends）**：現在のプレイヤーの友人の人数。
- **最小必要友人数（minFriends）**：遊びの解禁条件。
- **友人経験値ボーナス（friendBonusPerPerson）**：友人数1人ごとの経験値加算量。

## 3. 入力（Input）
- プレイヤーの現在の友人数。
- 遊びマスタの `minFriends`, `friendBonusPerPerson`。
- ランダムイベント「新しい友達ができた」の効果（SPEC-004）。

## 4. 出力（Output）
- 遊びのロック判定結果（SPEC-002）。
- 獲得経験値の加算（SPEC-003）。
- 友人数の増減。

## 5. 挙動（Behavior）

### 5.1 解禁条件
| 遊び | `minFriends` |
|---|---|
| 鬼ごっこ | 1 |
| ごっこ遊び | 1 |
| （将来）サッカー・野球 | 4 |
| （将来）ボードゲーム会 | 3 |
| （将来）飲み会 | 2 |

### 5.2 経験値ボーナス
- 遊びに `friendBonusPerPerson` が設定されている場合、以下のルールで主カテゴリに加算する。
  ```
  bonus = friendBonusPerPerson × player.friends
  gain[主カテゴリ] += bonus
  ```
- 主カテゴリ：`play.gain` のうち最大値を持つカテゴリ（同値なら先頭）。

### 5.3 友人数の増減
- 増加：ランダムイベント「新しい友達ができた」で +1。
- 将来的な予定：入学・就職などライフイベントでの増加、引っ越し等での減少。

## 6. UIへの反映
- 遊びカードに「👥 要友人 X人以上」と「1人ごと経験値+N」を明記（SPEC-002）。
- HUDに友人数（👥 アイコン）を常時表示。

## 7. 想定される利用シナリオ
- 友人2人で鬼ごっこ（身体+15, 交流+10, 友人ボーナス+3×2=6）→ 交流+16, 身体+15 を獲得。
- 友人0人で鬼ごっこを選択すると「友人1人以上必要」でロック。

## 8. 例外・エッジケース
- 友人数が0でも実行できる遊び（一人遊び）は、`minFriends` を未指定にする。
- 友人数は負にならない。
