## 0. 画面一覧とコアループ

### 設計方針（タップ数最小化）

旧フロー（8画面）は遷移が多すぎたため、**遊び終了／パラメーター反映／時間経過** の3画面を
**S3「遊びの描写」** に統合し、全体を **5画面 + 1イベント画面** に削減した。

旧：`起床 → 選ぶ → 描写 → イベント → 終了 → 反映 → 時間経過 → 就寝 → 翌日`（8画面）
新：`起床 → 選ぶ → 描写(結果含む) → (イベント) → 選ぶ → 就寝 → 翌日`（5画面＋1）

### 画面ID一覧（新）
| ID | 画面名 | 主目的 | 対応仕様 |
|---|---|---|---|
| S1 | 起床 | 1日の開始、今日の状態表示 | SPEC-001, SPEC-006 |
| **S6** | **コアタイム** | **学び/仕事の時間を消化、発見で遊びを解禁** | **SPEC-010, SPEC-011〜018** |
| S2 | 遊びを選ぶ | 余剰時間の投資先を選択 | SPEC-002, SPEC-007 |
| S3 | 遊びの描写（結果統合） | 演出 → 獲得結果 → 時間経過 を1画面で | SPEC-003, SPEC-005, SPEC-007 |
| S4 | ランダムイベント | 遊び中の突発的な出来事（確率35%） | SPEC-004 |
| S5 | 就寝 | 就寝時刻の選択。生活リズムに影響 | SPEC-006, SPEC-008 |

## 1. グローバル仕様（SPEC-009 準拠）

### 1.1 画面共通HUD（上部固定）
全画面の上部に以下を **スクロールしても常に** 表示する。

```
┌───────────────────────────────────────────┐
│ 5歳 / 3日目(夏) 06:30                     │
│ ❤80 🌙90 💰0 ⏳6h 👥2 🔥0                │
└───────────────────────────────────────────┘
```

### 1.2 画面下部ボタンエリア（固定・常に表示）
- **画面解像度に依存せず常に画面内に表示される**（SPEC-009 参照）。
- 高さ最低 52px（タップしやすさ優先）。
- ボタンは最大2つまで横並び。

### 1.3 画面レイアウト三段構造
```
[固定HUD]
[Scene（スクロール可）]
[固定Actions]
```
詳細は SPEC-009 参照。

### 1.4 画面遷移アニメーション
- フェード + ほんの少しの上方向からのスライド（220ms）。
- S3→S1 のみ（翌朝遷移）は明度変化を伴う遷移。

### S6. コアタイム画面（学びごと／仕事）

対応仕様：SPEC-010（フレームワーク）、SPEC-011〜SPEC-018（各ライフステージ）

**現在の実装対象：SPEC-011 保育園（1〜3歳）のみ。**
他のライフステージは Draft 仕様書で定義されているが未実装。

#### 2.6.1 目的
- 朝起きた後、該当ライフステージのコアタイム（学校・仕事）を自動消化する。
- コアタイム中の発見で新しい遊びを解禁し、経験値・友人を獲得する。
- 時刻を `coreTime.endHour` に進める。

#### 2.6.2 ワイヤーフレーム（保育園バージョン / SPEC-011）

```
┌─────────────────────────┐
│  [共通HUD]              │← 固定

│                         │
│  09:00 登園             │
│  10:00 朝の会           │
│  12:00 給食             │
│  13:00 公園遊び         │
│  16:00 降園             │
│                         │
│ ┌─ 発見！ ───────────┐ │
│ │ 🎢 大きな滑り台の   │ │
│ │   滑り方を教わった！│ │
│ │ → 大きな滑り台 解禁 │ │
│ └─────────────────────┘ │
│                         │
│ ┌─ 獲得 ─────────────┐ │
│ │ 身体 +8 / 社会 +3  │ │
│ │ 体力 80→65         │ │
│ │ 時刻 06:30→16:00   │ │

```

#### 2.6.3 IPO
| 区分 | 内容 |
|---|---|
| Input | プレイヤー状態（年齢・日付・既存解禁遊び） |
| Process | ①ライフステージ判定、②コアタイム消化、③発見抽選、④解禁・経験値反映 |
| Output | 画面への発見・獲得値表示、player state 更新、時刻 endHour へ進行 |

#### 2.6.4 遷移
- **遷移元**：S1 起床（「今日を始める」押下 かつ age に coreTime がある場合）
- **遷移先**：S2 遊びを選ぶ（[家に帰る] 押下）
