## 1. グローバル仕様

### 1.1 画面共通HUD（上部ステータスバー）
全画面の上部に以下を固定表示する。

```
┌───────────────────────────────────────────┐
│ 5歳 / 3日目(夏) | 体力:80 | リズム:90     │
│ 所持金:¥0  | 余剰時間:6h  | 友人:2       │
└───────────────────────────────────────────┘
```

### 1.2 画面下部ボタンエリア
- 高さ 64px 以上（タップしやすさ優先）
- ボタンは最大2つまで横並び
- Primary=濃い色 / Secondary=薄い色

### 1.3 画面間遷移ルール
- 画面遷移は **スライドアニメーション（右→左）** を基本とする
- S7→S1 の遷移時のみ **フェード + 朝焼けエフェクト**

## 2. 各画面仕様

---

### S1. 起床画面

#### 2.1.1 目的
- 新しい1日の開始を演出
- プレイヤーの現在状態（年齢・体力・生活リズム・季節）を確認させる
- 「今日を始める」アクションの明示的なタップ起点

#### 2.1.2 ワイヤーフレーム

```
┌─────────────────────────┐
│  [共通HUD]              │
├─────────────────────────┤
│                         │
│     🌅 イラスト         │
│   （季節・時間で変化）  │
│                         │
│    おはよう！           │
│    あなたは 5歳         │
│    夏の朝 6:30          │
│                         │
│  今日のコンディション： │
│  ● 体力 80 / 100        │
│  ● 生活リズム 90 / 100  │
│  ● 友人 2 人            │
│                         │
│  今日のひとこと：       │
│ 「よく眠れた。元気だ！」│
│                         │
├─────────────────────────┤
│   [ 今日を始める ]      │
└─────────────────────────┘
```

#### 2.1.3 IPO（Input / Process / Output）
| 区分 | 内容 |
|---|---|
| Input | 前日の終了時ステータス（体力、生活リズム、所持金、友人数、年齢、日付） |
| Process | ①日付+1、②季節の再計算、③起床時刻を生活リズムから計算、④体力回復量を計算、⑤コンディションメッセージを選択 |
| Output | 現在年齢 / 日付 / 季節 / 起床時刻 / コンディション文 / 画面描画 |

#### 2.1.4 遷移
- **遷移元**：S7 時間経過（就寝後の翌朝）、アプリ起動時（初回は年齢1からスタート）
- **遷移先**：S2 遊びを選ぶ（[今日を始める]ボタン押下）

#### 2.1.5 イベント/ロジック
- 生活リズムが 40 未満：起床時刻が遅れる（+1〜3h）。メッセージ「寝坊した…」
- 生活リズムが 80 以上：体力回復 +10 のボーナス
- 体力が 20 未満：起床時にランダムで「体調不良」発生 → 当日の余剰時間 -2h

### S2. 遊びを選ぶ画面

#### 2.2.1 目的
- 現在のライフステージ・季節・所持リソースで **実行可能な遊び** を提示
- プレイヤーの能動的選択の場

#### 2.2.2 ワイヤーフレーム

```
┌─────────────────────────┐
│  [共通HUD]              │
├─────────────────────────┤
│ 何して遊ぶ？ (残6h)     │
│                         │
│ ┌─────────────────────┐ │
│ │ 🏖 砂場遊び         │ │
│ │ 1h / ¥0             │ │
│ │ 身体+10             │ │
│ │         [ 遊ぶ ]    │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 🤼 鬼ごっこ         │ │
│ │ 2h / ¥0             │ │
│ │ 身体+15 交流+10     │ │
│ │ 要友人1人以上 ✅   │ │
│ │         [ 遊ぶ ]    │ │
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 🏊 プール           │ │
│ │ 3h / ¥300           │ │
│ │ ⚠ 所持金不足       │ │
│ │       [ ロック ]    │ │
│ └─────────────────────┘ │
│ ...                     │
├─────────────────────────┤
│  [何もしない(1h休む)]   │
└─────────────────────────┘
```

#### 2.2.3 IPO
| 区分 | 内容 |
|---|---|
| Input | 現在の年齢、季節、体力、所持金、友人数、原体験Lv、残り余剰時間、遊びマスタJSON |
| Process | ①年齢・季節・身長で候補を絞り込み、②所持金/友人数/体力で可否判定、③不足要素は「ロック」表示、④3〜6件をリストに並べる |
| Output | 遊び候補カードのリスト描画、選択可否、選択された遊びID |

#### 2.2.4 遷移
- **遷移元**：S1 起床 / S7 時間経過（余剰時間残あり）
- **遷移先**：
  - S3 遊びの描写（遊びカードの「遊ぶ」を押下）
  - S7 時間経過（「何もしない」選択＝1h進む）

#### 2.2.5 イベント/ロジック
- 所持金不足／友人数不足／体力不足／季節外の遊びは **ロック表示**（グレーアウト）
- 「遊ぶ」押下で選択した遊びIDを状態に保存

### S3. 遊びの描写画面

#### 2.3.1 目的
- 選んだ遊びを "体験している感" の演出で表現する
- 没入感と楽しさを表現

#### 2.3.2 ワイヤーフレーム

```
┌─────────────────────────┐
│  [共通HUD]              │
├─────────────────────────┤
│                         │
│     🏖 砂場遊び         │
│                         │
│   （中央イラスト）      │
│                         │
│  あなたは砂場で         │
│  山を作り始めた…        │
│                         │
│  ████████░░ 80%         │
│                         │
│  時間：残り 12分        │
│                         │
├─────────────────────────┤
│   [ スキップ ]          │
└─────────────────────────┘
```

#### 2.3.3 IPO
| 区分 | 内容 |
|---|---|
| Input | 選ばれた遊びID、遊びマスタ（所要時間、描写文セット） |
| Process | ①描写文をランダムに1〜3個選択、②プログレスバーを擬似的に進行、③一定確率でランダムイベント発火判定 |
| Output | 描写文、プログレスバー、演出終了通知、ランダムイベント発火フラグ |

#### 2.3.4 遷移
- **遷移元**：S2 遊びを選ぶ
- **遷移先**：
  - S4 ランダムイベント（イベント発火時、確率30%）
  - S5 遊び終了（イベント発火せず）
- **スキップボタン**：描写演出をスキップ（演出をカットして次画面へ）

### S4. ランダムイベント画面

#### 2.4.1 目的
- プレイヤーの選択に関わらず発生する **偶発性** の演出
- 「ゲームらしさ」と「人生の不条理さ」を表現

#### 2.4.2 ワイヤーフレーム

```
┌─────────────────────────┐
│  [共通HUD]              │
├─────────────────────────┤
│                         │
│   ⚡ ランダムイベント   │
│                         │
│   （イベント用イラスト）│
│                         │
│  「近所の子に話しかけ   │
│    られた！友達になった」│
│                         │
│   → 友人数 +1           │
│   → 交流 +5             │
│                         │
├─────────────────────────┤
│   [ なるほど ]          │
└─────────────────────────┘
```

#### 2.4.3 IPO
| 区分 | 内容 |
|---|---|
| Input | 現在年齢、遊びカテゴリ、プレイヤーステータス、運パラメータ |
| Process | ①イベント候補の絞り込み、②年齢フィルタ、③ポジ/ネガの重み付け抽選、④ステータス差分の決定 |
| Output | イベントテキスト、差分（友人数、体力、カネ、原体験pt など） |

#### 2.4.4 遷移
- **遷移元**：S3 遊びの描写
- **遷移先**：S5 遊び終了

#### 2.4.5 イベント例
- (+) 友達が増えた、 (+) お小遣いをもらった、 (+) スキル開眼
- (−) 転んでケガ、 (−) 道具を壊した、 (−) 天気が急変

#### 2.8.3 IPO
| 区分 | 内容 |
|---|---|

#### 2.8.4 遷移
- **遷移元**：S7 時間経過

- **夜更かし**：余剰時間 +1h 追加で遊べるが、翌日リズム -15 & 体力 -10
- **リズム60未満の状態で夜更かし**：**体調不良確率 +30%**（翌日の余剰時間がさらに減る）

## 3. データモデル（プロトタイプ用）

### 3.1 PlayerState
```ts
type PlayerState = {
  age: number;              // 年齢
  day: number;              // ゲーム内の通算日
  season: "spring" | "summer" | "autumn" | "winter";
  clockHour: number;        // 現在時刻(0-24)
  spareHours: number;       // 余剰時間(今日の残り)
  stamina: number;          // 体力 0-100
  bioRhythm: number;        // 生活リズム 0-100
  money: number;            // 所持金
  friends: number;          // 友人数
  passion: number;          // ジョウネツ
  exp: {
    physical: number;
    creative: number;
    explore: number;
    social: number;
    competitive: number;
  };
};
```

### 3.2 Play マスタ
```ts
type Play = {
  id: string;
  name: string;
  icon: string;               // 絵文字
  timeCost: number;           // h
  moneyCost: number;
  seasons?: Season[];         // 未指定=通年
  ageMin?: number;
  ageMax?: number;
  minFriends?: number;        // 必要友人数
  friendBonusPerPerson?: number; // 友人数に応じた経験値ボーナス
  gain: {
    physical?: number;
    creative?: number;
    explore?: number;
    social?: number;
    competitive?: number;
  };
  staminaCost?: number;
  descriptions: string[];     // S3 用
};
```
