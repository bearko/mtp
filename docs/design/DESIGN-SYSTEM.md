## 0. この文書の役割

本書は、ゲーム UI を増改築するときに参照する **見た目・言葉・構造の共通ルール**である。

- 画面設計書: 画面 ID、遷移、HUD、オーバーレイ構造を管理する
- 機能仕様書: 挙動、計算、表示条件を管理する
- **デザインシステム**: 色、余白、タイポグラフィ、コンポーネント、演出、文体を管理する

コード実装時は、対応する SPEC に加えて本書の該当章を参照する。

## 1. デザイン原則

### 1.1 ひと目で「今日どうする？」がわかる

プレイヤーが毎ターン判断したいことは次の 3 つに集約する。

1. **いまの状態**: 年齢、日付、場所、体力、残り時間
2. **できること**: 選べる遊び、行ける場所、進行中の目標
3. **選ぶとどうなるか**: 消費、伸びる素養、達成に近づく目標

UI はこの 3 つを常に近接させる。特に S2 では、選択肢と結果予測を離さない。

### 1.2 数値は「物語への入口」として扱う

体力、時間、素養値、スキル Lv は必要だが、数字だけを主役にしない。

- 数字は HUD・カード・影響リストで簡潔に表示する
- 数字が変わった理由を、短いエピソードや周囲の反応で補う
- 保育園・幼稚園期は順位、偏差値、上位率、ランキングを表示しない

### 1.3 幼少期はやわらかく、大人期は現実味を増す

ライフステージに応じて UI の言葉と密度を変える。

| ライフステージ | UI の感触 | 情報密度 | 主な表現 |
|---|---|---:|---|
| 保育園・幼稚園 | 絵本、連絡帳、親のまなざし | 低 | 「できたね」「だいすき」「おうち」 |
| 小学生 | 探検、図鑑、発見 | 中 | 「あたらしい」「チャレンジ」「ともだち」 |
| 中高生 | 自分らしさ、選択、競争の芽 | 中 | 「目標」「大会」「関係」 |
| 大学生以降 | 資源配分、人生設計 | 高 | 「時間」「カネ」「体力」「家族」 |

### 1.4 操作は片手で終わる

`SPEC-009` の固定 HUD / 固定 Actions を前提に、主要操作は下半分へ寄せる。

- 主要ボタンの高さは 52px 以上
- 一画面の主要アクションは原則 1 つ、補助アクションは 1-2 個まで
- 横並びボタンは最大 2 個を基本とする。3 個以上はアイコンドックに逃がす

## 2. ブランド人格

### 2.1 キーワード

- あたたかい
- なつかしい
- やさしい
- でも、人生の制約はちゃんとある
- 「がんばれ」より「見守る」

### 2.2 近い雰囲気

- 絵本の余白
- 保育園の連絡帳
- 家族アルバム
- 手帳・日記
- 公園や夕方の帰り道

### 2.3 避ける雰囲気

- ソーシャルゲームのランキング画面
- 金融アプリのようなダッシュボード
- RPG の装備最適化画面
- 赤字アラートだらけの管理画面
- 幼少期から競争を煽る UI

## 4. タイポグラフィ

### 4.1 フォント

既存 CSS と同様、端末標準フォントを優先する。

```css
font-family: -apple-system, BlinkMacSystemFont, "Hiragino Sans", "Noto Sans JP", "Segoe UI", sans-serif;
```

将来的にタイトルロゴのみ手書き風フォントを検討してよいが、本文には使わない。

### 4.2 サイズ

| 用途 | サイズ | 行間 | 備考 |
|---|---:|---:|---|
| 大見出し | 24px | 1.3 | S3/S10 画面タイトル |
| 中見出し | 18px | 1.4 | カード内見出し |
| 本文 | 15-16px | 1.7 | 読ませる文 |
| 補助文 | 12-14px | 1.5 | 日付、ラベル |
| HUD | 11-13px | 1.4-1.6 | 詰めすぎ注意 |
| ボタン | 16px | 1.2 | 太字 |

### 4.3 表記ルール

- 幼少期 UI では漢字を減らす: 「挑戦」より「ちょうせん」
- 大人期 UI では意味が明確なら漢字を使う: 「余剰時間」「所持金」
- 単位は短く: `8h` より、幼少期では `あと8時間`
- 重要な状態は絵文字 + 言葉で表示する: `🤩 絶好調`
- 絵文字単独で意味を完結させない

## 5. レイアウトと余白

### 5.1 画面基本構造

`SPEC-009` の構造を維持する。

```
.phone-frame
  .hud / banner / prelude
  .screen.active
    .scene
    .actions
```

### 5.2 余白スケール

| Token | 値 | 用途 |
|---|---:|---|
| `--space-1` | 4px | アイコンとラベル |
| `--space-2` | 8px | 小要素間 |
| `--space-3` | 12px | カード内余白 |
| `--space-4` | 16px | 画面左右余白 |
| `--space-5` | 20px | セクション間 |
| `--space-6` | 24px | 大きな区切り |

### 5.3 カード

既存 `.card` を基本コンポーネントとする。

- 角丸: 14px
- 枠線: 1px `--color-card-border`
- 影: 低く、紙が浮く程度
- カードタイトルは小さく控えめにし、内容を主役にする

### 5.4 情報密度

1 画面に置く主情報は最大 3 ブロック。

例: S10 日サマリ

1. 連絡帳
2. 今日の遊び
3. 成長差分

それ以上は折りたたみ、タブ、または次画面に逃がす。

## 6. コンポーネント

### 6.1 HUD

既存 `SPEC-034` の 3 エリア HUD を継続する。

| エリア | 役割 | 表示 |
|---|---|---|
| 左 | 人生上の現在地 | ライフステージ、年齢、残り時間 |
| 中央 | 今日の状態 | 日付、リズム、体力 |
| 右 | 行動環境 | カネ、友人、現在地 |

#### 改善方針

- 幼少期は金額表示を隠し、親・場所・体力を優先する
- 場所は短縮名だけでなく、タップ時に「ここでできること」を見せる余地を残す
- 体力ゲージは「できる / できない」の判断に直結するため、最も視認性を高くする

### 6.2 遊びドック

下部ドックは「何して遊ぶ？」の主操作。

#### 表示ルール

- アイコン + 短いラベルをセットにする
- 選択中はカードと同じアクセント色で囲む
- 選べない遊びは薄くするだけでなく、タップ時に理由を短く出す
- 遊びが多い場合はカテゴリタブではなく「場所」や「いまの目標に近い」で絞る

#### 推奨ラベル

| 悪い例 | 良い例 |
|---|---|
| `play_sandbox` | `砂場` |
| `reading` | `絵本` |
| `location_tree` | `できること` |
| `profile` | `自分` / `アルバム` |

### 6.3 素養カード

`SPEC-034`, `SPEC-035` の中核コンポーネント。

#### 役割

- 現在の成長状態を一覧する
- 遊び選択時に「何が伸びるか」を予告する
- 結果画面で「どれだけ伸びたか」を実感させる

#### 表示

```
[icon] [label] [grade] [+N] [value or before→after]
```

#### 原則

- グレードを大きく、実数値を小さくする
- `+N` は専用セルに置き、既存値に重ねない
- グレードアップは金色発光 + 短いリアクションを付ける
- 幼少期では `F` や `G` を失敗に見せない。色は灰色でも、文言で否定しない

### 6.4 目標・ミッションカード

`SPEC-050`, `SPEC-052`, `SPEC-055` と連動する。

#### 役割

- 今日の遊び選択に意味を与える
- 数値条件をストーリーに接続する
- 達成時に「できた」「認められた」「親が喜んだ」を回収する

#### 必須要素

- 目標名
- いまの状態
- 次にできること
- 達成条件
- 関係者の一言

#### 推奨文体

- `身体30以上` だけでなく `もう少し体を動かすと、挑戦できそう` と補足する
- `未達成` ではなく `いま練習中`
- `失敗` ではなく `まだむずかしかった`

### 6.5 イベントモーダル

`SPEC-056` の `#event-overlay` を基準とする。

#### 表示順

1. 大きな絵 / 絵文字
2. 何が起きたか
3. 情景説明
4. 影響
5. 誰かのコメント
6. 閉じるボタン

#### 影響の見せ方

- 数値差分は短く: `体力 -5`, `余剰時間 -2h`
- 理由は説明文とコメントで補う
- 悪いイベントでも「罰」ではなく「生活の一部」として見せる

### 6.6 連絡帳カード

`SPEC-027`, `SPEC-035` と連動する、本作らしさの核。

#### 役割

- 1 日を感情で閉じる
- プレイヤーを親・先生の目線に戻す
- 数字でなく「見守られている」実感を作る

#### 構成

```
📒 連絡帳
先生から:
  今日、○○ちゃんは...
おうちの人へ:
  ...
今日のひとこと:
  ...
```

#### 原則

- S10 ではファーストビューに置く
- 1 文を短くし、2-4 行で読める量にする
- 数値差分は本文に混ぜすぎない

### 6.7 プロファイル / アルバム

`SPEC-051` の「自分の宝箱」を基準とする。

#### タブ

| タブ | 目的 |
|---|---|
| 🎯 今やっている目標 | 次に何をしたいか |
| 🏆 できるようになったこと | 達成の記念 |
| 💖 だいすきな遊び | 自分らしさ |
| 📖 思い出のアルバム | 感情の振り返り |

#### 禁止

- グローバルランキング
- 他プレイヤー平均
- 詳細統計を主役にしたグラフ

## 7. アイコンとイラスト

### 7.1 現段階

プロトタイプでは絵文字を主要ビジュアルとして使う。

- 開発速度が速い
- スマホ上で意味が伝わりやすい
- 後からイラストに差し替えやすい

### 7.2 将来の方向性

- 絵本タッチ
- 輪郭はやわらかい
- 背景は淡い
- 表情を重視
- 遊びの瞬間より「周囲が喜ぶ瞬間」を描く

### 7.3 絵文字運用

- 1 コンポーネントに主絵文字は 1 つ
- HUD では意味補助に限定
- 重要な達成では `🎉` を使いすぎず、カード発光やセリフで表現する

## 8. モーション

### 8.1 基本

- 画面遷移: fade + 上方向 slide 220ms
- モーダル: pop 180-220ms
- 数値増加: count up 300-500ms
- グレードアップ: 600ms 程度の発光

### 8.2 原則

- 重要な変化だけ動かす
- 自動モードでは動きを短くし、ハイライトだけ丁寧に見せる
- 低体力・病気などのネガティブ状態で過度に派手な演出をしない

## 9. アクセシビリティ

### 9.1 視認性

- 本文 15px 以上、主要ボタン 16px 以上
- 色だけで意味を伝えない。`+10`, `-2h`, `できた` などテキストを併記
- 赤 / 緑の差分には `+` / `-` を必ず付ける

### 9.2 操作

- タップ領域は 48px 以上、推奨 52px
- 片手親指の範囲に主要操作を配置
- 連打が必要な UI を作らない

### 9.3 音なし前提

README の方針どおり、音を出さなくても理解できる UI にする。

- 成功は視覚と文言で伝える
- SE 前提のフィードバックにしない
- 通勤中でも読み切れる文量にする

## 10. コピーライティング

### 10.1 声のトーン

| 状況 | トーン |
|---|---|
| 成功 | 一緒に喜ぶ |
| 失敗 | まだ途中として見守る |
| 体力不足 | 休むことを自然に促す |
| 発見 | 小さく驚く |
| 選択 | プレイヤーの意思を尊重する |

### 10.2 推奨表現

- `できたね！`
- `もう少しでできそう`
- `今日はここまでにしよう`
- `おうちの人がうれしそう`
- `先生が見ていてくれた`
- `今ならちょうせんできるかも`

### 10.3 避ける表現

- `失敗`
- `効率が悪い`
- `最適解`
- `未達`
- `ランキング`
- `上位`
- `損`
- `周回効率`

大人期では制約や効率の概念を扱ってよいが、ゲーム全体の語り口は冷たくしない。

## 11. 画面別ガイド

### 11.1 S0 タイトル

目的: 世界観への入口。

改善余地:
- 「カネとジカンとジョウネツと」の核心コピーを小さく添える
- `はじめから` の下に「1歳から100歳まで、遊びを選ぶ人生へ」の短文を置く
- HUD は非表示を継続

### 11.2 S2 遊びを選ぶ

目的: 今日の投資先を決める。

改善余地:
- 遊びドックを「現在地でできる遊び」として明示
- 選択中の遊びカードに「何が伸びる」「何を消費する」「目標に近づく」をまとめる
- ミッションバナーとドックの関係を近づける

### 11.3 S3 遊びの描写

目的: 遊びを体験し、成長を実感する。

改善余地:
- 結果の前に 1 行だけ情景テキストを出す
- 素養カードの差分に、親・先生の短い反応を添える
- 同じ遊びでも複数フレーバーを出して反復感を減らす

### 11.4 S4 / Event Overlay

目的: 予期しない出来事を理解する。

改善余地:
- 将来的には S4 も `#event-overlay` へ統合し、画面種別を減らす
- 影響だけでなく「どう受け止めればいいか」のコメントを入れる

### 11.5 S10 連絡帳サマリ

目的: 1 日を感情で閉じる。

改善余地:
- 連絡帳を最上段に維持
- 「今日できたこと」を 1 つだけ大きく見せる
- 数値集計は折りたたみでもよい

### 11.6 S9 週末ハイライト

目的: 1 週間の意味づけ。

改善余地:
- 「今週いちばん心に残った日」を先頭に置く
- 遊び回数トップ 5 は「だいすきだった遊び」として表示
- 次週の小さな予告を 1 つ入れる

### 11.7 S17 プロファイル（Draft）

目的: 自分の宝箱。

改善余地:
- 実装時は数字より先に称号・思い出を見せる
- ドック上のラベルは `自分` より `アルバム` も検討する
