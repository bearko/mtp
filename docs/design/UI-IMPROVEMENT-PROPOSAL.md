# UI 改善提案 / UI Improvement Proposal

> 対象: 現行ブラウザプロトタイプ（`prototype/`）および将来のスマートフォン版。
> 参照: [`DESIGN-CHARTER.md`](./DESIGN-CHARTER.md), [`DESIGN-SYSTEM.md`](./DESIGN-SYSTEM.md), [`../screens/SCREEN-DESIGN.md`](../screens/SCREEN-DESIGN.md)
> 最終更新: 2026-04-26

---

## 1. 目的

現状のゲーム UI を、既存仕様・開発憲章・ゲームデザインレビューに照らして整理し、より直感的で分かりやすい UI へ向けた改善方針を提案する。

本提案は即時実装 SPEC ではなく、次に起こすべき SPEC / PR の優先順位を決めるための設計レビュー文書である。

---

## 2. 現状 UI の良い点

### 2.1 タップ数削減の方向性が明確

- S1 起床画面を S2 に統合し、`S2 -> S3 -> S2` の軽いループを作っている。
- 下部固定 Actions により、次の行動がスクロールなしで押せる。
- S3 結果画面ではステータスや時間カードを削り、素養カード中心に寄せている。

### 2.2 情報の置き場が定まり始めている

- HUD: 年齢・日付・残り時間・体力・場所。
- S2: 素養カードと遊び選択。
- S10: 連絡帳。
- S17（Draft）: 思い出・できるようになったこと。

この分担は、DESIGN-SYSTEM の「状態 / 選択 / 結果 / 記録」モデルと相性が良い。

### 2.3 幼少期らしい言葉への転換が進んでいる

SPEC-051 では「称号」を「できるようになったこと」、「通算記録」を「思い出のアルバム」と言い換えている。これはデザイン憲章の「数字よりエピソード」に合っている。

## 3. 現状 UI の主な課題

### 3.1 HUD が「読めるが、初見で意味が入りにくい」

現行 HUD は 3 カラムで多くの情報を収めているが、初見では以下の認知負荷がある。

| 課題 | 例 | 影響 |
|---|---|---|
| ラベルが短く意味を推測する必要がある | 「あと8時間」「👥2」「🏠自宅」 | 何が重要か判断しにくい |
| 幼少期と後年の資源が混在する | money は幼少期非表示だが、HUD 構造は同じ | 「カネ」が今の選択に関係するか分かりにくい |
| 現在状態と次の行動の関係が弱い | 体力が減る理由はプレビューまで見ないと分からない | 行動選択の判断が遅れる |

### 3.2 S2 の「何をすればよいか」が弱い

S2 は素養カード中心に整理された一方で、プレイヤーが最初に見るべき意図が不足している。

- 「今日は何を目指す日か」が常時見えない。
- mission-banner はあるが、優先行動に翻訳されていない。
- ドックの遊びアイコンは選択肢として並ぶが、「おすすめ」「今できる」「もう少しで達成」が視覚的に区別されにくい。

### 3.3 成長の実感が「数値差分」に寄りやすい

S3 / S10 では素養カード・スキル行で成長を示しているが、DESIGN-CHARTER が重視する 4 つの成長実感のうち、現在強いのは「できるようになった」の数値側である。

不足しやすいもの:

- 誰が認めてくれたか。
- 親・先生がどう喜んだか。
- その日の出来事が後から思い出になるか。

### 3.4 モーダル種別が増え、重要度が見分けにくい

現在は以下が存在する。

- `#event-overlay`
- `#interrupt-overlay`
- `#mission-prelude`
- `#mission-banner`
- `S-mission-modal`
- `S-attempt-prompt`

機能上は整理されているが、視覚言語が統一されないと「告知」「事件」「達成」「選択」が同じ強さに見える。

### 3.5 ドックの拡張余地が不足しつつある

遊び選択、ツリー、将来のプロフィール（SPEC-051）がドックに集まる。今後、場所・アルバム・設定が増えると、下部にアイコンが詰まり、片手操作の分かりやすさが落ちる。

## 4. 改善方針

### 方針 A: S2 を「選択画面」から「今日の作戦画面」にする

#### 提案

S2 の上部（HUD 直下 / 素養カード上）に `今日の焦点カード` を追加する。

```
┌──────────────────────────┐
│ きょうのめあて             │
│ 🧩 パズルの練習をすると、   │
│    9ピースに近づけそう      │
│ [絵本] [パズル4P] が近道    │
└──────────────────────────┘
```

#### 表示内容

- 進行中ミッションがある場合: 「次に役立つ遊び」を 1-2 件提示。
- ミッションがない場合: 「まだ少ない素養」「未発見の場所」「最近好きな遊び」から 1 件提示。
- 自動モード時: ダイジェストの見どころをここに集約。

#### 期待効果

- プレイヤーが「どのアイコンを押せばよいか」を迷いにくくなる。
- Challenge を、数値目標ではなく物語上の「めあて」として伝えられる。

#### 関連 SPEC 候補

- SPEC-050 ミッション
- SPEC-055 予告ヒント
- SPEC-034 S2 HUD

---

### 方針 B: 遊びアイコンに「意味のバッジ」を付ける

#### 提案

ドックまたは遊びカードに、選択理由を示す小さなバッジを付ける。

| バッジ | 意味 | 色 |
|---|---|---|
| `めあて` | 進行中ミッションに近づく | 黄色 |
| `もうすぐ` | 達成条件の不足を埋める | 紫 |
| `好き` | 累積回数が多い / だいすきな遊び | 桃 |
| `新しい` | 初回または最近解禁 | 緑 |
| `つかれる` | 体力不足に注意 | 赤ではなく淡い橙 |

#### 期待効果

選択肢の意味が直感的に分かる。特に幼児期の「比較しない」世界観を壊さず、行動の助けを出せる。

---

### 方針 C: S3 結果を「できた！ → 認められた → どう変わった」の順にする

#### 現状

結果は獲得素養・スキルの差分が中心。

#### 提案レイアウト

```
┌──────────────────────────┐
│ 🎉 できた！                │
│ ひとりで絵本を最後まで読んだ │
├──────────────────────────┤
│ 先生「さいごまで聞けたね」   │
│ 母「この本、好きなんだね」    │
├──────────────────────────┤
│ 🌱 そだったこと             │
│ 知性 +10 / 感性 +3          │
├──────────────────────────┤
│ 🏷 読書 Lv.1 を覚えた        │
└──────────────────────────┘
```

#### 期待効果

- 数字の前にエピソードを置くことで、成長が物語として伝わる。
- SPEC-048 の「4 つの成長実感」を S3 で直接回収できる。

---

### 方針 D: S10 連絡帳を「今日の結論」にする

#### 提案

S10 のファーストビューは、現在の連絡帳カードをさらに強め、以下の順で固定する。

1. 今日のひとこと（先生 / 親）
2. できるようになったこと
3. 親が喜んだこと / 褒められたこと
4. 数字で見る今日（折りたたみ）

#### 期待効果

1 日の終わりが「集計」ではなく「思い出」になる。自動モード・スキップで薄くなりがちな体験も、最後に読める。

---

### 方針 E: モーダルの視覚階層を 4 種に統一する

| 種別 | 役割 | 主色 | 例 |
|---|---|---|---|
| Notice | 告知 | Blue / Mist | チュートリアル、境界日 |
| Event | 事件・状態変化 | Amber / Peach | 感染症、遠出 |
| Mission | 物語目標 | Lavender / Gold | 発端、触媒、挑戦 |
| Growth | 達成・承認 | Gold / Leaf | できた、褒められた |

#### ルール

- `Growth` は最も強い演出（紙吹雪、発光）を許可する。
- `Event` は影響リストを必ず見せるが、怖くしすぎない。
- `Notice` は 1 タップで消える軽い見た目。
- `Mission` は会話カード中心で、選択の意思を明確にする。

---

### 方針 F: プロフィール導線は「自分」より「アルバム」を優先する

SPEC-051 ではドックに `📊 自分` を追加する案があるが、幼少期のトーンに合わせるなら、初期表示名は以下を推奨する。

- 表示名: `📖 アルバム`
- 画面内タブ: `目標 / できた / 好き / 思い出`
- 数字タブではなく、感情タブをファーストビューにする。

`📊` は意味が明確だが分析的なので、初期ステージでは `📖` の方がデザイン憲章と一致する。中高生以降に `自分` / `記録` へ表現を変えるのはあり。

## 6. 優先順位

### Tier 1: 次の UI 改善 PR 候補

1. S2 今日の焦点カード
2. 遊びアイコンの意味バッジ
3. モーダル種別カラーの統一

理由: 選択の迷いを減らし、既存のミッション / 予告ヒント / 遊び選択と接続しやすい。

### Tier 2: 体験価値を大きく上げる改善

1. S3 結果画面の「できた！」ファースト化
2. S10 連絡帳の感情ファースト化
3. プロフィール導線を `📖 アルバム` として実装

理由: DESIGN-CHARTER の核である「数字よりエピソード」を画面体験に反映できる。

### Tier 3: アセット・演出を伴う改善

1. タイトル画面背景のアルバム演出
2. Growth モーダルの紙吹雪・発光
3. 季節別背景・場所別背景

理由: 見た目の魅力は上がるが、先に情報設計を固める方が安全。

## 7. Why not

### 7.1 Why not: HUD にすべての情報を集約する

HUD が重くなり、初見の読み取り負荷が上がる。HUD は状態、S2 は選択理由、S10 / S17 は記録に分担する。

### 7.2 Why not: 効率順におすすめ遊びを自動ソートする

効率プレイを促し、幼少期の「比較しない」「好きな遊びを大切にする」感覚を損ねる。おすすめは「近道」ではなく「めあてに関係する」程度に留める。

### 7.3 Why not: 成長グラフを大きく表示する

グラフは分かりやすいが、数字ゲームの印象が強くなる。必要な数値は素養カードで十分に見せ、主役はエピソードに置く。

### 7.4 Why not: すべてのイベントをフルスクリーン化する

重要度の低い通知まで重くなり、テンポを損なう。Notice / Event / Mission / Growth の階層で強弱をつける。

### 7.5 Why not: ドックに全機能を並べる

片手操作は維持できるが、アイコン数が増えると意味が曖昧になる。主要行動を 3-5 個に絞り、記録系は `📖 アルバム` にまとめる。

## 9. オープン質問

足りない情報として、次の判断があると実装 SPEC に落とし込みやすい。

1. 幼少期の UI トーンは、より「絵本寄り」か、現状の「ゲーム HUD 寄り」か。
2. S2 の「今日の焦点」は、毎日必ず出すか、ミッション発生中だけ出すか。
3. プロフィール導線名は `📖 アルバム` / `📊 自分` / `🌱 きろく` のどれを採用するか。
4. 親・先生コメントの文体は、現実寄りか、少し物語寄りか。
5. 将来の中高生以降で、UI トーンを年齢に合わせて変化させるか。
